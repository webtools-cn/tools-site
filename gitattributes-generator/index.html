常见问题

什么是 .gitattributes？

.gitattributes 是 Git 的属性配置文件，用于控制 Git 如何处理特定文件类型。包括行尾转换（CRLF/LF）、二进制文件检测、差异算法、合并策略、Git LFS 规则等。通常放在项目根目录。

行尾归一化是什么意思？

行尾归一化（text=auto）是 Git 自动处理换行符的机制：检出时根据操作系统自动转换（Windows 用 CRLF，macOS/Linux 用 LF），提交时统一转为 LF。这能防止跨平台协作时出现行尾混乱问题。

什么是 Git LFS？

Git LFS（Large File Storage）是 Git 的大文件存储扩展。通过 .gitattributes 配置特定文件类型使用 LFS（如 *.psd filter=lfs diff=lfs merge=lfs -text），Git 会将文件内容存储在远程 LFS 服务器上，仓库中只保留指针文件。
