❓ 常见问题

什么是HTTP状态码？

HTTP状态码是服务器对客户端请求的响应状态标识，由3位数字组成，分为5类：1xx信息、2xx成功、3xx重定向、4xx客户端错误、5xx服务器错误。

最常见的HTTP状态码有哪些？

最常见的是200(成功)、301(永久重定向)、404(未找到)、500(服务器内部错误)、403(禁止访问)。

404和500有什么区别？

404表示客户端请求的资源不存在，是客户端错误；500表示服务器内部发生错误，是服务端错误。

数据会上传到服务器吗？

不会。所有查询和筛选在浏览器本地完成，数据绝不上传服务器。

支持搜索功能吗？

支持。可以按状态码数字、英文名称或中文描述进行实时搜索筛选。

REST API开发中哪些状态码最常用？

REST API常用：200(获取成功)、201(创建成功)、204(删除成功)、400(请求错误)、401(未认证)、403(无权限)、404(资源不存在)、500(服务器错误)。

扩展知识

HTTP状态码由RFC 7231（原RFC 2616）定义，分为5类：1xx（信息性状态码，表示请求已接收继续处理）、2xx（成功状态码，表示请求已成功处理）、3xx（重定向状态码，表示需进一步操作完成请求）、4xx（客户端错误，表示请求包含语法错误或无法完成）、5xx（服务器错误，表示服务器处理请求出错）。非标准状态码如Nginx的497、Cloudflare的520-527等不在本工具范围内。
