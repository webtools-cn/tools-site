使用教程

本键盘按键测试工具操作简单直观，以下是详细的使用指南：

基本测试：打开工具页面后，直接按下键盘上的任意按键，虚拟键盘会实时高亮显示对应的按键位置，同时下方的按键详情面板会显示该按键的完整信息，包括 key、code、keyCode、location 等属性。您可以逐个测试键盘上的每一个按键，确认它们是否正常工作。

组合键测试：工具支持同时检测多个按键的按下状态。您可以尝试按下常见的组合键，如 Ctrl+C（复制）、Ctrl+V（粘贴）、Alt+Tab（切换窗口）等，观察虚拟键盘上多个按键同时高亮的效果，以及按键详情面板中修饰键状态的显示变化。

无冲测试：在测试过程中，工具会自动统计同时按下的最大按键数量。如果您的键盘支持全键无冲（NKRO），您可以尝试同时按下尽可能多的按键，观察统计面板中的"最高同时"数值。普通薄膜键盘通常支持2-3键无冲，而高端机械键盘可能支持6键或全键无冲。

按键历史：每次按键操作都会被记录在历史列表中，包括按键值、编码、修饰键状态和按下时间。您可以随时查看历史记录，或者点击"清空"按钮清除所有记录。历史记录有助于您追踪测试过程和排查问题。

扩展知识

键盘事件模型：现代浏览器支持三种键盘事件：keydown（按键按下）、keypress（按键字符输入，已废弃）和 keyup（按键释放）。本工具主要使用 keydown 和 keyup 事件来检测按键状态。keydown 事件在按键按下时触发，如果按住不放会持续触发（重复）；keyup 在按键释放时触发一次。

key vs code vs keyCode：key 属性表示按键产生的字符（如 "a"、"Enter"），受键盘布局影响；code 属性表示物理按键位置（如 "KeyA"、"Enter"），不受布局影响，是更可靠的选择；keyCode 是旧的数字编码（如 65），已被弃用但仍有兼容性需求。现代Web开发应优先使用 key 和 code。

键盘扫描矩阵：大多数键盘使用扫描矩阵来检测按键。按键被排列成行和列的网格，控制器依次扫描每一行和列，检测哪些交叉点被按下。这种设计的限制是：如果同时按下形成矩形的四个角上的键，控制器可能无法正确识别（鬼键现象），这就是键盘无冲设计的由来。

全键无冲实现：实现全键无冲主要有两种方式：USB HID 报告描述符中声明支持全键无冲（大多数高端机械键盘），或者使用双模/三模通信协议。部分键盘通过固件优化可以在USB模式下实现全键无冲，但在蓝牙模式下可能降级为6键无冲。

常见问题（FAQ）

什么是全键无冲（NKRO）？

全键无冲（NKRO, N-Key Rollover）指键盘能够同时识别所有按键的按下状态，不会因为同时按下多个键而产生按键冲突或丢失信号。这对于游戏玩家和专业用户尤为重要。

key、code 和 keyCode 有什么区别？

key 表示按键的实际字符值（如 'a' 或 'Enter'），code 表示物理按键位置（如 'KeyA' 或 'Enter'），keyCode 是过时的数字编码（如 65）。现代开发应优先使用 key 和 code 属性。

笔记本键盘和机械键盘测试结果一样吗？

测试结果取决于键盘本身的硬件规格。机械键盘通常支持更高的无冲键数（如6键或全键无冲），而普通薄膜键盘可能只支持2-3键无冲。工具会显示每个按键的响应情况，但不会改变键盘本身的硬件能力。

为什么有些按键检测不到？

某些系统级快捷键（如 Ctrl+Alt+Del、Win+L 等）会被操作系统拦截，浏览器无法捕获。此外，某些Fn组合键是硬件层面处理的，浏览器也无法检测。这是正常的安全机制。

测试工具需要联网吗？

不需要。本工具完全在浏览器本地运行，所有按键检测都在您的设备上完成，不会将任何数据上传到服务器。您可以断开网络后继续使用。

如何测试组合键是否正常工作？

本工具支持组合键检测。您只需同时按下多个键（如 Ctrl+C），工具会显示所有按下的键及其修饰键状态。您可以在按键历史区域查看组合键的完整信息。

虚拟键盘支持所有键盘布局吗？

虚拟键盘显示的是标准美式布局（QWERTY），但实际检测的是物理按键。无论您使用什么键盘布局（如DVORAK、AZERTY），code属性都会正确反映物理按键位置。key属性则会根据您的系统键盘布局显示对应的字符。

为什么按住按键不放会持续触发？

这是浏览器的自动重复机制（Auto-repeat）。当您按住某个按键时，操作系统会以一定频率发送重复的keydown事件（通常是每秒30次左右），直到您释放按键。这是正常的系统行为，并非工具bug。
