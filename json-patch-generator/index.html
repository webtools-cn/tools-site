JSON Patch生成器能做什么？

JSON Patch生成器是一款免费在线开发者工具，通过对比两个JSON文档自动生成RFC 6902标准的JSON Patch补丁。支持add、remove、replace、move、copy、test六种操作，可应用补丁验证结果是否与目标一致。适用于API部分更新、配置变更追踪、版本差异同步等场景。所有计算在浏览器本地完成，数据绝不上传服务器。

核心功能

  • 自动对比：递归对比两个JSON，自动生成最优补丁操作
  • 6种操作：add/remove/replace/move/copy/test全覆盖
  • 应用验证：将补丁应用到原始JSON，验证结果是否正确
  • 可视化列表：每个操作用颜色标签区分，直观易读
  • JSON Pointer：自动生成RFC 6901标准路径

使用教程

1. 在"原始JSON"输入框粘贴源JSON数据。

2. 在"目标JSON"输入框粘贴修改后的JSON数据。

3. 点击"生成Patch"查看差异操作列表和补丁代码。

4. 点击"应用Patch验证"确认补丁正确性。

扩展知识

JSON Patch（RFC 6902）是一种描述JSON文档变更的标准格式，由一组有序操作组成。每个操作包含op（操作类型）、path（JSON Pointer路径）和value（值，部分操作需要）。JSON Pointer（RFC 6901）用/分隔路径，特殊字符~和/需转义为~0和~1。JSON Patch是原子操作——要么全部成功，要么全部失败。test操作可用于条件判断，值不匹配时整个Patch失败。与JSON Merge Patch（RFC 7396）相比，JSON Patch更强大，能处理数组操作，但语法更复杂。

❓ 常见问题

什么是JSON Patch？

RFC 6902定义的JSON文档修改格式，用操作列表描述变更，常用于API部分更新。

支持哪些操作？

6种：add、remove、replace、move、copy、test。

JSON Pointer是什么？

RFC 6901定义的路径表达式，用/分隔键名，如/user/name。数组用数字索引。

和JSON Merge Patch的区别？

JSON Patch用显式操作列表，可处理数组。Merge Patch用合并方式，更简单但不能处理数组部分修改。

数据会上传到服务器吗？

不会。所有对比和补丁生成在浏览器本地完成。
