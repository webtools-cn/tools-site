关于Java格式化

Java代码格式化遵循Google Java Style Guide和Sun编码规范。本工具帮助你自动美化Java代码：

• 自动缩进 — 识别类、方法、控制流层级，统一缩进
• 大括号风格 — 支持K&R风格（同行{）和Allman风格（换行{）
• 运算符空格 — 在 =, +, -, == 等运算符周围添加规范空格
• 空行清理 — 移除连续多余空行，方法间保留合理间距
• Import排序 — 将import语句按字母顺序排列

注意：本工具实现基础格式化，不支持复杂的AST分析（如Lambda表达式内的精细格式化）。所有处理在浏览器本地完成。

常见问题 (FAQ)

格式化会改变代码逻辑吗？

不会。格式化只调整代码的排版和空白字符，不改变任何代码逻辑。格式化前后的代码在功能上完全等价。

K&R风格和Allman风格有什么区别？

K&R风格（同行）：左大括号与语句在同一行，如 if (x) {。Allman风格（换行）：左大括号单独一行。两种风格都不影响代码功能，只是排版习惯不同。

支持Java的哪些语法特性？

支持类、接口、枚举、注解、泛型、Lambda表达式、switch表达式等Java基础语法结构的格式化。对于非常复杂的嵌套结构，建议结合IDE的格式化功能使用。
