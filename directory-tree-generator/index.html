常见问题

    
目录树生成器支持哪些输入方式？
支持三种：拖拽上传文件夹自动读取、手动输入路径列表、粘贴已有文件列表文本。

    
如何过滤不需要的文件和目录？
在排除框中输入目录名或文件名，用逗号分隔。内置node_modules/.git/dist等常见预设。

    
生成的目录树可以直接用在README中吗？
可以。选择Markdown代码块输出格式，一键复制后直接粘贴到README.md。

    
数据会上传到服务器吗？
完全不会！所有文件读取和目录树生成都在浏览器本地完成，文件数据绝不上传。

    
支持多深的目录层级？
理论上无限制，建议不超过20层。可设置最大显示深度，超过的用省略号表示。

    
可以自定义目录树的样式吗？
可以。支持Unicode/ASCII/简洁三种连接线风格，可调缩进空格数、排序方式等。

    
上传文件夹时浏览器兼容性如何？
Chrome/Edge/Firefox主流浏览器均支持。Safari部分版本可能不支持，可使用手动输入方式。

  









    
目录树生成器能做什么？

    
目录树生成器是一款免费在线工具，可以将项目目录结构转换为ASCII树形图。支持上传文件夹自动读取、手动输入路径列表两种方式，可自定义连接线样式、缩进、过滤规则和输出格式。生成的目录树可直接用于README文档、项目说明、代码审查等场景。所有处理在浏览器本地完成，文件数据绝不上传服务器。

使用教程

    
1. 选择输入方式：拖拽文件夹到上传区域，或在文本框中手动输入文件路径（每行一个）

    
2. 设置过滤规则：在排除框中输入要忽略的目录名或文件名，用逗号分隔

    
3. 调整显示选项：选择连接线样式、缩进空格数、是否显示文件大小、排序方式等

    
4. 点击"生成目录树"按钮，在输出区域查看结果

    
5. 点击"复制结果"将目录树复制到剪贴板，或选择Markdown格式直接粘贴到README.md

    
应用场景

    
场景1：编写项目README

    
在GitHub项目README.md中展示项目目录结构，帮助新贡献者快速了解代码组织方式。使用Markdown代码块输出，直接粘贴即可。

    
场景2：代码审查文档

    
在代码审查或技术文档中引用项目结构，配合说明文字解释架构设计，比截图更清晰、可搜索。

    
场景3：项目迁移规划

    
在项目重构或迁移前，生成当前目录树作为参考，对比新旧结构，确保不遗漏文件。

    
场景4：教学演示

    
在编程教学中展示标准项目结构，学生可直观理解MVC、分层架构等组织方式。

    
扩展知识

    
ASCII目录树是Unix/Linux系统中tree命令的输出格式，已成为展示项目结构的事实标准。标准格式使用Unicode制表符（├──、└──、│）或ASCII字符（|--、+--、|）表示层级关系。在Markdown中使用三个反引号包裹的代码块展示目录树，可保持等宽字体对齐。常见的过滤规则包括排除版本控制目录（.git/.svn）、依赖目录（node_modules/vendor）、构建输出（dist/build）和系统文件（.DS_Store/Thumbs.db）。
