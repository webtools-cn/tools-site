什么是 XML Sitemap？

XML Sitemap（可扩展标记语言网站地图）是一种标准化文件格式，用于向搜索引擎提供网站中可供索引的页面完整列表。Sitemap文件遵循 sitemaps.org 协议规范，是SEO（搜索引擎优化）技术架构中的核心组成部分。

每个Sitemap条目可以包含四个关键字段：<loc>（页面URL，唯一必需字段）、<lastmod>（最后修改日期）、<changefreq>（预计更新频率）和<priority>（页面相对优先级）。这些信息帮助搜索引擎爬虫更智能地分配抓取资源，优先处理重要且更新频繁的页面。

Sitemap 协议规范

Google、Bing、百度等主流搜索引擎均支持Sitemap协议。单个Sitemap文件最多包含50,000个URL，文件大小（未压缩）不得超过50MB。如果网站规模超出此限制，应采用Sitemap索引文件（Sitemap Index）将多个Sitemap文件组合管理。

Sitemap文件必须采用UTF-8编码，所有URL需要经过HTML实体编码（特别是&、<、>等特殊字符）。Sitemap文件通常放置在网站根目录（如 https://example.com/sitemap.xml），并通过robots.txt或站长工具提交给搜索引擎。

使用教程

本Sitemap生成器操作简单且功能完善，以下是详细的使用指南：

第一步：输入URL列表。在顶部文本框中批量粘贴网站的URL地址，每行一个。支持直接从浏览器控制台复制、从Excel表格粘贴或从现有Sitemap文件中提取。URL必须包含完整的协议头（https:// 或 http://），建议使用绝对路径。粘贴完成后点击「解析URL」按钮，工具会自动提取并格式化每个URL。

第二步：配置URL属性。解析后，每个URL会显示为一个可配置的行，包含四个属性：URL地址（可编辑）、优先级（0.0-1.0滑块）、更新频率（下拉选择）和最后修改日期（日期选择器）。您可以逐条设置，也可以使用「统一优先级」和「统一频率」按钮批量应用。

第三步：生成并导出。点击「生成Sitemap」按钮后，工具会根据Sitemap协议规范生成标准的XML格式内容。生成的XML会显示在结果区域，您可以点击「复制XML」复制到剪贴板，或点击「下载sitemap.xml」直接保存文件。下载后上传到网站根目录，然后提交到各大搜索引擎站长平台即可。

应用场景

Sitemap生成器在网站运营和SEO优化中有着广泛的应用场景：

新站收录加速：新上线的网站往往面临搜索引擎收录慢的问题。通过生成Sitemap并主动提交给Google Search Console和百度站长平台，可以显著缩短搜索引擎发现新页面所需的时间。对于拥有数百甚至数千页面的电商网站、博客站点和内容平台，Sitemap是确保所有页面都被发现的必备工具。

大型网站结构优化：当网站页面数量超过数千时，站内链接结构可能变得复杂，搜索引擎爬虫可能无法遍历所有深层页面。Sitemap为搜索引擎提供了直达每个页面的"地图"，特别是对于通过JavaScript动态渲染、需要登录才能访问的页面，Sitemap的引导作用尤为重要。

内容更新通知：通过设置lastmod（最后修改日期）和changefreq（更新频率）字段，Sitemap可以向搜索引擎传达页面内容的新鲜度信息。对于新闻网站、博客和技术文档站点，及时更新Sitemap中的修改日期有助于搜索引擎更快抓取更新后的内容，提升内容的时效性排名。

扩展知识

Sitemap索引文件：当单个Sitemap文件超出50,000个URL或50MB限制时，需要使用Sitemap索引文件。索引文件本身也是一个XML文件，使用<sitemapindex>根元素，其中每个<sitemap>条目引用一个子Sitemap文件的URL。这种分层结构使得大型网站（如拥有数百万页面的电商平台、门户网站）也能高效管理Sitemap体系。

图片与视频Sitemap：除了标准网页Sitemap，Google还支持图片Sitemap（Image Sitemap）和视频Sitemap（Video Sitemap），分别用于帮助搜索引擎更好地发现和索引网站中的媒体内容。图片Sitemap可以包含图片标题、说明文字和地理定位信息；视频Sitemap可以包含视频标题、描述、时长、缩略图URL等丰富元数据，提升富媒体内容在搜索结果中的展示效果。

Sitemap与robots.txt的配合：robots.txt文件中的Sitemap声明（如Sitemap: https://example.com/sitemap.xml）是搜索引擎发现Sitemap文件的重要途径之一。即使不通过站长工具手动提交，只要robots.txt中正确声明了Sitemap路径，搜索引擎爬虫在抓取网站时也会自动读取。最佳实践是同时采用robots.txt声明和站长平台提交双重策略。

常见问题 (FAQ)

什么是XML Sitemap？

XML Sitemap（网站地图）是一种XML格式的文件，用于向搜索引擎提交网站中所有可供抓取的页面URL列表。它帮助搜索引擎更快、更全面地发现和索引网站内容，提升SEO效果。

Sitemap有URL数量限制吗？

根据Sitemap协议，单个sitemap.xml文件最多包含50,000个URL，文件大小不超过50MB（未压缩）。如果超出限制，可以拆分为多个sitemap文件并通过sitemap索引文件进行管理。

生成的Sitemap需要提交到哪里？

生成sitemap.xml文件后，需要将其上传到网站根目录，然后提交到Google Search Console、百度站长平台、Bing Webmaster Tools等搜索引擎站长工具中。也可以在robots.txt文件中添加Sitemap声明。

priority优先级如何设置？

priority表示URL相对于网站其他页面的重要性，取值范围0.0到1.0。建议：首页1.0，栏目页0.8，内容页0.6-0.5，标签/分类页0.3。注意：priority只是建议值，搜索引擎不会严格按照此值排序。

changefreq更新频率应该设置多久？

changefreq表示页面内容预计的更新频率。常见设置：首页daily或weekly，博客文章weekly或monthly，产品页monthly，联系我们等静态页面yearly或never。这个值同样是建议性的，搜索引擎会根据实际抓取行为调整。

Sitemap中必须包含哪些字段？

Sitemap中唯一必需的字段是loc（页面URL）。priority、lastmod和changefreq都是可选字段。即使只提供loc字段，Sitemap也是有效的。添加可选字段可以帮助搜索引擎更好地理解网站结构和更新策略。

这个工具安全吗？我的URL数据会被上传吗？

绝对安全。本工具完全在浏览器中运行，所有数据处理都在本地完成，URL列表和生成的XML文件不会上传到任何服务器。您可以放心使用，无需担心数据泄露。
