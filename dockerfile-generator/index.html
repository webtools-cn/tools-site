📖 Dockerfile 优化建议

• 使用 .dockerignore 排除 node_modules、.git 等文件
• 合并 RUN 指令减少镜像层数
• 使用 --no-cache 或 --no-install-recommends 减小体积
• 固定基础镜像版本标签而非使用 latest
• 多阶段构建可将镜像体积减少 5-10 倍

常见问题 (FAQ)

Dockerfile是什么？

Dockerfile是用于构建Docker镜像的文本文件，包含一系列指令（FROM、RUN、COPY、CMD等），描述了如何从基础镜像开始构建应用镜像。每个指令创建一个新的镜像层。

多阶段构建有什么好处？

多阶段构建允许在同一个Dockerfile中使用多个FROM语句。第一阶段安装编译工具构建应用，第二阶段只复制编译产物，大幅减小最终镜像体积。例如Go应用可以从1GB+减小到20MB左右。

如何选择基础镜像？

选择基础镜像的原则：alpine系列最小但兼容性差，slim系列较小且兼容性好，full标准版最大但工具齐全。生产环境推荐使用alpine或slim版本以减小镜像体积。
