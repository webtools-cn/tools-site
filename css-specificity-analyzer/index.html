CSS优先级分析器能做什么？

CSS优先级分析器是一款免费在线开发者工具，能够实时计算CSS选择器的特异性（Specificity）分数，并可视化对比多个选择器的优先级高低。它解析选择器中的ID、类、属性、伪类、元素、伪元素等组成部分，计算(0,A,B,C)特异性值，按优先级排序展示。适用于前端开发者调试CSS覆盖问题、理解CSS层叠规则、优化选择器设计。所有计算在浏览器本地完成。

核心功能

  • 特异性计算：精确计算每个选择器的(A,B,C)特异性分数
  • 可视化对比：彩色方块直观展示ID/类/元素三级权重
  • 优先级排序：按特异性分数从高到低排序，标注排名
  • 批量分析：支持多行输入，一次分析多个选择器
  • 详细解析：显示选择器各组成部分的类型和计数

使用教程

1. 在输入框中输入CSS选择器，每行一个。

2. 点击"分析"或使用Ctrl+Enter，工具立即计算所有选择器的特异性。

3. 查看结果：彩色方块展示三级权重，排名标注优先级顺序。

4. 红色=ID权重，黄色=类/属性/伪类权重，青色=元素/伪元素权重。

5. 复制或下载分析报告，用于团队讨论或文档记录。

扩展知识

CSS特异性（Specificity）是浏览器决定同一元素上多条CSS规则优先级的算法。用(A,B,C)表示：A=ID选择器数量，B=类选择器+属性选择器+伪类数量，C=元素选择器+伪元素数量。通配符(*)不计分。比较规则：从左到右比较，A层大则胜出；A相同比B，B相同比C。特殊情况：inline样式优先级为(1,0,0,0)；!important覆盖一切非!important；:where()特异性为0；:is()取参数中最高特异性。CSS Cascade Layers（@layer）提供了新的层叠控制方式，layer内的规则优先级低于未分层的规则。

❓ 常见问题

CSS优先级是如何计算的？

CSS优先级用(A,B,C)三个数表示：A=ID选择器数量，B=类选择器+属性选择器+伪类数量，C=元素选择器+伪元素数量。通配符*不计分。比较时从左到右，A最大则优先级最高。

!important如何影响优先级？

!important会将声明提升到最高优先级层，覆盖任何非!important声明。当多个!important冲突时，再按正常优先级规则比较。建议避免过度使用!important。

inline样式优先级是多少？

inline样式（style属性）的优先级为(1,0,0,0)，高于任何选择器的优先级(0,A,B,C)，但低于!important。

数据安全吗？

完全安全。所有计算在浏览器本地完成，CSS内容绝不上传服务器。

支持CSS选择器4吗？

支持大多数CSS3和CSS4选择器，包括:nth-child()、:is()、:where()、:has()等。:where()特异性为0，:is()取参数中最高特异性。

为什么我的样式被覆盖了？

样式被覆盖通常是因为另一个选择器的特异性更高，或者后声明的同等特异性规则覆盖了前者。用此工具对比冲突选择器的特异性分数即可定位原因。

如何降低选择器优先级？

减少ID选择器和嵌套层级，使用BEM等命名规范（单一类名），或使用:where()包裹选择器（特异性为0）。
