使用教程

JSON转TypeScript接口生成器是一款面向前端开发者的实用工具，能够将JSON数据自动转换为规范的TypeScript类型定义。以下是详细的使用指南：

基础使用方法：在左侧JSON输入框中粘贴你的JSON数据（支持从API响应、配置文件或数据库导出结果中复制）。确保JSON格式正确后，点击「生成TypeScript」按钮，工具会在右侧输出对应的TypeScript接口定义。你可以直接将结果复制到项目的.ts文件中。

高级选项设置：工具提供多种选项供你定制生成结果。「使用type别名」开关控制生成interface还是type，两者在TypeScript中功能等价但风格不同。开启「生成JSDoc注释」后，工具会自动为每个属性添加描述性注释。开启「可选字段」后，如果某个属性值为null或在数组元素中不始终存在，该属性会被标记为可选（带?修饰符）。「严格null检查」选项会在类型中包含null联合类型，适配strictNullChecks配置。

嵌套对象处理：对于嵌套的JSON对象，工具会自动递归生成独立的接口定义，并为嵌套接口命名（如Root、RootProfile等）。数组会被分析为所有元素类型的联合类型（如(string|number)[]）。你可以通过重命名使生成的代码更符合你的项目规范。

应用场景

JSON转TypeScript接口生成器在多个开发场景中都能发挥重要作用。以下是几个典型的应用案例：

API接口类型定义：当后端提供RESTful API接口文档时，通常只返回JSON示例数据。你可以将API响应示例粘贴到工具中，快速生成对应的TypeScript类型定义，然后在项目中导入使用。这大大减少了手动编写接口类型的工作量，同时降低了类型错误的风险。

配置文件类型安全：现代前端项目（如Next.js、Vite）通常使用配置文件（如next.config.js、vite.config.ts）。你可以将配置示例转为TypeScript接口，获得IDE的智能提示和类型检查，避免拼写错误和配置遗漏。

数据迁移与文档化：在数据迁移或重构项目中，工具可以帮助你快速理解未知JSON数据的结构。生成的TypeScript接口天然具备文档化效果，团队成员可以直观地了解数据模型的层次和属性类型。

扩展知识

TypeScript接口与类型别名：在TypeScript中，interface和type在很多场景下可以互换，但存在细微差别。interface支持声明合并（declaration merging），即同名的多个interface定义会自动合并，这在扩展第三方库类型时非常有用。type则支持更复杂的类型操作，如条件类型、映射类型等。对于简单的对象结构，两者没有功能差异，可根据团队偏好选择。

JSON Schema与TypeScript：除了手动编写TypeScript接口，你还可以使用JSON Schema定义数据结构，再通过工具自动生成TypeScript类型。JSON Schema的优势在于它是语言无关的，可以被多种语言和工具消费。本工具提供了一种更轻量的方案——直接从JSON示例推断类型，无需学习JSON Schema语法。

类型推断的局限性：自动推断虽然方便，但并非万能。例如，工具无法区分string和string枚举（如&literal;status: 'active' | 'inactive'&literal;），也无法知道某个number实际上是ID还是普通数值。因此，生成的类型定义通常需要人工审核和调整，特别是用于生产环境时。

常见问题解答

JSON转TypeScript接口生成器支持哪些类型推断？

支持string、number、boolean、null、数组、对象等基本类型，以及ISO日期字符串自动识别。对于数组，工具会分析所有元素类型并生成联合类型。对于嵌套对象，会自动递归生成独立的接口定义。

生成的TypeScript接口可以直接使用吗？

可以。生成的接口符合TypeScript标准语法，可直接复制到.ts文件中使用。建议根据实际业务需求适当调整可选属性标记和类型精度。

支持生成type别名还是interface接口？

支持两种模式切换。默认生成interface，你可以在设置中选择使用type别名。两者在TypeScript中功能等价，可根据团队代码规范选择。

如何处理JSON中的空值和可选字段？

工具提供了可选字段检测功能。开启后，如果某个属性在部分数据中存在而在其他数据中缺失，该属性会被标记为可选（带?）。对于null值，会生成|null联合类型。

数据安全性如何保障？

所有处理均在浏览器本地完成，数据不会上传到任何服务器。工具不依赖任何外部API，是纯前端实现，你可以放心粘贴敏感数据进行转换。
