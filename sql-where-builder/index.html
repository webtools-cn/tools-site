扩展知识

SQL WHERE子句用于过滤记录，支持比较操作符、逻辑操作符（AND/OR/NOT）、范围操作符（BETWEEN/IN）、模式匹配（LIKE）和空值判断（IS NULL）。在复杂查询中，建议使用参数化查询而非字符串拼接，以防止SQL注入攻击。索引优化时应注意WHERE条件的字段顺序和选择性。

❓ 常见问题

支持哪些SQL操作符？

支持=, !=, >, <, >=, <=, LIKE, NOT LIKE, IN, NOT IN, BETWEEN, IS NULL, IS NOT NULL等常见操作符，覆盖大部分SQL查询场景。

可以构建嵌套条件吗？

可以。通过添加条件组实现嵌套逻辑，每个组有自己的AND/OR连接符，支持任意层级的嵌套条件组合。

生成的SQL是否安全？

本工具生成标准SQL WHERE子句，但生成的SQL应作为参考和起点。在生产环境中使用前，建议结合参数化查询防止SQL注入。

支持哪些数据库？

生成标准SQL语法，兼容MySQL、PostgreSQL、SQLite、SQL Server、Oracle等主流数据库。

如何添加多个条件？

点击"添加条件"按钮新增一行条件，选择字段名、操作符和值，并设置AND/OR连接符。

生成的SQL可以复制吗？

可以。生成的SQL会实时显示在结果区域，支持一键复制和下载为SQL文件。
