HAR文件查看器 - Web性能分析利器

HAR文件查看器是一款专为Web开发者和运维工程师设计的在线工具。通过解析HTTP Archive（HAR）文件，您可以直观地查看浏览器与服务器之间的所有HTTP交互，包括请求瀑布图、响应时间分析、资源加载统计等关键性能指标。

核心功能

  • 请求瀑布图 - 可视化展示每个请求的DNS、TCP、SSL、TTFB、下载各阶段耗时
  • 智能筛选 - 按URL、资源类型、状态码快速过滤请求
  • 域名统计 - 按域名汇总请求数、传输大小和耗时
  • 请求详情 - 查看完整的请求头、响应头和请求体
  • 性能概览 - 总请求数、传输大小、页面加载时间等关键指标

适用场景

  • 性能优化 - 定位慢请求，分析资源加载瓶颈
  • 问题排查 - 检查请求状态码、响应头，诊断网络问题
  • 安全审计 - 检查请求中的敏感信息泄露
  • API调试 - 查看XHR/Fetch请求的完整交互过程

常见问题

什么是HAR文件？

HAR（HTTP Archive）是一种记录浏览器与服务器之间HTTP交互的JSON格式文件。它包含请求URL、方法、请求头、响应状态码、响应头、响应体大小、时间戳等完整信息，是Web性能分析和问题排查的重要工具。

如何导出HAR文件？

在Chrome浏览器中：1）打开开发者工具（F12）；2）切换到Network标签；3）执行需要记录的操作；4）右键点击请求列表，选择"Save all as HAR with content"。Firefox和Edge也支持类似操作。

HAR文件包含敏感信息吗？

是的，HAR文件可能包含Cookie、Authorization头、POST请求体等敏感信息。本工具完全在浏览器本地处理，不会上传任何数据到服务器。分享HAR文件前请务必检查并移除敏感信息。

支持多大的HAR文件？

本工具支持解析最大50MB的HAR文件。对于包含大量请求的文件，建议使用筛选功能按域名或类型过滤，以获得更好的浏览体验。

瀑布图中的时间指标代表什么？

瀑布图展示每个请求的各阶段耗时：DNS查询（紫色）、TCP连接（橙色）、SSL握手（绿色）、请求发送（深蓝）、等待响应（浅蓝）、内容下载（灰色）。通过分析各阶段耗时可以定位性能瓶颈。
