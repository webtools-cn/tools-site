在线随机数生成器能做什么？

我们的在线随机数生成器是一个纯前端工具——无需安装任何软件，打开网页即可使用。它支持从简单的随机数抽取到复杂的批量生成和统计分析，适用于抽奖摇号、数据采样、教学演示、游戏随机化等多种场景。

所有随机数均通过浏览器内置的 Web Crypto API 生成，这是浏览器中最高安全级别的随机数生成接口，基于操作系统提供的高熵随机源，确保每个数字出现的概率完全相等。

如何设置随机范围？

在单个生成或批量生成模式下，您可以自由设置最小值和最大值。支持任意整数区间，如 1-100、1000-9999，甚至包含负数的范围如 -50 到 50。系统会自动校验范围的合法性，确保最大值大于最小值。

  • 正整数范围：适合抽奖号码、骰子模拟、随机抽样
  • 大数范围：支持百万级，适合大型活动抽奖
  • 负数支持：支持包含负值的范围

如何使用抽奖摇号模式？

抽奖模式提供两种方式：范围抽奖适合按号码抽奖（如年会从 1-500 中抽取 10 个中奖号码）；名单抽奖适合按姓名抽奖（如从参与者名单中抽取中奖者）。两种模式均保证不重复——已中奖的号码或人员不会再次被抽中。

批量生成与分布统计

批量生成模式可一次性生成最多 10,000 个随机数，支持复制到剪贴板或直接下载为文本文件。开启"逗号分隔"选项后，可直接粘贴到 Excel 进行进一步分析。

分布统计模式可以验证随机数的均匀性。在理想情况下，每个数字出现的次数应该大致相等。我们提供频率直方图以及平均值、中位数、标准差等统计指标，帮助您直观了解随机分布的特征。

使用教程

选择"单个生成"模式，设置最小值和最大值，点击"生成"按钮即可获得随机数。批量模式可一次生成最多 10,000 个数字，支持复制到剪贴板或直接下载为文本文件。开启逗号分隔选项后可直接粘贴到 Excel 分析。

抽奖模式提供两种方式："范围抽奖"用于按号码抽奖（如从 1-500 中抽取 10 个中奖者），"名单抽奖"用于按姓名抽奖（如从参与者名单中抽取中奖者）。两种模式均保证不重复。

分布统计模式用于验证随机数的均匀性。系统生成指定数量的随机值并计算每个数字的出现频率，以直方图展示，同时给出平均值、中位数和标准差等指标。

应用场景

随机数生成器在技术领域和日常生活中都有广泛应用。在技术领域，密码学安全随机数用于生成密钥、会话令牌和验证码。在数据分析中，随机抽样是统计调查和质量控制的基础。在游戏开发中，随机数驱动战利品掉落、伤害计算和程序化地图生成。

在日常生活中，随机数用于抽奖活动、随机分组和随机排序。本工具采用密码学安全随机数生成器（基于 Web Crypto API），确保公平性和不可预测性，适用于正式抽奖和安全敏感场景。

扩展知识

在计算机上生成"真正的"随机数极其困难，因为计算机程序本质上是确定性的。常见的解决方案是利用物理熵源（硬件噪声、鼠标移动、键盘敲击间隔等）产生种子，然后生成密码学安全的伪随机数。本工具的 Web Crypto API 使用操作系统的熵源，产生满足密码学安全要求的随机性。

普通的 Math.random() 不适合安全敏感场景，因为其确定性实现可以被预测。密码学安全伪随机数生成器（CSPRNG）使用复杂的数学变换，确保即使知道部分输出也无法推断其余部分。对于抽奖、密码生成、加密密钥等高安全场景，务必使用 CSPRNG。

常见问题（FAQ）

这个随机数生成器公平吗？

完全公平。我们使用浏览器内置的 Web Crypto API（crypto.getRandomValues）生成密码学安全随机数，而非普通的 Math.random()。该算法基于操作系统提供的熵源（如硬件噪声、鼠标移动、键盘敲击间隔等），确保指定范围内每个数字出现的概率完全相等，且无法被预测或操控。

抽奖模式支持多少参与者？

抽奖模式支持两种形式：① 范围抽奖——设定最小值和最大值，从范围内抽取指定数量的不重复中奖号码，支持百万级范围；② 名单抽奖——粘贴参与者名单（每行一个），从名单中随机抽取指定数量的中奖者。名单长度没有硬性限制（建议不超过1万行以保证性能）。所有抽奖均在浏览器本地完成。

批量模式一次最多生成多少个随机数？

批量生成模式支持单次生成最多 10,000 个随机数。如果开启"去重"选项，生成数量不能超过范围大小（例如 1-100 的范围最多生成 100 个不重复数字）。结果支持一键复制和导出为文本文件下载。

分布统计功能能做什么？

分布统计功能可以直观展示生成随机数的统计特征，包括：每个数字的出现频率直方图，以及平均值、中位数、标准差、最大值、最小值等指标。这有助于验证随机分布的均匀性，适用于教学演示和数据分析场景。

这个工具的随机算法和普通随机数有什么区别？

普通的 Math.random() 是基于确定性算法种子的伪随机数生成器（PRNG），虽然满足日常需求，但不适合安全敏感场景。我们的工具使用 crypto.getRandomValues()，调用操作系统的高熵随机源（如 /dev/urandom、Windows CSP、硬件 RNG）。这是密码学安全随机数生成器（CSPRNG），满足密码学安全要求，更适合抽奖、密钥生成等对公平性要求高的场景。

数据会上传到服务器吗？

绝对不会。所有随机数生成、名单抽奖和统计计算均在浏览器本地完成，没有任何数据上传到服务器。您可以离线使用本工具，也可以放心输入任何参与者信息，数据完全在您的设备上处理。
