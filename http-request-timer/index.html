HTTP请求计时器能做什么？

HTTP请求计时器是一款纯前端在线工具，用于精确测量HTTP请求的各阶段耗时。输入目标URL，工具发送请求并利用浏览器Performance API记录DNS查询、TCP连接、TLS握手、请求发送、等待响应（TTFB）、内容下载等每个阶段的精确时间，以可视化时间条和详细数据展示结果。支持GET/POST/PUT/DELETE/HEAD方法，可自定义请求头和请求体，支持多请求历史对比。所有计时在浏览器本地完成。

核心功能

  • 全阶段计时：测量DNS、TCP、TLS、发送、TTFB、下载各阶段耗时
  • 可视化展示：时间条直观展示各阶段占比，一眼定位瓶颈
  • 多方法支持：GET/POST/PUT/DELETE/HEAD，可自定义请求头和请求体
  • 请求历史：保存最近10次请求结果，方便对比分析
  • 多格式导出：支持复制JSON和下载文件

使用教程

步骤1：选择HTTP方法，在URL输入框输入目标地址。

步骤2：如需自定义请求头或请求体，在对应输入框填写JSON格式内容。

步骤3：点击"发送请求"或Ctrl+Enter，等待请求完成。

步骤4：查看各阶段耗时时间条和数据，点击"复制JSON"导出结果。

应用场景

API性能调试

开发者测量API端点的TTFB和总耗时，定位慢请求是DNS解析慢、网络延迟还是服务端处理慢。

CDN效果验证

对比开启和关闭CDN时的请求各阶段耗时，验证CDN加速效果。

跨区域延迟对比

多次请求同一URL，对比不同时间段的网络延迟变化，分析网络质量。

前端性能优化

测量关键资源加载的TTFB，优化首屏渲染依赖的API请求性能。

扩展知识

TTFB（Time To First Byte）是从发送请求到收到第一个响应字节的时间，包含DNS+TCP+TLS+服务端处理时间。浏览器Performance Resource Timing API提供详细的请求阶段数据：domainLookupStart/End为DNS阶段，connectStart/End为TCP阶段，secureConnectionStart/End为TLS阶段，requestStart到responseStart为TTFB，responseStart到responseEnd为下载阶段。优化TTFB通常需要服务端优化（如缓存、CDN、数据库查询优化）。

常见问题 (FAQ)

能测量哪些请求阶段？

可测量DNS查询、TCP连接、TLS握手、请求发送、等待响应(TTFB)、内容下载等各阶段耗时，以及总耗时。

为什么有些阶段显示0ms？

浏览器会缓存DNS和TCP连接，重复请求同一域名时DNS和TCP阶段可能显示0ms。建议清除缓存后首次测量。

支持POST请求吗？

支持GET和POST请求，可自定义请求头和请求体。

CORS限制怎么办？

浏览器CORS策略限制跨域请求。建议测试支持CORS的API，或使用浏览器扩展临时禁用CORS进行开发调试。

数据会上传服务器吗？

不会。请求从浏览器直接发送到目标URL，计时数据由浏览器Performance API本地计算，不经过任何中间服务器。

能导出测量结果吗？

可以。支持复制JSON格式结果和下载为文件，方便记录和对比分析。
