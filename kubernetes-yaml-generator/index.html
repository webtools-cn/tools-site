Kubernetes YAML生成器能做什么？

Kubernetes YAML生成器是一款免费在线K8s配置工具，通过可视化界面帮助用户快速生成标准的Kubernetes YAML配置文件。无需记忆复杂的YAML结构和API版本，只需填写应用名称、镜像、副本数、资源限制等参数，即可生成包含Deployment、Service、ConfigMap、Secret、Ingress等完整资源定义的YAML文件。所有生成在浏览器本地完成。

核心功能

  • Deployment配置：可视化设置镜像、副本数、资源限制、环境变量
  • Service配置：支持ClusterIP、NodePort、LoadBalancer三种类型
  • ConfigMap/Secret：键值对输入，Secret自动Base64编码
  • Ingress配置：配置域名路由、TLS证书、Ingress类
  • 一键生成：合并所有资源为多文档YAML，kubectl apply直接使用

使用教程

1. 在Deployment标签页填写应用名称、镜像、副本数等基本信息。

2. 按需配置Service、ConfigMap、Secret、Ingress等资源。

3. 点击"生成YAML"按钮，工具自动合并所有配置生成完整YAML。

4. 复制或下载YAML文件，使用kubectl apply -f部署到集群。

5. 使用Ctrl+Enter快速生成，Ctrl+Shift+C复制结果。

扩展知识

Kubernetes使用声明式配置，YAML文件描述资源的期望状态，K8s控制器负责将实际状态收敛到期望状态。API版本：apps/v1用于Deployment，v1用于Service/ConfigMap/Secret，networking.k8s.io/v1用于Ingress。最佳实践：始终设置resources limits防止资源争抢；使用readinessProbe/livenessProbe保障服务可用；通过nodeSelector/affinity实现调度策略；使用HPA自动扩缩容。

❓ 常见问题

什么是Kubernetes YAML？

Kubernetes YAML是定义K8s集群资源的声明式配置文件，描述Deployment、Service等资源的期望状态，kubectl apply即可创建或更新资源。

生成的YAML可以直接用kubectl apply吗？

可以。生成的YAML符合Kubernetes API规范，保存为.yaml文件后直接kubectl apply -f即可部署。

支持哪些K8s资源类型？

支持Deployment、Service、ConfigMap、Secret、Ingress等常用资源。

数据安全吗？

完全安全。所有YAML生成在浏览器本地完成，配置信息绝不上传服务器。

如何配置环境变量？

在Deployment配置中添加env字段，或引用ConfigMap/Secret中的值。本工具支持两种方式的可视化配置。

什么是资源限制？

resources limits定义容器的CPU和内存上限，requests定义最低保障。建议生产环境必须设置，防止资源争抢和OOM。

Secret数据如何编码？

Secret中的值必须Base64编码。本工具自动将输入值进行Base64编码，无需手动转换。
