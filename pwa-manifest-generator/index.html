📖 使用说明

PWA Manifest（Web App Manifest）是PWA必需的配置文件。在HTML中通过 <link rel="manifest" href="manifest.json"> 引入。

必填字段：只有 name 是必须的，但推荐填写 short_name、start_url、display 和 icons 以获得最佳体验。

图标建议：至少提供192x192和512x512两种尺寸的PNG图标。512x512图标用于启动画面，192x192用于桌面图标。

🎯 适用场景

  • PWA开发：快速生成标准manifest.json配置文件
  • 站点升级：为现有网站添加PWA支持，提升用户体验
  • 应用发布：准备Google Play（Trusted Web Activity）所需的配置
  • 学习参考：了解PWA Manifest各字段的含义和用法

❓ 常见问题

PWA Manifest必须放在网站根目录吗？

不一定，manifest.json可以放在任何位置。但强烈建议放在根目录，因为start_url和scope的相对路径是相对于manifest文件位置的。如果放在根目录，所有路径行为更可预测。

Manifest文件可以用其他名称吗？

虽然习惯用manifest.json，但可以是任何名称（如 app.json）。只需在HTML中通过 <link rel="manifest" href="app.json"> 正确引用即可。

修改manifest后需要更新Service Worker吗？

不需要。修改manifest.json后，用户下次打开网站时浏览器会自动读取最新版本。但部分字段（如图标）的更新可能需要用户重新添加到桌面才会生效。
