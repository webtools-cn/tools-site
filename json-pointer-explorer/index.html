❓ 常见问题

什么是JSON Pointer？

RFC 6901定义的JSON文档寻址标准，用/分隔路径引用值，如/user/name。

特殊字符如何转义？

~转义为~0，/转义为~1。例如键名a/b~c变为/a~1b~0c。

和JSONPath的区别？

JSON Pointer是RFC标准，精确引用单个值。JSONPath支持过滤通配符，但非标准。

空字符串Pointer代表什么？

空字符串""代表整个JSON文档的根。

数组如何引用？

数组用数字索引，如/items/0表示第一个元素，/-表示末尾追加位置。

数据会上传到服务器吗？

不会。所有处理在浏览器本地完成。

JSON Pointer路径浏览器能做什么？

JSON Pointer路径浏览器是一款免费在线开发者工具，输入JSON数据后自动遍历生成所有RFC 6901标准的JSON Pointer路径，显示每个路径对应的值和类型。支持路径搜索、类型筛选、值提取、键名转义等功能。适用于API开发调试、JSON Patch编写、JSON Schema路径引用等场景。所有处理在浏览器本地完成，数据绝不上传服务器。

核心功能

  • 自动路径生成：递归遍历JSON，生成所有RFC 6901标准Pointer路径
  • 类型筛选：按string/number/boolean/object/array/null筛选路径
  • 路径搜索：实时搜索路径或值内容
  • 值提取：点击路径查看完整值，一键复制
  • 转义工具：键名自动转义（~→~0，/→~1）

扩展知识

JSON Pointer（RFC 6901）是IETF定义的JSON文档寻址标准，用空字符串""表示整个文档，用/分隔的路径引用子值。特殊字符~和/在键名中需转义：~变为~0，/变为~1。JSON Pointer是JSON Patch（RFC 6902）和JSON Schema的核心依赖，用于定位操作目标。与JSONPath相比，Pointer更简单但只能引用单个确定位置，不支持过滤和通配符。
