❓ FAQ

        

            
What is a Web Component?

            
Web Components are a set of W3C standard technologies that allow developers to create reusable custom HTML elements. They consist of three core technologies: Custom Elements, Shadow DOM, and HTML Templates, enabling encapsulation and reuse.

        

        

            
What is Shadow DOM?

            
Shadow DOM is a DOM encapsulation technology that allows isolating a component's internal structure and styles from the rest of the page. CSS inside Shadow DOM doesn't affect the outside, and external CSS doesn't affect the inside, achieving true style encapsulation.

        

        

            
Why must component names contain a hyphen?

            
Per the HTML specification, custom element names must contain at least one hyphen (-) to distinguish them from standard HTML elements. This ensures namespace collision avoidance. For example, my-button and custom-card are both valid names.

        

        

            
What lifecycle callbacks do Web Components support?

            
Web Components support the following lifecycle callbacks: connectedCallback (called when element is inserted into DOM), disconnectedCallback (called when element is removed from DOM), adoptedCallback (called when element is moved to a new document), attributeChangedCallback (called when observed attributes change).
