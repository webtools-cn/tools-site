使用教程

第一步：上传PDF文件。点击上传区域选择PDF文件，或直接将文件拖拽到页面上。工具支持任意大小的PDF文件，由于采用纯前端处理技术，文件不会上传到服务器，确保您的数据隐私安全。

扩展知识

DPI与图片质量：DPI（每英寸点数）是衡量图片分辨率的重要指标。72 DPI适合屏幕显示，150 DPI是网页和一般文档的理想选择，200-300 DPI则适用于高质量打印。DPI越高，图片越清晰，但文件体积也会显著增加。

PNG vs JPEG：PNG格式采用无损压缩，支持透明背景，适合需要保留精确像素信息的场景，但文件通常较大。JPEG采用有损压缩，文件更小，适合照片和复杂图像，但不支持透明背景。

PDF.js 技术原理：本工具底层使用 Mozilla 开源的 PDF.js 库，它通过 JavaScript 在浏览器中解析PDF文件的内部结构，提取文本、图像和矢量图形信息，然后渲染到 HTML Canvas 元素上。整个过程完全在客户端完成，不需要服务器参与。

常见问题 FAQ

PDF转图片后图片质量如何？

工具支持自定义DPI，最高可达300 DPI。默认使用150 DPI，在清晰度和文件大小之间取得平衡。您可以根据实际需求在转换前调整DPI参数。

支持哪些图片格式？

目前支持PNG和JPEG两种格式。PNG格式支持透明背景，无损压缩；JPEG格式文件更小，适合网页和社交媒体使用。

PDF文件会发送到服务器吗？

不会。本工具完全在浏览器端运行，您的PDF文件不会上传到任何服务器，转换过程完全在您的设备上完成，确保数据隐私和安全。

一次最多能转换多少页？

理论上没有硬性限制，转换页数主要取决于您的设备性能和内存大小。对于大型PDF文件（数百页），建议分批转换或选择特定的页面范围。

转换后的图片可以打包下载吗？

可以。工具支持将转换后的所有图片打包成ZIP文件一次性下载，也可以单独下载某一张图片。打包下载功能同样完全在浏览器端完成。

为什么转换后的图片有模糊感？

如果感觉图片不够清晰，建议在转换前将DPI设置为200或300。部分扫描版PDF本身分辨率较低，转换后的图片质量受限于原始PDF的分辨率。

支持扫描版PDF吗？

支持。扫描版PDF本质上是嵌入在PDF中的图片，工具可以将其提取并转换为您选择的图片格式。但请注意，扫描版PDF的清晰度取决于原始扫描质量。

转换速度取决于什么？

转换速度主要取决于PDF的页数、复杂程度以及您的设备性能。包含大量图片、图表的PDF转换时间会较长，纯文字PDF转换速度则很快。
