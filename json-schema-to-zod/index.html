常见问题


什么是Zod？
Zod是TypeScript优先的运行时验证库，提供类型安全的模式定义和验证。它可以在运行时验证数据结构，同时自动推断TypeScript类型，是前端和Node.js项目的首选验证方案。


JSON Schema和Zod有什么区别？
JSON Schema是标准的JSON数据验证规范，使用JSON格式定义。Zod是TypeScript库，使用链式API定义验证规则。Zod的优势：1）更好的TypeScript类型推断；2）更简洁的API；3）更好的错误信息；4）支持transform和refine。


支持哪些JSON Schema特性？
支持大部分常用特性：string/number/integer/boolean/array/object类型、嵌套对象、数组元素验证、枚举(enum)、最小/最大值约束、正则模式(pattern)、必填字段(required)、可选字段(optional)、默认值(default)、description注释等。


转换后的代码可以直接使用吗？
可以。生成的Zod代码是完整的、可直接运行的TypeScript代码。只需确保项目中已安装zod包（npm install zod）。代码包含import语句，可以直接复制到项目中使用。


数据安全吗？
完全安全。所有转换都在浏览器本地完成，不会将您的JSON Schema发送到任何服务器。
