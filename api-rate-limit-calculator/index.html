❓ 常见问题

什么是API速率限制？

API速率限制是控制客户端在指定时间窗口内对API发起请求次数的机制，防止服务过载和滥用。

令牌桶和漏桶有什么区别？

令牌桶允许突发流量，漏桶以固定速率处理请求。令牌桶更适合允许短时突发的场景。

如何计算QPS？

QPS=总请求数/时间窗口（秒）。例如1000请求/60秒≈16.67 QPS。

数据会上传到服务器吗？

不会。所有计算在浏览器本地完成。

支持哪些限流算法？

支持令牌桶、漏桶、固定窗口计数器和滑动窗口日志四种主流限流算法。

如何确定合适的限流参数？

根据服务容量、用户规模和业务容忍度确定，一般建议限流值设为容量的70-80%。

API速率限制计算器能做什么？

API速率限制计算器是一款免费在线开发者工具，帮助设计和计算API限流策略参数。支持令牌桶、漏桶、固定窗口和滑动窗口四种主流限流算法，计算QPS、RPM、最大并发数、利用率等关键指标。适用于API网关配置、微服务限流设计和容量规划。所有计算在浏览器本地完成，数据绝不上传服务器。

应用场景

场景1：API网关配置

后端开发者在配置Kong/Nginx/API Gateway时，需要计算合适的限流参数防止服务过载。

场景2：微服务限流设计

架构师在设计微服务限流策略时，根据服务容量计算各算法的限流阈值。

场景3：容量规划

运维团队根据业务增长预测，计算需要调整的限流参数和资源配额。

扩展知识

API限流是分布式系统的核心保护机制。令牌桶算法以固定速率生成令牌，允许突发流量（桶中有令牌时快速消费），适合大多数API场景。漏桶算法以固定速率处理请求，平滑输出流量，适合需要严格控制处理速率的场景。固定窗口算法在固定时间窗口内计数，实现简单但存在窗口边界突发问题。滑动窗口算法将窗口细分为多个小窗口，解决边界突发问题，精度更高。Little's Law（利特尔法则）是并发计算的基础：并发数=QPS×平均延迟。
