📖 Terraform HCL详解

什么是HCL

HCL（HashiCorp Configuration Language）是HashiCorp公司开发的配置语言，专为Terraform等基础设施即代码（IaC）工具设计。它结合了JSON的机器可读性和人类可读性，支持块结构、属性赋值、表达式和注释，是云原生基础设施管理的核心语言。

HCL核心语法

  • 块（Block）：resource "aws_instance" "web" { ... }
  • 属性：ami = "ami-12345"
  • 变量插值：var.instance_type
  • 注释：#、//（行注释）和/* */（块注释）
  • 条件表达式：condition ? true_val : false_val
  • for表达式：[for s in var.list: upper(s)]

使用教程

  1. 粘贴Terraform HCL代码到输入框
  2. 选择缩进方式（2空格或4空格）
  3. 点击"格式化"美化代码，或"压缩"减小体积
  4. 复制结果或下载.tf文件

应用场景

  • 代码审查：格式化后更易发现配置错误
  • 团队协作：统一代码风格减少Git冲突
  • 学习研究：格式化压缩代码便于理解
  • CI/CD：自动化格式检查确保代码质量

📖 常见问题

什么是Terraform HCL?

HCL（HashiCorp Configuration Language）是HashiCorp公司开发的配置语言，用于Terraform等工具。它结合了JSON的机器可读性和人类可读性，支持块结构、属性赋值、表达式和注释。

HCL和JSON有什么区别?

HCL是JSON的超集：1)HCL支持注释（#和//），JSON不支持；2)HCL支持块结构（resource块等），JSON用对象嵌套；3)HCL支持表达式和变量插值，JSON只支持字面量；4)HCL更易读写，JSON更严格。

如何格式化Terraform代码?

本工具自动将HCL代码按照terraform fmt的标准格式化：统一缩进2空格、对齐等号、规范块结构、整理注释。也可使用命令行terraform fmt命令格式化。

Terraform代码格式化有什么好处?

格式化后的Terraform代码：1)更易阅读和维护；2)团队协作时减少风格冲突；3)符合terraform fmt标准；4)便于代码审查发现配置错误；5)提高基础设施即代码的专业性。

代码会上传到服务器吗?

不会。所有HCL格式化和验证都在浏览器本地完成，您的基础设施配置不会发送到任何服务器，保障安全。
