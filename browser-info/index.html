使用教程

本浏览器信息检测工具操作简单且功能完善，打开页面即可自动获取所有信息，以下是详细的使用指南：

自动检测

打开本工具页面后，所有浏览器和设备信息会自动检测并显示，无需任何手动操作。检测过程完全在您的浏览器本地完成，navigator和window对象的属性被读取后直接展示在页面上，数据不会上传到任何服务器。

复制User Agent

页面顶部的「User Agent 完整字符串」区域显示了浏览器发送给服务器的原始UA字符串。点击「复制UA」按钮，即可将完整的UA字符串复制到剪贴板，方便您在开发调试、填写技术支持表单或配置CDN规则时使用。

重新检测

如果您调整了浏览器设置（如缩放比例、语言偏好）或切换了设备方向，点击「重新检测」按钮可以刷新所有检测项，获取最新的浏览器环境参数。

逐项解读

检测结果分为五个类别：浏览器与操作系统（名称、版本、渲染引擎）、屏幕与显示（分辨率、颜色深度、设备像素比）、硬件信息（CPU核心数、设备内存、触摸支持）、网络与时区（在线状态、时区、语言偏好）、浏览器功能与API（WebGL、Service Worker、推送通知等）。每个类别下的具体项目都配有清晰的标签和实时数值。

应用场景

浏览器信息检测在Web开发和技术支持中应用广泛，以下是几个典型的使用场景：

前端开发调试

开发响应式网站时，需要了解目标用户的屏幕分辨率、设备像素比（DPR）和视口尺寸，以验证媒体查询断点是否正确生效。通过本工具快速获取这些参数，可以判断布局问题是否由特定设备环境引起。此外，navigator.userAgentData 和 Client Hints 的相关调试也可以借助本工具完成。

浏览器兼容性排查

当用户报告网站功能异常时，技术支持人员通常需要了解用户的浏览器版本、操作系统和启用的API支持情况。让用户将本工具的检测结果截图或复制UA发送给您，可以快速定位是否为旧版浏览器缺少某项Web API支持（如WebGL 2.0、Clipboard API、Web Share API）导致的问题。

CDN与代理配置验证

使用Cloudflare、AWS CloudFront等CDN服务时，常需要根据User Agent或Accept-Language头部进行内容分发规则配置。本工具显示的UA字符串和语言偏好可以帮助您验证CDN的边缘规则是否正确识别了访客特征，确保不同地区的用户获得正确的本地化内容。

隐私与安全审计

浏览器通过JavaScript暴露了大量设备信息，这些信息可能被网站用于指纹识别（Browser Fingerprinting）。通过本工具查看您的浏览器暴露了多少信息，您可以评估自己的数字隐私暴露程度，并考虑安装隐私保护扩展（如uBlock Origin、Privacy Badger）或调整浏览器隐私设置来减少可识别特征。

扩展知识

User Agent 的演进

User Agent字符串最初由Netscape浏览器引入，格式为Mozilla/[版本] ([系统信息]) [平台信息]。由于历史原因，现代浏览器的UA字符串中普遍包含「Mozilla」和「KHTML, like Gecko」等字样，这并非因为它们基于Mozilla或Gecko引擎，而是为了保证与早期网站的兼容性。Google正在推动User Agent Client Hints (UA-CH)替代方案，将详细的设备信息从UA字符串中移除，改为通过HTTP请求头按需获取。

设备像素比 (DPR) 与响应式图片

window.devicePixelRatio表示物理像素与CSS像素的比值。在Retina/HiDPI屏幕上，这个值通常为2或3。响应式图片使用srcset属性时，浏览器会根据DPR选择合适的图片分辨率，确保在高清屏幕上显示清晰的图像而不浪费带宽。例如：<img srcset="img-1x.jpg 1x, img-2x.jpg 2x">。

Client Hints 与隐私保护

传统的User Agent字符串包含了过多的设备信息，容易被用于被动指纹识别。Chrome从版本89开始支持User-Agent Client Hints，允许网站通过HTTP头部（如Sec-CH-UA、Sec-CH-UA-Platform、Sec-CH-UA-Mobile）获取精简的浏览器信息，而非解析冗长的UA字符串。Firefox和Safari也在逐步跟进这一标准，未来UA字符串将变得越来越简化。

常见问题 (FAQ)

浏览器信息检测工具会泄露我的隐私吗？

不会。本工具完全在浏览器本地运行，所有信息通过JavaScript的navigator和window对象获取，数据不会上传到任何服务器。您看到的所有信息都是浏览器主动暴露给网站的公开信息，任何网站都可以获取相同的数据。

User Agent是什么？为什么网站需要它？

User Agent是浏览器向服务器发送的标识字符串，包含浏览器名称、版本、操作系统等信息。网站通过它来判断访客使用的设备和浏览器，以便提供适配的页面内容或功能。例如，移动端网站通常会检测UA中的「Mobile」字样来重定向到移动版页面。

为什么检测到的屏幕分辨率和我的显示器分辨率不一致？

浏览器报告的是当前视口的CSS像素尺寸，而非物理像素。如果系统开启了缩放（如125%或150%），或者浏览器缩放比例不是100%，检测到的分辨率会小于显示器的物理分辨率。此外，浏览器窗口非最大化时也会显示窗口尺寸而非屏幕尺寸。

为什么有些硬件信息（如CPU型号）检测不到？

出于安全和隐私保护，浏览器不会暴露具体的CPU型号、GPU型号等敏感硬件信息。本工具只能获取浏览器API公开的信息，如逻辑CPU核心数（navigator.hardwareConcurrency）和设备内存大小（navigator.deviceMemory），这些值均为近似值而非精确型号。

开发者可以用这个工具做什么？

前端开发者可用此工具快速了解目标用户的浏览器环境，调试响应式布局问题，验证User Agent字符串，检测Web API兼容性，排查浏览器特有的Bug，以及验证CDN或代理是否正确修改了请求头。

检测结果在不同浏览器中为什么有差异？

不同浏览器实现的Web API标准和暴露的信息范围各不相同。例如，Safari不暴露navigator.deviceMemory，Firefox不暴露navigator.connection的有效类型。这些差异正是开发者进行跨浏览器兼容性测试时需要关注的重点。

本工具支持检测移动浏览器吗？

支持。本工具使用标准的Web API，在任何支持JavaScript的浏览器中都可以正常工作，包括Chrome、Safari、Firefox、Edge的桌面版和移动版。移动浏览器检测到的信息会包含设备方向、触摸支持、移动网络类型等移动端特有的参数。

为什么有时网络状态显示不准确？

navigator.onLine只能检测浏览器是否连接到网络，无法判断网络是否真正可用（例如，连接到无法访问互联网的WiFi时仍可能显示在线）。更准确的网络状态检测需要结合fetch请求超时或navigator.connection.effectiveType等高级API。
