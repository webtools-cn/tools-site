📖 CSS @keyframes Animation Guide

Basic Syntax

@keyframes name { from { styles } to { styles } } or use percentages 0% {} 50% {} 100% {}

Animation Shorthand

animation: name duration timing-function delay iteration-count direction fill-mode play-state

Performance Tips

  • Prefer transform and opacity for GPU-accelerated animations
  • Avoid animating width, height, margin which trigger layout reflow
  • Use will-change to hint the browser for pre-optimization
  • Consider contain property to isolate reflow scope for complex animations

📖 Frequently Asked Questions

What is CSS @keyframes animation?

@keyframes is a CSS3 rule for defining animation keyframes. By specifying style states at different points in time, the browser automatically creates smooth transitions between keyframes. You can control duration, delay, iteration count, direction, and timing function.

What is the difference between CSS animation and transition?

CSS transitions only switch between two states and require a trigger (like hover). CSS animations can define multiple keyframes, auto-play, and loop, making them more flexible. Animations are ideal for complex multi-step effects, transitions for simple state changes.

How to choose the right timing function?

ease (default) suits most cases, ease-in for fade-out effects, ease-out for fade-in effects, ease-in-out for symmetric animations, linear for constant speed (like rotation), cubic-bezier for custom precise curves.

Do CSS animations affect page performance?

Animations using transform and opacity are GPU-accelerated and perform best. Avoid animating width, height, margin and other properties that trigger layout reflow. Use will-change to hint the browser for pre-optimization.

Is my data uploaded to any server?

No. All animation generation and preview is done locally in your browser. No data is sent to any server.
