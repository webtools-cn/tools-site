HTML Entity Encoding

What is HTML entity encoding?

HTML entity encoding converts special characters in HTML into their corresponding entity references. For example, &lt; represents < and &amp; represents &. This allows these characters to be safely displayed on web pages without being misinterpreted by the browser.

Common HTML Entities

• &amp; → & (ampersand)

• &lt; → < (less than)

• &gt; → > (greater than)

• &quot; → " (double quote)

• &#39; → ' (single quote)

• &nbsp; → non-breaking space

• &copy; → © (copyright)

Named vs Numeric Entities

Named entities use names (e.g., &lt;), while numeric entities use Unicode code points (e.g., &#60; or hex &#x3C;). Numeric entities can represent any Unicode character; named entities only cover predefined characters.

What is HTML entity encoding?

HTML entity encoding converts special characters in HTML (such as <, >, &, ", ') into their corresponding entity references (such as &lt;, &gt;, &amp;, &quot;, &#39;), preventing these characters from being parsed as HTML tags or attributes by the browser.

What's the difference between HTML entity encoding and URL encoding?

HTML entity encoding is used to safely display special characters in HTML documents, using entity references starting with &. URL encoding is used to transmit special characters in URLs, using percent-encoding starting with %. They serve different purposes and should not be mixed.

Is my data safe? Does it get uploaded to a server?

Completely safe. This tool is a pure frontend application. All encoding and decoding operations are performed locally in your browser. Your data is never uploaded to any server.

URL Encoding

What is URL encoding?

URL encoding (also called percent-encoding) converts special characters in URLs into %XX hexadecimal format. For example, a space becomes %20, and the Chinese character "中" becomes %E4%B8%AD.

encodeURI vs encodeURIComponent

• encodeURI: Used for encoding complete URLs, preserves reserved characters like ;/?:@&=+$,#

• encodeURIComponent: Used for encoding URL parameter values, encodes all special characters (except letters, digits, and -_.~!*')

Common Use Cases

• URL Parameters: Pass special characters in query strings

• API Requests: Encode paths and parameters in REST APIs

• International URLs: Encode non-ASCII characters to percent format

• Form Submission: application/x-www-form-urlencoded format

Is space %20 or + in URL encoding?

Both are valid. %20 is the standard URL encoding, while + is used in the application/x-www-form-urlencoded format (primarily for form submissions). This tool uses the standard %20 format.

Does it support Chinese/Unicode?

Yes. The tool uses the browser's native encodeURI / encodeURIComponent APIs, with full support for Chinese, Emoji, and all other Unicode characters.

Unicode Conversion

What is Unicode escaping?

Unicode escaping represents characters as \uXXXX escape sequences, where XXXX is the hexadecimal representation of the character's Unicode code point. For example, the Chinese character "中" can be represented as \u4e2d.

Common Unicode Notations

• \uXXXX: Escape sequence in JavaScript/JSON

• U+XXXX: Unicode standard code point notation

• &#XXXX;: HTML numeric entity

• %XX: URL percent-encoding (UTF-8 bytes)

BMP and Supplementary Planes

Characters within the Basic Multilingual Plane (BMP, U+0000 to U+FFFF) can be represented with a single \uXXXX. Characters beyond the BMP (like some Emoji) require surrogate pairs (two \uXXXX values) or the ES6 \u{XXXXX} syntax.

What does the Unicode escape \uXXXX mean?

\uXXXX is a Unicode escape sequence where XXXX is the hexadecimal representation of the character's Unicode code point. For example, \u4e2d represents the Chinese character "中". This notation is commonly used in JavaScript strings, JSON, and other contexts.

How are Emoji converted?

Most Emoji are in the Unicode supplementary plane (above U+10000) and require surrogate pairs (two \uXXXX values). This tool supports encoding and decoding of surrogate pairs, as well as the ES6 \u{XXXXX} syntax.
