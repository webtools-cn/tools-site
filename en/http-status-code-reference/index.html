What is the HTTP Status Code Reference?

The HTTP Status Code Reference is a free online developer lookup tool covering all status codes defined in the HTTP/1.1 and HTTP/2 specifications. Search by number, name, or keyword; filter by category; and click any code for detailed explanations including common causes and troubleshooting tips. All data is embedded in the page — no network requests needed, works offline.

Core Features

  • Complete Coverage: All 60+ status codes across 1xx Informational, 2xx Success, 3xx Redirection, 4xx Client Error, and 5xx Server Error categories
  • Real-time Search: Filter by status code number, English name, or description keywords instantly
  • Category Filter: One-click category tabs to quickly narrow down to specific code types
  • Expandable Details: Click any card to see detailed explanation, common causes, and solutions
  • Statistics Overview: Real-time display of filtered results count and per-category counts
  • Offline Ready: All data embedded in the page — works without internet

How to Use

1. Search: Type a status code number (e.g., 404) or keyword (e.g., "unauthorized", "redirect") in the search box. Results filter in real-time.

2. Filter by Category: Click the category tabs (1xx/2xx/3xx/4xx/5xx) to show only codes in that category. Click "All" to reset.

3. View Details: Click any status code card to expand it and see the full explanation, including what the code means, common causes, and troubleshooting advice.

4. Combined Filtering: Select a category first, then search within it for more precise results.

5. Direct Navigation: Append a hash with the code number (e.g., #404) to the URL to jump directly to that status code.

Use Cases

Use Case 1: Developer Debugging

When building web applications and encountering unexpected status codes, quickly look up their meaning and common causes. For example, receiving a 429 status code tells you the client is being rate-limited — you need to add request throttling or implement exponential backoff.

Use Case 2: DevOps Troubleshooting

When investigating production issues, Nginx/Apache logs may show 502/503/504 codes. This tool helps you quickly understand the differences: 502 means upstream returned an invalid response, 503 means the service is temporarily unavailable, and 504 means the upstream timed out.

Use Case 3: API Design

When designing RESTful APIs, choosing the right status code is crucial. Use 201 for resource creation, 204 for success with no content, 409 for conflicts, and 422 for semantically invalid requests despite correct syntax.

Use Case 4: SEO Optimization

Proper HTTP status codes are essential for SEO. Use 301 for permanent redirects to transfer link equity, 302 for temporary redirects to keep the original URL indexed, 404 for pages that no longer exist, and 451 for content blocked for legal reasons.

Extended Knowledge

HTTP Status Code Structure: Status codes are 3-digit numbers where the first digit defines the response category. 1xx = informational, 2xx = success, 3xx = redirection, 4xx = client error, 5xx = server error. RFC 7231 defines the core codes, with additional codes added through subsequent RFCs and the IANA registry.

Non-standard Status Codes: Nginx defines custom codes like 444 (close connection with no response), 494 (request header too large), 495 (SSL certificate error), 496 (SSL certificate required), and 497 (HTTP sent to HTTPS port). Cloudflare defines 520-527 series codes. These non-standard codes may appear in specific scenarios.

Status Code Best Practices: For success responses, prefer 200 (general success), 201 (created), and 204 (no content). For client errors, prefer 400 (general error), 401 (unauthenticated), 403 (forbidden), 404 (not found), 409 (conflict), and 422 (semantic error). Avoid wrapping errors in 200 responses — use proper 4xx/5xx codes instead.

What are the HTTP status code categories?

HTTP status codes are divided into 5 categories: 1xx Informational (request received, continuing process), 2xx Success (request successfully processed), 3xx Redirection (further action needed), 4xx Client Error (request contains bad syntax or cannot be fulfilled), 5xx Server Error (server failed to fulfill valid request).

What are the most common HTTP status codes?

The most common status codes include: 200 OK, 301 Moved Permanently, 302 Found, 304 Not Modified, 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 500 Internal Server Error, 502 Bad Gateway, and 503 Service Unavailable.

What is the difference between 301 and 302 redirects?

301 is a permanent redirect — search engines transfer link equity to the new URL and remove the old URL from their index. 302 is a temporary redirect — search engines keep the old URL indexed and do not transfer link equity. Use 301 for permanent URL changes in SEO.

What is the difference between 401 and 403?

401 Unauthorized means the client must authenticate itself (log in). 403 Forbidden means the client is authenticated but does not have permission to access the resource. Simply put: 401 asks 'who are you?', 403 says 'you can't come in'.

What is the difference between 502 and 504?

502 Bad Gateway means the gateway received an invalid response from the upstream server. 504 Gateway Timeout means the gateway did not receive a timely response from the upstream server. 502 means upstream returned an error; 504 means upstream didn't respond at all.
