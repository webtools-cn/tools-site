What Can the HTTP Status Code Reference Do?

The HTTP Status Code Reference is a free online developer tool providing a complete reference table of HTTP status codes, covering all 63 standard codes across 1xx informational, 2xx success, 3xx redirection, 4xx client error, and 5xx server error categories. It supports category filtering, keyword search, and one-click copy — an essential reference for web development and API debugging. All queries run locally in your browser with no server uploads.

Core Features

  • Complete Status Code Table: All 63 standard HTTP status codes defined in RFC 7231/6585/7538/7725
  • Category Filter: Quick filter by 1xx-5xx five major categories
  • Real-time Search: Search by code number, English name, or description
  • One-Click Copy: Click copy button to quickly copy status code info
  • Batch Export: Copy all results or download as CSV file

How to Use

1. Type a status code number (e.g., 404), name (e.g., Not Found), or description in the search box for real-time filtering.

2. Click category buttons (1xx-5xx) to quickly view codes in a specific category.

3. Click the "Copy" button next to any status code to copy its information.

4. Use "Copy All Results" or "Download CSV" to batch export filtered results.

Extended Knowledge

HTTP status codes are defined by RFC 7231 (formerly RFC 2616) and divided into 5 classes: 1xx (informational — request received, continuing process), 2xx (success — request successfully processed), 3xx (redirection — further action needed), 4xx (client error — request contains bad syntax or cannot be fulfilled), 5xx (server error — server failed to fulfill valid request). Non-standard codes like Nginx's 497 or Cloudflare's 520-527 are not covered in this tool.

❓ FAQ

What are HTTP status codes?

HTTP status codes are 3-digit numbers returned by a server in response to a client's request, divided into 5 classes: 1xx informational, 2xx success, 3xx redirection, 4xx client error, 5xx server error.

What are the most common HTTP status codes?

The most common are 200 (OK), 301 (Moved Permanently), 404 (Not Found), 500 (Internal Server Error), and 403 (Forbidden).

What's the difference between 404 and 500?

404 means the requested resource doesn't exist (client error), while 500 means the server encountered an internal error (server error).

Is my data uploaded to a server?

No. All search and filtering happens locally in your browser.

Does it support search?

Yes. You can search by status code number, English name, or description in real-time.

Which status codes are most used in REST API development?

Common REST API codes: 200, 201, 204, 400, 401, 403, 404, 409, 422, 500.
