📖 Git Merge Conflict Guide

Understanding Conflict Markers

When Git encounters a merge conflict, it inserts special markers into the file:

  • <<<<<<< HEAD — Start of the current branch (yours)
  • ======= — Separator between the two versions
  • >>>>>>> branch-name — End of the incoming branch (theirs)

Resolution Strategies

  1. Keep Yours: Use the current branch version, discard incoming changes
  2. Keep Theirs: Use the incoming branch version, discard current changes
  3. Keep Both: Include both versions (may need manual editing)
  4. Manual Edit: Combine parts from both sides into a new version

Best Practices

  • Resolve conflicts as soon as possible to avoid compounding issues
  • Communicate with team members who made the conflicting changes
  • Test the resolved code before committing
  • Use small, focused commits to minimize conflicts
  • Pull/rebase frequently to stay up to date with the main branch

📖 Frequently Asked Questions

What are git merge conflicts?

Git merge conflicts occur when two branches modify the same lines of a file and Git cannot automatically determine which version to keep. Git marks the conflicting sections with special markers and requires manual resolution before the merge can be completed.

What do the conflict markers mean?

The HEAD marker starts the current branch's version. The equals marker separates the two versions. The branch-name marker ends the incoming branch's version. Everything between HEAD and equals is yours, and everything between equals and branch-name is theirs.

How do I resolve a merge conflict?

To resolve a conflict: 1) Open the conflicted file; 2) Find the conflict markers; 3) Choose which version to keep (yours, theirs, or a combination); 4) Remove the conflict markers; 5) Save the file; 6) Stage the file with git add; 7) Commit the merge. This tool helps you visualize and choose interactively.

Can I combine both versions of a conflict?

Yes. You can choose to keep both versions by selecting 'Both' in this tool, or manually edit the resolved output to combine elements from each side. This is common when both branches add different functions or imports to the same file section.

What happens after I resolve all conflicts?

After resolving all conflicts, copy the resolved output and replace the conflicted file content. Then run 'git add filename' to stage the resolved file, and 'git commit' to complete the merge. The conflict markers will be removed and the merge will be finalized.

Is my code sent to any server?

No. All conflict parsing and resolution is done locally in your browser. No code or file content is sent to any server. Your code remains completely private.
