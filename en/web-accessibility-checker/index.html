What Can the Web Accessibility Checker Do?

The Web Accessibility Checker is a free online tool that checks HTML code for WCAG 2.1 compliance. It automatically detects missing alt attributes on images, form inputs without labels, heading hierarchy jumps, ARIA label errors, insufficient color contrast, missing landmark regions, and 30+ accessibility rules. All checks run locally in your browser — HTML code is never uploaded to any server.

Core Features

  • 30+ Check Rules: Covers major WCAG 2.1 Level A/AA rules including image alt, form labels, heading hierarchy, ARIA, semantic HTML, and more
  • Graded Reports: Errors (must fix), Warnings (should fix), Info (best practices) — each annotated with WCAG rule numbers
  • Real-time Checking: Instant feedback as you input HTML
  • Multiple Input Methods: Paste code, drag-and-drop files, or import from URL
  • Report Export: Copy summary or download JSON/HTML reports

How to Use

1. Paste HTML code in the input box, or drag and drop an HTML file.

2. Select the WCAG check level (A or AA).

3. Click "Run Check" to see results.

4. Review the issue list by severity, each annotated with WCAG rule numbers.

5. Fix issues based on suggestions and export the report.

Technical Background

WCAG (Web Content Accessibility Guidelines) is the W3C standard for web accessibility, currently at version 2.2. WCAG defines three compliance levels: Level A (minimum), Level AA (recommended, basis for most laws), and Level AAA (highest). The four core principles are: Perceivable, Operable, Understandable, and Robust. Common issues include: missing alt text on images (1.1.1), form inputs without labels (3.3.2), insufficient color contrast (1.4.3), keyboard inaccessibility (2.1.1), and heading hierarchy jumps (1.3.1). Approximately 15% of the global population has some form of disability — accessibility is not just a moral obligation but a legal requirement in many regions.

What issues are checked?

Missing alt, ARIA errors, form labels, heading hierarchy, color contrast, missing landmarks, keyboard accessibility — 30+ WCAG rules.

Which WCAG level?

Covers WCAG 2.1 Level A and AA, the standard for most legal compliance.

Is data uploaded?

No. Pure frontend — all checks run in your browser. HTML never leaves your device.

How to input HTML?

Paste code, drag-and-drop HTML files, or import from URL.

Are results 100% accurate?

Automated checks find most common issues but can't replace manual testing. Dynamic content needs manual verification.

Yes. Copy summary, download JSON or HTML reports.
