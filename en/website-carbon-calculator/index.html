📖 Website Carbon Basics

The internet consumes about 4% of global electricity and produces about 3.7% of greenhouse gas emissions — comparable to the aviation industry. Every web page visit consumes energy: data centers process requests, networks transmit data, user devices render pages.

Calculation Method

Based on the Shift Project: transferring 1GB of data produces approximately 0.81g CO2 (mixed energy), 0.18g (green energy), or 1.34g (fossil energy). A mature tree absorbs about 21kg CO2 per year.

Emission Standards

  • 🟢 <0.095g/visit — Very Clean (A+)
  • 🟢 <0.186g/visit — Clean (B+)
  • 🟡 <0.347g/visit — Average (C)
  • 🟠 <0.493g/visit — Dirty (D)
  • 🔴 ≥0.493g/visit — Very Dirty (F)

❓ FAQ

How is website carbon calculated?

Based on the Shift Project method: transferring 1GB of data produces approximately 0.81g CO2. Formula: Carbon = Page Size (GB) × 0.81g × Data Center Factor × Visits.

What is a green website?

A green website produces less than 0.2g CO2 per page visit. Standards: <0.095g is very clean, 0.095-0.186g is clean, 0.186-0.347g is average, >0.347g is dirty.

How to reduce website carbon emissions?

1) Reduce page size: compress images, remove unused JS/CSS; 2) Reduce HTTP requests; 3) Use green hosting; 4) Implement caching; 5) Lazy loading.

Is the data accurate?

This tool provides estimates based on industry averages. Actual emissions depend on many factors. Estimates are useful for comparing relative carbon levels.

Is my data secure?

Completely secure. All calculations are done locally in your browser. No data is sent to any server.
