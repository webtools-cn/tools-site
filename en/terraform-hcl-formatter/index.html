Technical Background

HCL(HashiCorp Configuration Language)YesHashiCorp设计的配置语言，语法特点：块结构用{}包裹，标签用字符串引号，赋值用=，支持for表达式、条件运算符和模板。Terraform官方Format工具terraform fmt使用2 Spaces缩进，不Sort Blocks。HCL2引入了类型系统和函数，支持dynamic块和variable验证。建议团队统一使用terraform fmt或本工具保持代码风格一致。

❓ 常见问题

What is HCL?

HCL (HashiCorp Configuration Language) is a config language by HashiCorp, used in Terraform to define infrastructure. Syntax is between JSON and YAML, supporting comments, variables, and expressions.

Will formatting change HCL semantics?

不会。Format只调整空白、缩进和排列顺序，不改变任何逻辑或值。

Does it support Terraform 1.x syntax?

支持Terraform 1.x的大部分HCL语法，包括resource、data、module、variable、output、locals等块类型。

Is my data safe?

Absolutely safe. All formatting happens locally in your browser. No code is uploaded.

How does it differ from terraform fmt?

terraform fmt requires installing Terraform CLI. This tool runs in the browser without installation. Similar functionality with minor differences.

Does it support file drag & drop?

Yes. Drag and drop .tf files onto the input area to auto-read content.

How to handle large Terraform projects?

建议逐文件Format。对于模块化的项目，按模块分别处理效果最佳。
