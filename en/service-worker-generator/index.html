What Can the Service Worker Generator Do?

The Service Worker Generator is a free online PWA development tool，帮助开发者通过可视化界面快速配置Service Worker的缓存策略、离线回退、Push Notifications等功能，一键生成production-ready的代码。支持Workbox和纯原生两种Implementation，无需手动编写复杂的SW逻辑，大幅降低PWA开发门槛。

核心功能

  • 可视化缓存策略：5种缓存策略（Cache First/Network First/Stale-While-Revalidate/Cache Only/Network Only），按URL模式灵活配置
  • 预缓存管理：安装时预缓存关键资源URL列表，确保首次加载后即可离线访问
  • 离线回退：网络不可用时自动显示离线页面，提供友好的用户体验
  • Push Notifications：自动生成push事件监听和notificationclick处理代码
  • 双模式输出：Workbox CDN模式（功能丰富）和原生JS模式（零依赖），按需选择

使用教程

1. 在基本配置区设置Cache Name、选择Implementation（Workbox或原生JS）。

2. 在缓存策略区添加URL匹配规则，为不同类型资源选择合适的缓存策略（如图片用Cache First，API用Network First）。

3. 在功能选项区开启需要的功能：预缓存、离线回退、Push Notifications、Background Sync等。

4. 配置过期参数（Max Cache Entries数和天数），防止缓存无限增长。

5. 点击"生成代码"获取Service Worker代码，复制或下载sw.js文件。点击"生成注册代码"获取页面注册SW的代码片段。使用Ctrl+Enter快速生成，Ctrl+Shift+C复制结果。

应用场景

场景1：PWA离线应用

前端开发者将Web应用转为PWA时，使用本工具快速生成Service Worker代码，实现离线访问、缓存管理和安装提示，无需深入学习SW API。

场景2：性能优化

网站运维人员通过配置缓存策略优化资源加载速度，静态资源Cache First减少网络请求，API数据Network First保证实时性，Stale-While-Revalidate策略兼顾速度和新鲜度。

场景3：消息推送

后端开发者需要实现WebPush Notifications时，开启Push Notifications选项即可获得完整的push事件处理代码，支持自定义通知标题、内容和图标。

场景4：移动端优化

移动端H5应用在网络不稳定时需要提供离线体验，通过Service Worker的离线回退和缓存策略，确保弱网和断网场景下仍可正常使用。

扩展知识

Service Worker是PWA的核心技术，运行在浏览器后台的独立线程中，生命周期包括install、activate、fetch三个核心事件。Workbox是Google推出的SW工具库，封装了常用缓存策略和最佳实践，大幅简化SW开发。缓存策略选择原则：不常变化的资源(字体/图片)用Cache First，频繁更新的内容(API)用Network First，可容忍短暂过期的数据用Stale While Revalidate。Service Worker必须在HTTPS环境下运行（localhost例外），且注册后需刷新页面才能生效。

What is a Service Worker?

A Service Worker is a script that runs in the browser background, independent of web pages. It can intercept network requests, manage caches, and handle push notifications. It is the core technology for building PWAs (Progressive Web Apps).

Can the generated code be used directly?

Yes. The generated code follows Workbox best practices, including install, activate, and fetch event handlers with multiple caching strategies. It can be deployed to production directly.

What caching strategies are supported?

5 strategies are supported: Cache First, Network First, Stale While Revalidate, Cache Only, and Network Only.

How do I add an offline fallback page?

在配置中开启离线页面选项，工具自动在fetch事件中添加离线回退逻辑，网络不可用时显示指定离线页面。

Does it support push notifications?

支持。开启Push Notifications选项后，生成的代码包含push事件监听和notificationclick事件处理。

Is my data safe?

Completely safe. All configuration and code generation is done locally in your browser. No data is uploaded to any server.

Do I need to install Workbox?

不需要。选择Workbox CDN模式自动从CDN加载，选择原生模式则生成零依赖纯JS代码。
