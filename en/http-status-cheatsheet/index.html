What Can the HTTP Status Code Cheatsheet Do?

The HTTP Status Code Cheatsheet is a free online developer tool providing a complete HTTP status code reference manual. Covers all standard status codes from 1xx Informational to 5xx Server Error, with real-time search by number or keyword, category filtering, and detailed descriptions on click. All data is built into the browser — no network requests needed, works offline.

Core Features

  • Complete Coverage: All IANA-registered HTTP status codes from 100 Continue to 511 Network Authentication Required, 60+ codes total
  • Real-time Search: Instant filtering by status code number or English keywords with fuzzy matching
  • Category Filtering: Filter by 1xx/2xx/3xx/4xx/5xx with color-coded visual distinction
  • Detail View: Click any status code for full description including definition, usage, common causes, and solutions
  • Export: One-click copy or download filtered results for documentation and team sharing

How to Use

1. Type a status code number (e.g., 404) or keyword (e.g., "Not Found", "Forbidden") in the search box.

2. Click category tags (1xx-5xx) to filter by specific class. Click again to remove the filter.

3. Click any status code card to view detailed description including definition, usage, and common causes.

4. Use the export function to copy or download current filtered results. Ctrl+Shift+C to quickly copy.

Use Cases

Case 1: API Debugging

Backend developers quickly look up returned status codes when debugging REST APIs, determining whether issues are client-side or server-side.

Case 2: Operations Troubleshooting

Ops teams investigate abnormal status codes in Nginx/Apache logs, understanding the difference between 502/503/504 to quickly locate gateway, upstream, or load balancer issues.

Case 3: Frontend Development

Frontend developers handle fetch/axios request errors by choosing appropriate error handling strategies based on status codes: 401 redirect to login, 403 show permission denied, 404 show empty state.

Case 4: SEO Optimization

SEO engineers check website HTTP status codes to ensure 301/302 redirects are used correctly, avoiding soft 404 issues that affect search engine indexing.

Extended Knowledge

HTTP status codes are managed by IANA and defined in RFC 9110 and related specifications. 1xx codes were introduced in HTTP/1.1 for protocol-level information; 2xx indicates successful request processing; 3xx is for redirection (301/308 are permanent, 302/307 are temporary); 4xx indicates client errors with 404 Not Found being the most common; 5xx indicates server internal errors. Non-standard codes (like Nginx's 444 No Response, Cloudflare's 520-527) are not in RFC but are widely used.

What are the HTTP status code categories?

5 categories: 1xx Informational, 2xx Success, 3xx Redirection, 4xx Client Error, 5xx Server Error.

What are the most common HTTP status codes?

Most common: 200 OK, 301 Moved Permanently, 302 Found, 304 Not Modified, 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable.

What's the difference between 401 and 403?

401 means no valid authentication credentials provided; 403 means the server understands but refuses to authorize, even with valid credentials.

What's the difference between 301 and 302?

301 is permanent redirect — search engines transfer link equity. 302 is temporary — search engines keep original URL indexed. Use 301 for SEO.
