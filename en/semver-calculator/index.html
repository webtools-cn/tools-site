What Can the SemVer Calculator Do?

The SemVer Calculator is a free online developer tool based on the SemVer 2.0.0 specification, providing version comparison, version bumping, and range constraint checking. It supports prerelease version comparison (alpha/beta/rc), MAJOR/MINOR/PATCH bumping, and ^/~/>=/

Core Features

  • Version Compare: Compare two version numbers with prerelease support
  • Version Bump: MAJOR/MINOR/PATCH bumping with prerelease support
  • Range Check: Check if a version satisfies ^/~/>=/
  • Constraint Parsing: Parse npm-style version range constraint expressions
  • Real-time Calculation: Results update as you type

How to Use

1. Compare: Enter two version numbers to see their relationship.

2. Bump: Enter a version and click a bump button to calculate the new version.

3. Range Check: Enter a version and constraint to check if it satisfies the range.

Use Cases

Case 1: npm Package Management

Check if dependency versions satisfy the version range constraints in package.json.

Case 2: CI/CD Releases

Automatically bump version numbers in release pipelines based on commit types.

Case 3: Version Planning

Plan version roadmaps and determine compatibility relationships between versions.

Extended Knowledge

SemVer 2.0.0 defines version format as MAJOR.MINOR.PATCH: MAJOR changes indicate incompatible API changes, MINOR changes add backward-compatible features, and PATCH changes fix bugs. Prerelease versions append a hyphen and identifier after PATCH (e.g., 1.0.0-alpha.1), and build metadata appends a plus sign (e.g., 1.0.0+build.123). npm's ^ means compatible version (allows MINOR and PATCH changes), ~ means approximate version (only allows PATCH changes).

❓ FAQ

What is Semantic Versioning?

MAJOR.MINOR.PATCH format where MAJOR = incompatible changes, MINOR = backward-compatible features, PATCH = bug fixes.

How do I compare versions?

Compare MAJOR→MINOR→PATCH numerically. Prerelease versions are lower than releases.

What is a version range?

A set of versions defined by comparison operators like >=, <, ~, ^.

Is my data uploaded?

No. All calculations happen locally in your browser.

~ vs ^ difference?

~1.2.3 allows PATCH changes (<1.3.0), ^1.2.3 allows MINOR+PATCH changes (<2.0.0).

Prerelease support?

Yes. Parse and compare versions with prerelease identifiers like 1.0.0-alpha.
