Learn More

-webkit-line-clamp is the core CSS property for multi-line text truncation, requiring display:-webkit-box, -webkit-box-orient:vertical, and overflow:hidden. Despite the -webkit prefix, it is supported by all major browsers (Chrome 6+, Firefox 68+, Safari 5+, Edge 79+) with over 97% global coverage. The line-clamp property in CSS Overflow Level 4 is the standardized version, currently under development. For precise truncation control, JavaScript height calculation can be used as an alternative.

FAQ

How to implement multi-line text truncation with CSS line-clamp?

Use the -webkit-line-clamp property with display:-webkit-box and -webkit-box-orient:vertical, plus overflow:hidden to truncate multi-line text with an ellipsis. This is the simplest pure CSS solution available.

What is the browser compatibility of line-clamp?

-webkit-line-clamp is supported by all major browsers including Chrome, Firefox, Safari, and Edge. Despite the -webkit prefix, Firefox has supported it since version 68. Global coverage exceeds 97%.

How to add expand/collapse functionality?

Pure CSS approach uses the checkbox hack with a hidden checkbox to toggle the expanded state. JavaScript approach is more flexible — remove the line-clamp property on click to expand, and restore it to collapse. This tool supports both approaches.

What is the difference between single-line and multi-line ellipsis?

Single-line ellipsis uses text-overflow:ellipsis with white-space:nowrap and overflow:hidden. Multi-line uses -webkit-line-clamp:N to specify the number of lines. Single-line has better compatibility, multi-line is more flexible.
