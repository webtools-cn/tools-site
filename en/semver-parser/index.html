What Can the SemVer Parser Do?

The SemVer Parser is a free online developer tool for parsing, comparing, and validating semantic version numbers. Enter any version number to instantly parse its MAJOR, MINOR, PATCH, prerelease tag, and build metadata components. Supports version comparison, range queries, version bumping, and compatibility checks. All operations run locally in your browser with no dependencies to install.

Core Features

  • Version Parsing: Real-time parsing of version number components (MAJOR/MINOR/PATCH/prerelease/build metadata) with instant response on input
  • Version Comparison: Compare two version numbers according to SemVer specification, automatically determining compatibility (compatible/new feature/breaking change)
  • Version Bumping: One-click calculation of MAJOR/MINOR/PATCH incremented versions, with automatic prerelease tag clearing
  • Range Queries: Support ^, ~, >=, < and other range expressions to check if a version satisfies specified constraints
  • Batch Validation: Validate multiple version numbers at once, marking each line as valid or invalid

How to Use

1. Enter a version number in the input field, such as 1.2.3 or 2.0.0-beta.1+build.42.

2. The tool automatically parses and displays all components. The version bump section shows three increment options.

3. Enter two version numbers in the compare section to see the comparison result and compatibility assessment.

4. Enter a range expression and test version in the range query section to check if it satisfies the constraint.

5. The batch validate section can check multiple version numbers at once. Use Ctrl+Enter to quickly parse, Ctrl+Shift+C to copy results.

Use Cases

Case 1: Dependency Management

When configuring dependency version ranges in package.json, use this tool to verify range expressions are correct and determine if specific versions satisfy dependency requirements.

Case 2: Release Planning

Project maintainers use the version bump feature to quickly calculate the next version number after MAJOR/MINOR/PATCH increments, ensuring compliance with SemVer specification.

Case 3: CI/CD Pipelines

Operations teams validate version number formats in automated pipelines, ensuring build artifact versions conform to SemVer specification and preventing release process errors.

Case 4: API Compatibility

Backend developers use compatibility checks when publishing new API versions to determine if version changes require client upgrade notifications. MAJOR increments mean breaking changes.

Extended Knowledge

The Semantic Versioning specification (SemVer 2.0.0) was created by Tom Preston-Werner and has become the industry standard for software version management. Core rules: MAJOR increment indicates incompatible API changes, MINOR increment indicates backward-compatible new features, PATCH increment indicates backward-compatible bug fixes. Prerelease versions (alpha/beta/rc) have lower priority than release versions, and build metadata does not affect version priority. npm's ^ and ~ are common range shorthand: ^1.2.3 means >=1.2.3 <2.0.0, ~1.2.3 means >=1.2.3 <1.3.0.

What is Semantic Versioning (SemVer)?

Semantic Versioning is a version numbering convention with the format MAJOR.MINOR.PATCH. MAJOR indicates incompatible API changes, MINOR indicates backward-compatible new features, PATCH indicates backward-compatible bug fixes.

How do I compare two version numbers?

Enter two version numbers and the tool automatically compares them by MAJOR→MINOR→PATCH→prerelease tag priority.

What is a prerelease version?

A prerelease version adds a hyphen and identifier after PATCH, e.g., 1.0.0-alpha. Prerelease versions have lower priority than release versions.

Are version range queries supported?

Yes. You can enter range expressions like ^1.2.3, ~1.2.3, >=1.0.0 <2.0.0, and the tool checks if a specified version satisfies the range.

Is my data safe?

Completely safe. All version parsing and comparison runs locally in your browser. No data is uploaded to any server.

How does version bumping work?

Enter a base version number, select the bump type (MAJOR/MINOR/PATCH), and the tool automatically calculates the incremented version number.

What is compatibility checking?

Based on SemVer rules: same MAJOR means API compatible, MINOR increment means new features but compatible, MAJOR increment means breaking changes.
