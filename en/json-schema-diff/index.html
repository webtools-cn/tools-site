What Can the JSON Schema Diff Tool Do?

The JSON Schema Diff Tool is a free online tool that deeply compares two JSON Schemas for differences. It automatically detects added properties, removed properties, type changes, constraint modifications, required field changes, and more, displaying them visually. Supports recursive comparison of nested objects and arrays with dot-notation paths. All comparison runs locally in your browser — no data is ever uploaded to a server.

Core Features

  • Deep Recursive Comparison: Recursively compares nested properties and items, showing deep differences
  • Multi-dimensional Diff Detection: Type changes, format changes, constraint changes (min/max/length), required changes, enum changes, description changes
  • Visual Diff Display: Green=added, Red=removed, Yellow=modified — intuitive change visualization
  • Diff Filtering: Filter by change type (added/removed/modified)
  • Multi-format Export: Copy summary, download JSON diff report, download Markdown changelog

How to Use

1. Paste the old JSON Schema in the left input box.

2. Paste the new JSON Schema in the right input box.

3. Select comparison options (deep recursive, show unchanged, filter type).

4. Click "Compare Diff" to see visual diff results.

5. Export the diff report or changelog.

Use Cases

Case 1: API Version Upgrades

Compare API v1 and v2 JSON Schemas to quickly understand interface changes and update client code and documentation.

Case 2: Database Schema Migration

Compare pre- and post-migration data model schemas to confirm all changes are as expected.

Case 3: Code Review

Compare Schema changes in Pull Requests to review whether new fields and type modifications are reasonable.

Case 4: Documentation Maintenance

Auto-generate changelogs recording specific Schema changes per version, keeping API docs in sync with code.

Technical Background

JSON Schema is the standard specification for describing JSON data structures, with the latest version being Draft 2020-12. In API development, Schema change management is critical — adding fields is usually backward compatible, but removing fields or changing types can break clients. OpenAPI/Swagger uses JSON Schema to define request and response formats. Good Schema versioning practices include: declaring the version with $schema, not making new fields required, deprecating before removing, and transitioning types via oneOf. This tool helps developers quickly identify the impact scope of Schema changes.

❓ FAQ

What diffs can be detected?

Added/removed properties, type changes, constraint changes (min/max/length/pattern), required changes, enum changes, description changes, nested object recursive diffs.

Is data uploaded?

No. Pure frontend — all comparison runs in your browser. Data never leaves your device.

Which Schema drafts are supported?

Draft-04 through Draft 2020-12. Comparison is based on property structure, not version-specific syntax.

How are nested objects handled?

Recursively compares nested properties and items with dot-notation paths (e.g., user.address.city).

Can I export diffs?

Yes. Copy summary, download JSON diff report, or download Markdown changelog.

Is drag-and-drop supported?

Yes. Drag .json files onto the input boxes to auto-read content.
