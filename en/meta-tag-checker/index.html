📖 Meta Tags Guide

Core Meta Tags

  • <title> — Page title shown in search results, recommended under 60 characters
  • meta description — Page description shown as snippet, recommended 120-160 characters
  • meta keywords — Keywords (most search engines now ignore this)
  • canonical — Canonical URL to avoid duplicate content issues

Open Graph Tags

  • og:title — Social sharing title
  • og:description — Social sharing description
  • og:image — Sharing thumbnail
  • og:url — Canonical URL
  • og:type — Page type (website/article, etc.)

Other Important Tags

  • robots — Search engine crawler directives
  • viewport — Mobile responsiveness
  • charset — Character encoding
  • hreflang — Multi-language page markers
  • JSON-LD — Structured data (Schema)

📖 FAQ

What are Meta tags?

Meta tags are HTML head elements that provide metadata about a webpage to search engines and social media platforms. Common Meta tags include title, description, keywords, and og:title (social sharing title). They don't display on the page but are crucial for SEO and social sharing.

Why are Meta tags important for SEO?

Meta tags help search engines understand page content: the title appears in search results, the description serves as the snippet text, affecting click-through rates. Missing or poor-quality Meta tags lead to lower rankings and CTR. Open Graph tags also control how pages appear when shared on social media.

How do I check a webpage's Meta tags?

Enter a webpage URL in this tool. It automatically fetches and analyzes all Meta tags including title, description, OG tags, Twitter card tags, canonical links, hreflang tags, and provides SEO optimization suggestions.

What are Open Graph tags?

Open Graph (OG) tags were created by Facebook to control how webpages appear when shared on social media. They include og:title, og:description, og:image, and og:url. Twitter, LinkedIn, and other platforms also support OG tags.

Is URL data uploaded to a server?

No. This tool fetches webpage content through a CORS proxy, and analysis happens locally in your browser. URLs and webpage content are not stored or uploaded to our servers.
