📖 Base64 Guide

What is Base64

Base64 is an encoding scheme that represents binary data using 64 printable characters (A-Z, a-z, 0-9, +, /). It converts every 3 bytes of 8-bit data into 4 characters of 6-bit data, enabling binary data to be safely transmitted in text environments.

Encoding Process

  1. Group input data into 3-byte (24-bit) chunks
  2. Split each 24-bit group into four 6-bit groups
  3. Map each 6-bit value (0-63) to the Base64 character table
  4. Pad with = if the last chunk is less than 3 bytes

Common Uses

  • Email attachments (MIME encoding)
  • Embedding images in web pages (data URI)
  • HTTP Basic Authentication
  • JWT token payload
  • Transmitting binary data in APIs

📖 FAQ

What is Base64 encoding?

Base64 is an encoding scheme that represents binary data using 64 printable characters. It converts every 3 bytes (24 bits) of data into 4 Base64 characters (6 bits x 4 = 24 bits). It is commonly used to transmit binary data like images and files over text-based protocols such as HTTP and email.

Does Base64 encoding increase data size?

Yes. Base64 encoding increases data size by approximately 33%. Since 3 bytes of raw data are encoded as 4 Base64 characters, the encoded size is 4/3 of the original, which is about a 33% increase.

What is URL-safe Base64?

Standard Base64 contains + and / characters which have special meanings in URLs. URL-safe Base64 replaces + with - and / with _, and typically removes trailing = padding. This makes the encoded string safe to use in URL parameters.

Is Base64 encryption?

No. Base64 is encoding, not encryption. It provides no security - anyone can easily decode a Base64 string. If you need to protect data, use encryption algorithms like AES. Base64's purpose is to safely transmit binary data in text environments.

Is my data uploaded to a server?

No. All encoding and decoding happens locally in your browser. No data is sent to any server. Your text and files are processed entirely on your device, ensuring privacy.
