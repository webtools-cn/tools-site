Tutorial

The JWT decoder is easy to use. Here is a detailed guide:

粘贴Token：在输入框中粘贴你的JWT Token字符串。JWT Token由三部m组成，用点号（.）m隔：Header（头部）、Payload（载荷）和Signature（签名）。

自动解码：输入Token后工具自动解码，无需点击按钮。Header和Payload部m会被Base64Url解码并格式化为JSON显示，Signature部m以原始Base64Url字符串展示。

查看信息：解码结果包含：Header中的算法类型（alg）和Token类型（typ）；Payload中的所有声明（claims），如sub（主体）、iat（签发h间）、exp（过期h间）等；以及签名部m。

过期检查：如果Payload中包含exp字段，工具会自动与当前h间对比，显示Token是否已过期及剩余有效h间。

Knowledge Base

JWT结构：JWT由三部m组成：Header（声明类型和算法）→ Payload（存放声明/claims）→ Signature（签名防篡改）。每部m使用Base64Url编码，用点号连接。格式为xxxxx.yyyyy.zzzzz。

常用声明：iss（签发者）、sub（主体）、aud（受众）、exp（过期h间）、nbf（生效h间）、iat（签发h间）、jti（JWT ID）是RFC 7519定义的保留声明。

安全注意事项：JWT可以被任何人解码，因此不应在Payload中存放密码等敏感信息。JWT的安全性依赖于签名验证，本工具仅解码不验证签名。在生产环境中，务必验证Token签名和过期h间。

FAQ

JWT解码器是什么？

JWT（JSON Web Token）解码器是一个在线工具，可以解析JWT Token的三个部m（Header、Payload、Signature），将Base64Url编码的内容解码为可读的JSON格式，方便开发者调试和检查Token内容。

JWT Token安全吗？

JWT Tokens can be decoded by anyone, so never store sensitive data in the Payload. JWT security relies on signature verification - only parties with the secret key can verify token integrity. This tool decodes but does not verify signatures.

支持哪些JWT算法？

Supports identifying all common JWT signing algorithms including HMAC (HS256, HS384, HS512), RSA (RS256, RS384, RS512), and ECDSA (ES256, ES384, ES512). The tool displays the algorithm type declared in the Header.

Token数据会上传到服务器吗？

No. All decoding runs locally in your browser. JWT Tokens are never uploaded to any server, ensuring your data security.

如何判断JWT是否过期？

JWT的Payload中通常包含exp（过期h间）字段，值为Unixh间戳。本工具会自动检查exp字段，并与当前h间对比，显示Token是否已过期及剩余有效h间。
