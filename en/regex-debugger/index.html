What is the Regex Debugger?

The Regex Debugger is an online tool for developers that provides real-time regex matching, highlight visualization, and capture group display. Enter a regex and test text, and the tool instantly highlights matches and shows each match's capture groups, helping you quickly verify and debug regular expressions.

Core Features

  • Real-time Match Highlighting: Match as you type, with different colors for different capture groups
  • Capture Group Visualization: Shows each match's capture group contents, supports named groups
  • Multiple Flags: Supports g/i/m/s/u flags with checkbox toggles
  • Common Regex Quick-Select: One-click load of email, URL, IP, date and other common patterns
  • Match Statistics: Real-time display of match count, execution time

How to Use

Step 1: Enter a regex in the pattern input, or select a common regex button.

Step 2: Enter test text in the text area.

Step 3: Check the highlight area for real-time match visualization.

Step 4: View capture group details in the match results. Shortcuts: Ctrl+Enter, Ctrl+Shift+C.

Use Cases

Form Validation Regex

Frontend developers can test email, phone and other form validation regexes in real-time, ensuring coverage of edge cases.

Log Analysis

DevOps engineers use regex to extract key info (IPs, timestamps, error codes) from logs, with capture group visualization confirming correct extraction.

Data Cleaning

Data analysts use regex to match and extract specific format data, with the debugger helping verify matching rules.

Technical Background

Regular Expressions describe string patterns for search, match, replace and validation. JavaScript uses the ECMAScript regex spec, supporting greedy/non-greedy quantifiers, lookahead/lookbehind assertions, and named capture groups. Performance tips: avoid backtracking traps (nested quantifiers), use non-capturing groups (?:), prefer specific characters over wildcards.

❓ FAQ

What regex flags are supported?
g (global), i (case-insensitive), m (multiline), s (dotAll), u (Unicode) and other common flags, toggleable via checkboxes.
How to view capture groups?
The match results area shows each capture group's index, name (for named groups), and content with different colors.
Is data uploaded?
No. Pure front-end. All matching happens locally. Works offline.
Are named capture groups supported?
Yes. Named groups using (?<name>pattern) display the group name and matched content.
Why no matches?
Common: syntax error, missing flags, or no matching content. The tool shows specific error messages.
What regex syntax is supported?
JavaScript regex syntax including character classes, quantifiers, anchors, groups, backreferences, lookaheads/lookbehinds.
Can I replace matches?
This version focuses on match debugging. For replacement, use our Regex Replace tool.
