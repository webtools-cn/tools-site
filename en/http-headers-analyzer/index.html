HTTP Headers Analyzer

HTTP request and response headers are a critical part of web security. This tool helps you analyze HTTP header fields, detect missing security configurations, CORS policy issues, and caching configurations. All analysis runs locally in your browser—no data is uploaded.

Important Security Headers

  • Content-Security-Policy (CSP): Prevent XSS attacks
  • Strict-Transport-Security (HSTS): Enforce HTTPS
  • X-Content-Type-Options: Prevent MIME sniffing
  • X-Frame-Options: Prevent clickjacking
  • Referrer-Policy: Control referrer information leakage
  • Permissions-Policy: Control browser feature permissions
What are HTTP headers?
HTTP headers are metadata fields in HTTP requests and responses that convey additional information about the request or response, such as content type, caching policy, and security policies.
Which HTTP headers are security-related?
Important security headers include: Content-Security-Policy (CSP), Strict-Transport-Security (HSTS), X-Content-Type-Options, X-Frame-Options, X-XSS-Protection, and Referrer-Policy.
How to check CORS configuration?
Check response headers for Access-Control-Allow-Origin, Access-Control-Allow-Methods, Access-Control-Allow-Headers fields. This tool automatically analyzes the security of these fields.
Is my data uploaded to a server?
No. All analysis runs locally in your browser.
