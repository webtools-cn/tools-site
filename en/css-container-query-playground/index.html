Best Practices

Prefer inline-size over size for better performance. Name containers with container-name to avoid conflicts. Start with small-screen styles and enhance with min-width.

FAQ

What are CSS container queries?

CSS container queries (@container) allow styling based on parent container size rather than viewport. Components adapt to their container, not the screen.

Container queries vs media queries?

Media queries respond to viewport width; container queries respond to parent container width. Container queries are better for component-level responsive design.

Browser support?

Supported by all major browsers (Chrome 105+, Firefox 110+, Safari 16+). Global support over 90%.

inline-size vs size container-type?

inline-size establishes containment on inline axis, better performance; size on both axes, supports height queries but costs more.

Can container queries be nested?

Yes. Inner containers can have their own container-type and container-name, creating nested contexts.
