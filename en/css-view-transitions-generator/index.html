Background

The View Transitions API, introduced in Chrome 111, creates animated transitions by snapshotting old and new states. Core mechanism: browser captures old snapshot → DOM updates → captures new snapshot → CSS animates between snapshots. Key pseudo-elements: ::view-transition (root), ::view-transition-group, ::view-transition-old, ::view-transition-new. Compared to JS animation libraries, it offers better performance, less code, and graceful degradation.

FAQ

What is the View Transitions API?▼
A native browser API for creating smooth page transition animations. No JS animation library needed — just CSS for SPA and MPA page transitions.
Which browsers support it?▼
Chrome 111+ and Edge 111+ natively. Safari and Firefox in development. Graceful degradation when unsupported.
SPA vs MPA transitions?▼
SPA uses document.startViewTransition() for same-page DOM changes. MPA uses a meta tag for cross-page transitions. Same CSS, different JS calls.
What does view-transition-name do?▼
It assigns a transition name to an element, enabling morph animations between pages for elements with matching names. E.g., a list image morphs into a detail page image.
