📖 API Status Dashboard

What is API Status Monitoring?

API status monitoring involves periodically sending requests to API endpoints to check availability, response time, and status codes. Continuous monitoring helps detect API failures and performance degradation early, ensuring stable service operation.

Core Features

  • 📊 Multi-endpoint simultaneous monitoring
  • ⏱️ Response time measurement
  • 📈 History tracking and trend charts
  • 🔄 Scheduled automatic checks
  • 📋 Availability statistics
  • 🔒 Pure frontend processing, data stored in localStorage

How to Use

  1. Enter endpoint name and URL, click "Add"
  2. Select check interval (Manual/30s/1min/5min)
  3. Click "Check All" to start monitoring
  4. View endpoint status, response time, and history

Use Cases

  • 👨‍💻 Developers monitoring their own APIs
  • 🔧 Ops teams checking third-party service availability
  • 📊 Testing API performance and response times
  • 🔍 Troubleshooting API connection issues
  • 📋 Generating API availability reports

📖 FAQ

What is API Status Monitoring?

API status monitoring involves periodically sending requests to API endpoints to check availability, response time, and status codes. Continuous monitoring helps detect API failures and performance degradation early, ensuring stable service operation.

How does this tool check API status?

本工具使用浏览器的fetch API向目标端点发送HEAD或GET请求，测量响应时间和状态码。所有请求从你的浏览器发出，不经过任何中间服务器。注意：某些API可能因CORS策略restrictions, cannot从浏览器直接检测。

Will CORS restrictions affect check results?

会。如果目标API没有设置允许跨域访问的CORS头，浏览器会阻止请求。对于这种情况，工具会显示CORSError。建议监控自己控制的API（已配置CORS），或使用公共API（通常已开放CORS）。

What monitoring features are supported?

Supports: 1) Adding multiple API endpoints; 2) Manual or scheduled checks (30s/1min/5min); 3) Response time and status code history; 4) Availability percentage statistics; 5) Response time charts; 6) Status change notifications.

Will data be uploaded to a server?

No. All monitoring data is stored in the browser localStorage and is never uploaded to any server. API check requests are sent directly from your browser to the target API without going through any intermediate server.
