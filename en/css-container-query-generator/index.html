What Are CSS Container Queries?

CSS Container Queries are a modern CSS feature that allows you to style elements based on the size of their parent container rather than the viewport. Unlike media queries which respond to the browser window size, container queries enable truly responsive components that adapt to their layout context — whether in a sidebar, main content area, or card layout.

Core Features

  • Visual Builder: Add breakpoints and see generated CSS in real-time
  • Live Preview: Resize the preview container to see how styles change
  • Named Containers: Support for container-name for targeted queries
  • Multiple Breakpoints: Add as many breakpoints as needed with min-width/max-width
  • Unit Support: px, rem, and em units for breakpoint values

How to Use

1. Set the container name and type (inline-size is most common).

2. Add breakpoints with min-width or max-width conditions.

3. Define child styles for each breakpoint.

4. Copy the generated CSS and use it in your project.

Container Queries vs Media Queries

Media queries respond to the viewport: @media (min-width: 768px). Container queries respond to a parent element: @container (min-width: 400px). This means a component can look different in a sidebar vs the main content area, even at the same viewport width. Container queries are the key to building truly reusable, context-aware responsive components.

Browser Support

Container queries are supported in Chrome 105+, Firefox 110+, Safari 16+, and Edge 105+. This covers over 90% of global browser usage. For older browsers, you can use a progressive enhancement approach with media queries as fallback.

❓ FAQ

What are CSS container queries?

Styles that apply based on the size of a parent container, not the viewport. Enables truly responsive components.

Container queries vs media queries?

Media queries respond to viewport size. Container queries respond to parent container size, making components more reusable.

Browser support?

Chrome 105+, Firefox 110+, Safari 16+, Edge 105+. Over 90% global coverage.

Is my data uploaded?

No. All CSS generation happens locally in your browser.

What is container-type?

Defines containment type: inline-size (width queries), size (both axes), or normal (no containment).

Can I use named containers?

Yes. Use container-name and reference it: @container sidebar (min-width: 300px).
