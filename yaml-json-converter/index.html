YAML与JSON格式对比

YAML（YAML Ain't Markup Language）使用缩进和键值对表示数据，可读性强，常用于配置文件（Docker Compose、Kubernetes、Ansible等）。JSON（JavaScript Object Notation）使用大括号和引号，是Web API最广泛使用的数据交换格式。两者可以无损互转。

❓ 常见问题

YAML和JSON可以完全互转吗？

大多数情况下可以。YAML是JSON的超集，支持JSON所有数据类型。但YAML的一些高级特性（如锚点、别名、多行字符串等）在JSON中没有对应概念，转换时会丢失。

为什么我的YAML解析失败？

常见原因包括：缩进不一致（请使用空格而非Tab）、冒号后面缺少空格、字符串未加引号导致特殊字符被误解。本工具会显示详细的错误位置。

支持大文件转换吗？

纯前端处理，受浏览器内存限制。一般几百KB的文件可以正常处理。超大文件建议在本地使用专业工具转换。
