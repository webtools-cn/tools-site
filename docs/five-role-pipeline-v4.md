> 核心：数量 × 质量 × 流量 = 变现
> 铁律：开发不推送，验收才推送

## 二、流程

```
OP: 采集竞品信息/市场动态/用户反馈 → 同步给PM
  ↓
PM: 存量分级(A/B/C/D) + 竞品差距 + SEO策略 + 定验收标准
  ↓
UX: 定布局+风格规范 + SEO/GEO融合设计
  ↓
DEV: 开发/优化 → 自检 → 代码留工作区（禁止push！）
  ↓
QA-Test: 功能/代码/SEO/GEO/中英文验收
  ├── ❌ 不通过 → 打回DEV
  └── ✅ 通过 → 进入UI验收
       ↓
QA-UI: 视觉/布局/风格/移动端验收（Kimi WebBridge浏览器实测）
  ├── ❌ 不通过 → 打回DEV（说明视觉问题）
  └── ✅ 通过 → git commit + push + 线上验证
```

## 三、QA-Test 验收Checklist

## 四、QA-UI 验收Checklist（Kimi WebBridge实测）

### 视觉一致性
- [ ] 主色调与风格基线一致（--primary/--bg/--text）
- [ ] 按钮样式统一（圆角/颜色/大小/悬停效果）
- [ ] 输入框样式统一（边框/背景/圆角/聚焦效果）
- [ ] 字体大小统一（h1≈2rem / h2≈1.5rem / body≈0.95rem）
- [ ] 间距统一（padding/margin与标准页一致）

### 布局
- [ ] 无空白区域（布局紧凑，不浪费空间）
- [ ] 内容区宽度合理（不超宽不超窄）
- [ ] 卡片/面板对齐整齐
- [ ] 响应式布局正确（非固定宽度）

### 移动端
- [ ] 375px不崩不溢出
- [ ] 按钮可点击（不太小）
- [ ] 输入框可用（不遮挡）
- [ ] 横向无滚动条

### 交互体验
- [ ] 操作流程顺畅（输入→点击→输出）
- [ ] 结果展示清晰（不挤不散）
- [ ] 复制/清空按钮位置合理
- [ ] toast通知正常显示

### 与标准页对比
- [ ] 打开标准页(json-formatter)和新工具页对比
- [ ] 整体视觉感受一致，不像两个站

## 七、QA浏览器实测方法

使用Kimi WebBridge（daemon 127.0.0.1:10086）：
1. `navigate` 打开本地http.server 8899上的页面
2. `snapshot` 读取页面结构
3. `click` / `fill` 测试交互
4. `evaluate` 检查console/JS状态
5. `screenshot` 截图对比视觉
6. **用完立即 `close_tab`**，防止内存爆掉浏览器崩溃
7. 每次最多开3个tab，用完关掉再开新的

## 八、PM存量分级标准

| 级别 | 条件 | 动作 |
|:-----|:-----|:-----|
| A级 | 搜索量大+竞品弱+我们有优势 | 逐个深度优化到个税计算器水平 |
| B级 | 搜索量中等+需SEO/内容优化 | 批量提升SEO+FAQ+内容 |
| C级 | 搜索量小+维护成本低 | 保持现状 |
| D级 | 空壳/质量差/误导用户 | 下线或重做 |
