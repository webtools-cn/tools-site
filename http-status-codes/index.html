    什么是HTTP状态码

    HTTP状态码（HTTP Status Code）是服务器在响应客户端HTTP请求时返回的三位数字代码，用于表示请求的处理结果。HTTP状态码由RFC 2616规范定义，后在RFC 7231中更新扩展，是Web开发和API设计中的核心概念。

    每个状态码由三位数字组成，第一位数字表示响应类别。例如 200 表示请求成功，404 表示资源未找到，500 表示服务器内部错误。了解HTTP状态码对开发者调试Web应用、排查网络问题、优化SEO策略至关重要。

    HTTP状态码分类一览

    类别范围含义典型场景
    1xx100-199信息响应请求已接收，继续处理
    2xx200-299成功请求已成功接收和处理
    3xx300-399重定向需要进一步操作完成请求
    4xx400-499客户端错误请求有语法错误或无法完成
    5xx500-599服务器错误服务器处理请求时出错

    常见问题

    HTTP状态码是什么？

    HTTP状态码是服务器对客户端请求的响应状态标识，由三位数字组成。它告诉浏览器或API客户端请求是否成功、是否需要重定向、是否发生错误等。状态码分为5大类：1xx信息响应、2xx成功、3xx重定向、4xx客户端错误、5xx服务器错误。

    最常见的HTTP状态码有哪些？

    最常见的HTTP状态码包括：200 OK（请求成功）、301永久重定向、302临时重定向、304未修改、400错误请求、401未授权、403禁止访问、404未找到、500服务器内部错误、502网关错误、503服务不可用、504网关超时。其中404和500是最常遇到的错误状态码。

    404和403有什么区别？

    404 Not Found表示服务器找不到请求的资源，通常是因为URL错误或资源已被删除。403 Forbidden表示服务器理解请求但拒绝执行，通常是因为权限不足或服务器配置禁止访问。简单来说，404是"不存在"，403是"存在但你不能访问"。

    301和302重定向有什么区别？

    301 Moved Permanently表示永久重定向，搜索引擎会将原URL的权重转移到新URL，适合域名迁移或URL结构变更。302 Found表示临时重定向，搜索引擎会保留原URL的索引，适合临时维护或A/B测试。SEO角度应优先使用301进行永久性跳转。

    500、502、503、504都是服务器错误有什么区别？

    500 Internal Server Error是服务器内部错误，通常是代码异常或配置问题。502 Bad Gateway是网关/代理从上游收到无效响应，通常是后端服务崩溃。503 Service Unavailable是服务暂时不可用，通常是过载或维护中。504 Gateway Timeout是网关等待上游超时，通常是后端响应太慢。
