HTTP状态码分类

  • 1xx（信息） — 已接收请求，继续处理
  • 2xx（成功） — 请求已成功接收、理解和接受
  • 3xx（重定向） — 需要进一步操作以完成请求
  • 4xx（客户端错误） — 请求包含错误语法或无法完成
  • 5xx（服务器错误） — 服务器未能完成有效请求

常见问题

什么是HTTP状态码？

HTTP状态码是Web服务器用于指示HTTP请求是否成功完成的标准响应代码。分为五类：1xx（信息）、2xx（成功）、3xx（重定向）、4xx（客户端错误）和5xx（服务器错误）。

最常见的HTTP状态码是什么？

200 OK是最常见的成功响应。404 Not Found是最广为人知的错误码。500 Internal Server Error是最常见的服务器端错误。301和302是最常见的重定向码。

301和302有什么区别？

301永久重定向表示资源已永久移动到新URL，搜索引擎会更新索引到新URL。302临时重定向表示临时移动，搜索引擎保留原URL，不转移排名权重。

遇到403禁止访问怎么办？

403禁止访问意味着服务器理解请求但拒绝授权。检查：文件/目录权限、.htaccess或nginx配置、IP白名单/黑名单规则、认证要求、WAF或防火墙拦截规则。

401和403有什么区别？

401未授权表示需要认证且认证失败或未提供，客户端可携带凭据重试。403禁止访问表示服务器拒绝授权请求无论认证状态如何，重试凭据也无济于事。
