❓ 常见问题 (FAQ)

JSON转YAML工具的数据安全吗？
绝对安全。本工具完全在浏览器本地运行，你输入的JSON数据不会上传到任何服务器。所有转换操作都在你的设备上完成。即使断开网络连接，工具也能正常使用。
JSON和YAML的主要区别是什么？
JSON（JavaScript Object Notation）是一种轻量级的数据交换格式，语法严格，要求键名和字符串必须使用双引号。YAML（YAML Ain't Markup Language）则更注重可读性，使用缩进表示层级关系，支持注释、多行字符串等特性，语法更灵活简洁。
YAML在哪些场景下比JSON更合适？
YAML在以下场景中优势明显：配置文件（如Docker Compose、Kubernetes、Ansible、GitHub Actions）、复杂的数据结构描述、需要注释的配置文件、CI/CD流水线定义。YAML的可读性使其在配置管理领域成为事实标准。
转换后的YAML可以直接用于Kubernetes吗？
是的，本工具生成的YAML格式兼容Kubernetes配置文件的语法要求。但请注意Kubernetes对特定字段名称和格式有额外要求，建议转换后根据K8s文档验证配置的正确性。
支持哪些JSON特性？
本工具支持标准的JSON格式，包括对象、数组、字符串、数字、布尔值和null。嵌套结构、空对象和空数组都能正确处理。支持大文件（建议不超过5MB）和复杂嵌套结构。
