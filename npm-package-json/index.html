常见问题

什么是 package.json？

package.json 是 Node.js 项目的核心配置文件，包含项目名称、版本、依赖关系、脚本命令等元数据。每个 npm 包和 Node.js 项目都必须有一个 package.json 文件。它是 npm 生态系统的基础，用于管理依赖、定义脚本和发布包。

package.json 中的 scripts 字段有什么作用？

scripts 字段定义了一组可以通过 npm run 执行的命令。常见的脚本包括 start（启动应用）、test（运行测试）、build（构建项目）。使用 npm run 脚本名 即可执行对应命令，npm 会自动将 node_modules/.bin 添加到 PATH 中。

dependencies 和 devDependencies 有什么区别？

dependencies 是生产环境必需的依赖，如 Express、Lodash 等运行时库。devDependencies 仅用于开发环境，如测试框架（Jest）、构建工具（Webpack）、代码检查工具（ESLint）。安装时使用 npm install --save-dev 添加到 devDependencies。

如何初始化一个新的 npm 项目？

使用 npm init 命令可以交互式创建 package.json。如果想快速生成默认配置，可以使用 npm init -y。使用本工具可以可视化填写所有字段，生成后复制或下载到项目根目录即可。
