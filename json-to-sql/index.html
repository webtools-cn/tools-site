使用教程

本JSON转SQL工具操作简单，三步即可完成数据迁移准备工作：

第一步：准备JSON数据。将需要导入数据库的JSON数据粘贴到输入框。支持两种格式：单个对象（如{"name":"张三","age":25}）或对象数组（如[{"name":"张三","age":25},{"name":"李四","age":30}]）。工具会自动解析JSON结构，提取所有字段名。如果输入的是数组，工具会以第一个对象为准推断字段列表。

第二步：配置生成选项。在选项区域选择目标数据库类型（MySQL、PostgreSQL、SQLite或SQL Server），不同数据库的引号风格和标识符转义方式会自动适配。选择插入模式：逐条INSERT适合小数据量，每条数据独立执行；批量INSERT将多条数据合并为单个INSERT语句，大幅提升执行效率。还可以设置表名、开启IF NOT EXISTS等选项。

第三步：生成并复制SQL。点击「生成SQL」按钮，工具会立即解析JSON并生成对应的INSERT语句。生成结果会显示在下方区域，点击「复制SQL」即可复制到剪贴板，或点击「下载.sql文件」保存为SQL脚本文件。生成前建议先点击「格式化JSON」检查数据格式是否正确。

应用场景

JSON转SQL工具在多种开发和数据处理场景中发挥重要作用：

API数据导入数据库：当后端API返回JSON格式数据时，开发者经常需要将这些测试数据或迁移数据批量导入数据库进行调试。本工具可以快速将API响应转换为可直接执行的INSERT语句，省去手动编写SQL的繁琐工作，尤其适合处理包含数十甚至数百条记录的JSON数组。

测试数据生成：在开发阶段，团队通常需要大量模拟数据来测试应用功能。通过JSON Mock数据结合本工具，可以快速生成对应的SQL插入脚本，一次性填充测试数据库。配合「自动类型转换建议」功能，还可以辅助判断每个字段应使用何种数据库类型。

数据迁移与备份恢复：当需要将JSON格式的数据备份恢复为关系型数据库格式时，或者在异构系统间进行数据迁移（如从NoSQL迁移到SQL数据库），本工具可以作为中间桥梁，先将JSON数据标准化为SQL INSERT语句，再导入目标数据库。

扩展知识

不同数据库的INSERT语法差异：虽然SQL标准是统一的，但各数据库在引号风格上有细微差别。MySQL使用反引号 ` 包裹标识符，PostgreSQL使用双引号 "，SQL Server使用方括号 []，SQLite则兼容多种风格。本工具会根据您选择的数据库类型自动选择正确的引号风格。

批量插入的性能优势：逐条INSERT需要多次网络往返和事务提交，效率极低。而批量INSERT（如 INSERT INTO table VALUES (...), (...), (...)）将多条记录合并为单个语句，大幅减少网络开销和事务日志写入。对于大批量数据导入，批量模式通常比逐条模式快10倍以上。

数据类型推断的局限性：工具根据JSON值的类型进行简单推断：字符串→VARCHAR/TEXT，数字→INT/FLOAT，布尔→BOOLEAN/TINYINT，null→NULL。但JSON本身不携带精度、长度等元数据，实际建表时建议根据业务需求调整字段类型。工具提供的「自动类型转换建议」仅供参考，正式环境使用前请核对。

常见问题解答

支持哪些数据库类型？

目前支持MySQL、PostgreSQL、SQLite和SQL Server四种主流数据库。不同数据库在引号风格和标识符转义上有差异，工具会自动适配对应语法。

数据会发送到服务器吗？

不会。本工具完全在浏览器前端运行，所有JSON解析和SQL生成都由JavaScript在本地完成，数据不会上传到任何服务器，保障您的数据隐私安全。

支持嵌套JSON对象吗？

对于嵌套对象和数组类型的字段值，工具会自动将其序列化为JSON字符串（PostgreSQL使用::jsonb，其他数据库作为普通文本插入）。建议在实际使用前根据数据结构调整字段类型。

生成的SQL可以直接执行吗？

生成的INSERT语句是标准SQL语法，但建议在执行前先检查表名和字段名是否与实际数据库一致。如果JSON字段与数据库表结构不完全匹配，可能需要手动调整字段映射或补充缺失字段。

如何生成批量INSERT语句？

当输入JSON数组时，默认会生成单条INSERT语句（每条数据一个INSERT）。您可以在选项中开启批量插入模式，此时会生成VALUES多行语法，大幅提升导入效率。

NULL值如何处理？

JSON中的null值会被转换为SQL的NULL关键字（不加引号）。空字符串''和null是不同的，前者是空字符串，后者表示无值。工具会严格区分这两种情况。

可以自定义表名吗？

可以。在输入区域上方有表名输入框，您可以指定任意表名。如果留空，工具会自动使用'data_table'作为默认表名。表名中的特殊字符会自动转义。
