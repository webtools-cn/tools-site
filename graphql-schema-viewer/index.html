什么是GraphQL Schema？

GraphQL Schema是API的类型定义，使用SDL描述数据类型、查询入口、变更操作和订阅，是客户端和服务端的契约。

什么是SDL格式？

SDL是GraphQL的类型定义语言，用简洁文本语法定义类型、字段、参数和关系，如type User { id: ID! name: String! }。

支持哪些GraphQL类型？

支持所有标准类型：Object Type、Interface、Union、Enum、Input Type、Scalar，以及Query/Mutation/Subscription根类型。

数据安全吗？

完全安全。所有Schema解析和展示在浏览器本地完成，数据绝不上传。

如何查看类型详情？

点击类型卡片展开详情，查看字段列表、参数、返回类型和描述。

可以搜索类型吗？

可以。在搜索框中输入类型名或字段名，实时过滤匹配结果。

支持拖拽导入文件吗？

支持。直接拖拽.graphql文件到编辑区域即可自动解析。

GraphQL Schema查看器能做什么？

GraphQL Schema查看器是一款免费在线开发者工具，用于解析和可视化展示GraphQL Schema定义。支持SDL（Schema Definition Language）格式输入，自动识别所有类型定义，包括Object Type、Query、Mutation、Subscription、Enum、Input Type、Interface和Scalar。解析后以直观的卡片形式展示类型结构和字段信息，支持搜索筛选和分类浏览。

使用教程

1. 在文本框中粘贴GraphQL Schema SDL内容，或拖拽.graphql文件导入。

2. 点击"解析"按钮，工具自动识别所有类型定义。

3. 在统计区查看各类型数量，使用分类标签筛选特定类型。

4. 点击类型卡片展开查看完整字段列表、参数和描述。

5. 使用搜索框快速查找类型或字段。Ctrl+Enter快速解析，Ctrl+Shift+C复制结果。

扩展知识

GraphQL Schema是GraphQL API的核心，定义了数据的类型系统和操作入口。SDL是GraphQL的类型定义语言，使用简洁的文本语法描述类型和关系。Schema包含三种根操作类型：Query（读取数据）、Mutation（修改数据）和Subscription（实时订阅）。类型系统支持Object Type（对象类型）、Interface（接口）、Union（联合类型）、Enum（枚举）、Input Type（输入类型）和Scalar（标量）。字段可以带参数（arguments），使用!标记非空类型，使用[Type]表示列表类型。Introspection查询允许客户端在运行时获取Schema信息。
