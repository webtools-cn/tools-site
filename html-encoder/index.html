HTML实体编码知识

什么是HTML实体编码？

HTML实体编码是将HTML中的特殊字符转换为对应的实体引用。例如 < 表示 <，& 表示 &。这样可以安全地在网页中显示这些字符而不被浏览器误解析。

常用HTML实体

• &amp; → &（和号）

• &lt; → <（小于号）

• &gt; → >（大于号）

• &quot; → "（双引号）

• &#39; → '（单引号）

• &nbsp; → 不换行空格

• &copy; → ©（版权符号）

命名实体 vs 数字实体

命名实体使用名称（如 &lt;），数字实体使用Unicode码点（如 &#60; 或十六进制 &#x3C;）。数字实体可以表示任意Unicode字符，命名实体只能表示预定义的字符。

什么是HTML实体编码？

HTML实体编码是将HTML中的特殊字符（如 <、>、&、"、' 等）转换为对应的实体引用（如 &lt;、&gt;、&amp;、&quot;、&#39;），以避免这些字符被浏览器解析为HTML标签或属性，常用于在网页中安全地显示代码或特殊字符。

HTML实体编码和URL编码有什么区别？

HTML实体编码用于在HTML文档中安全显示特殊字符，使用 & 开头的实体引用。URL编码用于在URL中传输特殊字符，使用 % 开头的百分号编码。两者用途不同，不能混用。

数据是否安全？会上传到服务器吗？

完全安全。本工具是纯前端应用，所有编码解码操作均在浏览器本地完成，数据绝不上传到任何服务器，可放心使用。

URL编码知识

什么是URL编码？

URL编码（也称百分号编码）是将URL中的特殊字符转换为 %XX 格式的十六进制表示。例如空格编码为 %20，中文字符"中"编码为 %E4%B8%AD。

encodeURI vs encodeURIComponent

• encodeURI：用于编码完整URL，保留 ;/?:@&=+$,# 等URL保留字符

• encodeURIComponent：用于编码URL参数值，编码所有特殊字符（除字母、数字和 -_.~!*' 外）

常用场景

• URL参数：在查询字符串中传递特殊字符

• API请求：在REST API中编码路径和参数

• 中文URL：将中文字符编码为百分号格式

• 表单提交：application/x-www-form-urlencoded 格式

URL编码中的空格是 %20 还是 +？

两者都对。%20 是标准的URL编码，+ 是 application/x-www-form-urlencoded 格式中的表示法（主要用于表单提交）。本工具使用 %20 标准格式。

支持中文吗？

支持。工具使用浏览器原生的 encodeURI / encodeURIComponent API，完整支持中文、Emoji等所有Unicode字符的编码和解码。

Unicode转换知识

什么是Unicode转义？

Unicode转义是将字符表示为 \uXXXX 格式的转义序列，其中 XXXX 是字符Unicode码点的十六进制表示。例如汉字"中"可表示为 \u4e2d。

常见Unicode表示法

• \uXXXX：JavaScript/JSON 中的转义序列

• U+XXXX：Unicode标准码点表示法

• &#XXXX;：HTML数字实体

• %XX：URL百分号编码（UTF-8字节）

BMP与补充平面

基本多文种平面（BMP）内的字符（U+0000 至 U+FFFF）可用单个 \uXXXX 表示。超出BMP的字符（如部分Emoji）需要用代理对（两个 \uXXXX）表示，或使用ES6的 \u{XXXXX} 语法。

Unicode转义 \uXXXX 是什么意思？

\uXXXX 是一种Unicode转义序列，其中 XXXX 是字符的Unicode码点的十六进制表示。例如 \u4e2d 表示汉字"中"。这种表示法常用于JavaScript字符串、JSON等场景中。

Emoji表情怎么转换？

大部分Emoji位于Unicode补充平面（U+10000以上），需要用代理对表示（两个\uXXXX）。本工具支持代理对的编解码，也支持ES6的 \u{XXXXX} 语法。
