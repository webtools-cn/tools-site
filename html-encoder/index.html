使用教程

本工具整合了 HTML 实体编码、URL 编解码和 Unicode 转换三大功能，采用标签页切换的方式组织，操作简洁高效。以下是针对不同功能的详细使用步骤：

HTML 实体编码/解码：默认打开的是「HTML实体编码」标签页。在输入框中粘贴需要转换的文本，例如包含 <、>、& 等特殊字符的代码片段。点击「编码 →」按钮，工具会将这些字符转换为对应的 HTML 实体引用（如 &lt;、&gt;）。如果你希望将所有非 ASCII 字符也一并编码，可以勾选「完整编码」选项。勾选「使用十六进制实体」则会输出 &#x3C; 格式而非十进制的 &#60;。点击「← 解码」可将实体引用还原为原始字符。

URL 编解码：切换到「URL编解码」标签页。输入需要编码的 URL 或查询参数，点击「编码 →」即可转换为百分号编码格式（如空格变为 %20，中文变为 %E4%B8%AD）。勾选「使用 encodeURIComponent」选项会进行更严格的编码，适用于编码 URL 参数值而非完整 URL。点击「← 解码」可将编码后的 URL 还原。

Unicode 转换：切换到「Unicode转换」标签页。输入任意字符，点击「转义 →」会将其转换为 \uXXXX 格式的 Unicode 转义序列。勾选「大写十六进制」可使输出为大写形式（如 \u4E2D），勾选「补零至4位」可确保所有码点都是4位十六进制。点击「← 反转义」则将转义序列还原为原始字符。

应用场景

安全展示 HTML 源代码：在编写技术博客、文档或论坛帖子时，经常需要在网页中展示 HTML 代码片段。如果直接粘贴，浏览器会将其解析为实际标签，导致显示异常。使用本工具的 HTML 实体编码功能，可以安全地将代码转换为实体引用后粘贴，确保读者看到的是代码文本而非渲染结果。

URL 参数构造与调试：在开发 Web 应用或调用 REST API 时，经常需要将包含中文、空格或特殊符号的数据作为 URL 参数传递。手动编码容易出错且效率低下。使用 URL 编解码功能，可以快速准确地完成编码，避免因字符未正确转义导致的请求失败或安全问题。

跨平台字符处理：在 JavaScript、JSON 或某些不支持 Unicode 直接输入的编程环境中，需要将非 ASCII 字符（如中文、Emoji）转换为 \uXXXX 转义序列。Unicode 转换功能支持基本平面字符以及 Emoji 等补充平面字符（通过代理对或 \u{XXXXX} 语法），满足开发者的国际化字符处理需求。

扩展知识

HTML 实体的分类：HTML 实体分为命名实体（Named Entity，如 &lt;）和数字实体（Numeric Entity，如 &#60; 或 &#x3C;）。命名实体只覆盖预定义的字符子集（约250个），而数字实体可以表示任意 Unicode 字符（超过14万个码点）。本工具在编码时会优先使用命名实体表示常见字符，完整编码模式下则将所有非 ASCII 字符转为数字实体。

URL 编码的 RFC 3986 标准：URL 编码遵循 RFC 3986 标准，规定 URL 中只能使用 ASCII 字符集的子集（unreserved 字符：A-Z、a-z、0-9、-、_、.、~）。其他字符必须通过 UTF-8 编码后再转为百分号格式。encodeURI 和 encodeURIComponent 的区别在于：前者保留 URL 结构字符（如 /、?、=），后者则会编码这些字符，因此后者更适合编码参数值。

Unicode 与 UTF-8 的关系：Unicode 是字符集标准，为每个字符分配唯一的码点（Code Point）。UTF-8 是将这些码点编码为字节序列的编码方式。URL 编码使用的是 UTF-8 字节序列的十六进制表示（如「中」的 Unicode 码点是 U+4E2D，UTF-8 编码为 E4 B8 AD，URL 编码为 %E4%B8%AD）。理解这一层次关系，有助于正确选择本工具的三种编码模式。

HTML实体编码知识

什么是HTML实体编码？

HTML实体编码是将HTML中的特殊字符转换为对应的实体引用。例如 < 表示 <，& 表示 &。这样可以安全地在网页中显示这些字符而不被浏览器误解析。

常用HTML实体

• &amp; → &（和号）

• &lt; → <（小于号）

• &gt; → >（大于号）

• &quot; → "（双引号）

• &#39; → '（单引号）

• &nbsp; → 不换行空格

• &copy; → ©（版权符号）

命名实体 vs 数字实体

命名实体使用名称（如 &lt;），数字实体使用Unicode码点（如 &#60; 或十六进制 &#x3C;）。数字实体可以表示任意Unicode字符，命名实体只能表示预定义的字符。

什么是HTML实体编码？

HTML实体编码是将HTML中的特殊字符（如 <、>、&、"、' 等）转换为对应的实体引用（如 &lt;、&gt;、&amp;、&quot;、&#39;），以避免这些字符被浏览器解析为HTML标签或属性，常用于在网页中安全地显示代码或特殊字符。

HTML实体编码和URL编码有什么区别？

HTML实体编码用于在HTML文档中安全显示特殊字符，使用 & 开头的实体引用。URL编码用于在URL中传输特殊字符，使用 % 开头的百分号编码。两者用途不同，不能混用。

数据是否安全？会上传到服务器吗？

完全安全。本工具是纯前端应用，所有编码解码操作均在浏览器本地完成，数据绝不上传到任何服务器，可放心使用。

URL编码知识

什么是URL编码？

URL编码（也称百分号编码）是将URL中的特殊字符转换为 %XX 格式的十六进制表示。例如空格编码为 %20，中文字符"中"编码为 %E4%B8%AD。

encodeURI vs encodeURIComponent

• encodeURI：用于编码完整URL，保留 ;/?:@&=+$,# 等URL保留字符

• encodeURIComponent：用于编码URL参数值，编码所有特殊字符（除字母、数字和 -_.~!*' 外）

常用场景

• URL参数：在查询字符串中传递特殊字符

• API请求：在REST API中编码路径和参数

• 中文URL：将中文字符编码为百分号格式

• 表单提交：application/x-www-form-urlencoded 格式

URL编码中的空格是 %20 还是 +？

两者都对。%20 是标准的URL编码，+ 是 application/x-www-form-urlencoded 格式中的表示法（主要用于表单提交）。本工具使用 %20 标准格式。

支持中文吗？

支持。工具使用浏览器原生的 encodeURI / encodeURIComponent API，完整支持中文、Emoji等所有Unicode字符的编码和解码。

Unicode转换知识

什么是Unicode转义？

Unicode转义是将字符表示为 \uXXXX 格式的转义序列，其中 XXXX 是字符Unicode码点的十六进制表示。例如汉字"中"可表示为 \u4e2d。

常见Unicode表示法

• \uXXXX：JavaScript/JSON 中的转义序列

• U+XXXX：Unicode标准码点表示法

• &#XXXX;：HTML数字实体

• %XX：URL百分号编码（UTF-8字节）

BMP与补充平面

基本多文种平面（BMP）内的字符（U+0000 至 U+FFFF）可用单个 \uXXXX 表示。超出BMP的字符（如部分Emoji）需要用代理对（两个 \uXXXX）表示，或使用ES6的 \u{XXXXX} 语法。

Unicode转义 \uXXXX 是什么意思？

\uXXXX 是一种Unicode转义序列，其中 XXXX 是字符的Unicode码点的十六进制表示。例如 \u4e2d 表示汉字"中"。这种表示法常用于JavaScript字符串、JSON等场景中。

Emoji表情怎么转换？

大部分Emoji位于Unicode补充平面（U+10000以上），需要用代理对表示（两个\uXXXX）。本工具支持代理对的编解码，也支持ES6的 \u{XXXXX} 语法。
