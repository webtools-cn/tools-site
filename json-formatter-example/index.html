📖 JSON格式化示例详解

什么是JSON格式化？

JSON格式化（JSON Formatting / Pretty Print）是将压缩成一行的JSON字符串，通过添加缩进和换行，转换为人类可读的格式。例如将 {"name":"张三","age":25} 格式化为多行缩进结构，方便阅读和调试。

常见JSON格式示例

  • 用户信息：包含name、email、age等字段的用户对象
  • API响应：包含status、data、message的标准API返回格式
  • 配置文件：包含database、server、logging等配置项的JSON
  • 嵌套结构：多层嵌套的对象和数组组合
  • 地理位置：GeoJSON格式的经纬度坐标数据

JSON格式规范

JSON（JavaScript Object Notation）是一种轻量级数据交换格式。基本规则：键名必须用双引号包裹，字符串值也必须用双引号，不支持注释，最后一个元素后不能有逗号。

❓ 常见问题

JSON格式化是什么？

JSON格式化是将压缩的JSON字符串转换为缩进整齐、易读的格式。通常使用2或4个空格缩进，添加换行符，使数据结构一目了然。

如何验证JSON是否有效？

将JSON字符串粘贴到工具中，点击格式化按钮。如果JSON有效，会输出格式化后的结果；如果无效，会显示具体的错误位置和原因。

JSON和JSONP有什么区别？

JSON是纯数据格式，JSONP是JSON with Padding，通过回调函数包裹JSON数据来实现跨域请求。现代开发中推荐使用CORS代替JSONP。

JSON支持哪些数据类型？

JSON支持6种数据类型：字符串（string）、数字（number）、布尔值（boolean）、null、对象（object）和数组（array）。不支持注释、日期、undefined等。

数据会上传到服务器吗？

不会。所有JSON格式化和校验都在浏览器本地完成，不发送任何数据到服务器，隐私安全。
