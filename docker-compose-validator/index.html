❓ 常见问题

支持哪个版本的Docker Compose？
支持V2和V3语法，包括services、networks、volumes顶级键和常用服务配置项。
能检测哪些配置问题？
YAML语法错误、缺少必要字段、无效端口映射、无效卷挂载、重复服务名等。
数据会上传到服务器吗？
不会。所有验证都在浏览器本地完成。
验证通过是否代表一定能运行？
不一定。本工具检查语法和常见配置问题，不验证镜像是否存在等运行时问题。
如何修复检测到的问题？
每个错误都标注了行号和具体原因，根据提示修改对应行即可。
支持Docker Compose的extends吗？
V3版本已废弃extends，V2版本支持但本工具暂不验证extends引用的有效性。

Docker Compose验证器能做什么？

Docker Compose验证器是一款专为DevOps工程师和开发者设计的在线工具，能实时验证docker-compose.yml文件的语法正确性和配置合理性。输入YAML内容，工具即刻检查服务定义、端口映射、卷挂载、网络配置等，并精确定位错误行号，帮助快速排查配置问题。

核心功能

  • YAML语法检查：验证YAML缩进、格式、引号等基础语法问题
  • 服务配置验证：检查每个服务是否包含image或build、端口映射格式、卷挂载格式
  • 网络和卷验证：检查网络和卷的引用是否已定义
  • depends_on检查：验证服务依赖关系是否引用了已定义的服务
  • 错误行号定位：每个问题都标注具体行号，方便快速定位修复

使用教程

步骤1：在输入区域粘贴docker-compose.yml内容，或点击示例按钮加载。

步骤2：点击"验证"按钮或使用快捷键Ctrl+Enter执行验证。

步骤3：查看验证结果，包括错误数、警告数和通过项。

步骤4：根据错误提示修改对应行，重新验证直到通过。

扩展知识

Docker Compose是Docker官方的多容器编排工具，使用YAML文件定义服务、网络和卷。V3版本是当前推荐格式，支持deploy配置用于Swarm模式。常用顶级键：services（服务定义）、networks（网络定义）、volumes（卷定义）、configs（配置文件）、secrets（敏感数据）。端口映射格式为HOST:CONTAINER，卷挂载格式为HOST_PATH:CONTAINER_PATH:MODE。
