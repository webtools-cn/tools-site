📖 什么是 Conventional Commits？

Conventional Commits 是一种轻量级的提交信息规范，格式为：<type>(<scope>): <description>。

它提供了易于编写的历史记录集，有助于自动化生成CHANGELOG、自动语义版本号提升，以及更好地与团队协作。

推荐的提交类型

feat - 新功能 | fix - Bug修复 | docs - 文档变更 | style - 代码格式 | refactor - 代码重构 | perf - 性能优化 | test - 测试 | build - 构建系统 | ci - CI配置 | chore - 杂项 | revert - 回滚

示例

feat(api): 添加用户注册接口

fix(auth): 修复Token刷新时验证失败问题

feat!: 重构数据库架构 (BREAKING CHANGE)

docs(readme): 更新安装说明
