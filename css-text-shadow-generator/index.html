扩展知识

text-shadow 语法详解：text-shadow: offset-x offset-y blur-radius color; 各参数的含义是——offset-x：水平偏移，正值向右；offset-y：垂直偏移，正值向下；blur-radius：模糊半径，值越大阴影越弥散；color：阴影颜色，支持各种CSS颜色格式。多个阴影层用逗号分隔，后写的层会覆盖在先写的层之上。

与 box-shadow 的区别：text-shadow 仅作用于文字轮廓，不影响元素的盒模型；box-shadow 作用于整个元素矩形区域。text-shadow 不支持 inset 关键字，也不支持 spread-radius 扩展半径。在性能方面，text-shadow 通常比 box-shadow 更轻量，因为它只涉及文字轮廓的重绘。

浏览器兼容性：text-shadow 属性在现代浏览器中均有良好支持，包括Chrome、Firefox、Safari、Edge。IE9及以下版本不支持，IE10和IE11支持但不支持多重阴影。在需要兼容旧版IE的场景中，可以使用滤镜（filter）作为降级方案。对于文字描边效果，可以使用 -webkit-text-stroke 配合 text-shadow 实现更精细的轮廓控制。

常见问题 FAQ

text-shadow 支持多少层阴影？

CSS text-shadow 属性支持任意数量的阴影层，各层之间用逗号分隔。本工具最多支持10层阴影叠加，足以实现霓虹灯、3D文字、火焰等复杂效果。超过10层通常没有必要，且可能对渲染性能产生负面影响。

文字阴影会影响页面性能吗？

一般情况下影响很小。现代浏览器对 box-shadow 和 text-shadow 都有硬件加速优化。但如果阴影层数过多（超过20层）或在大段文字上应用复杂阴影，可能会引起重绘开销。建议对标题等少量文字使用复杂阴影效果，对正文保持简洁或不用阴影。

如何实现内阴影效果？

标准CSS不支持文字的内阴影（inset text-shadow），但可以通过其他技巧模拟：如使用背景裁剪（background-clip: text）配合text-stroke，或者使用混合模式（mix-blend-mode）。本工具提供了"浮雕"预设，通过白色上阴影和深色下阴影的组合，可以模拟出内凹或外凸的视觉效果。

文字阴影颜色可以用渐变吗？

标准CSS text-shadow 不支持渐变颜色。但可以通过多层纯色阴影的精细叠加来模拟渐变效果——例如火焰效果就是通过红色、橙色、黄色三层阴影实现的。另一种方案是使用 background-clip: text 配合渐变背景，但这需要额外的HTML结构支持。

模糊半径为0时阴影是什么样的？

当 blur-radius 设为0时，阴影不会有任何模糊，产生的是一个与文字形状完全相同的硬边副本，看起来就像是文字的精确复制偏移了一定距离。利用这个特性可以创建文字描边、双重文字等效果。本工具的"描边"预设就是基于0模糊半径的多层偏移实现的。

文字阴影和文字描边有什么区别？

text-shadow 是通过偏移文字的副本来创造"阴影"或"描边"效果，而 text-stroke 是直接给文字笔画添加边框。text-shadow 的描边效果不够精确（在文字拐角处会有缺口），而 text-stroke 的边缘更清晰。但 text-stroke 不支持模糊，且兼容性略差（需要 -webkit- 前缀）。两种方法可以结合使用以获得最佳效果。

生成的代码可以直接用于生产环境吗？

可以。本工具生成的CSS代码是标准CSS，无需任何预处理器或框架即可直接使用。代码中已经包含了浏览器前缀（如果需要的话）。建议在将复杂效果用于生产环境前，在目标用户的浏览器环境中进行测试，确保渲染效果一致。

移动端浏览器支持文字阴影吗？

完全支持。iOS Safari、Chrome for Android、微信内置浏览器等主流移动端浏览器都完整支持 text-shadow 属性，包括多重阴影。移动端由于屏幕像素密度更高（Retina屏），同样的阴影参数看起来会比PC端更细腻。建议在移动端适当减小模糊半径以获得更锐利的视觉效果。
