❓ 常见问题

支持哪些JSON结构？

支持对象、数组、嵌套对象、基本类型，自动推导类型。

生成的接口可以直接用吗？

可以。代码符合TypeScript标准语法，直接粘贴到.ts文件。

如何处理null值？

null值标记为对应类型|null联合类型，如string|null。

支持数组类型推导吗？

支持。数组根据元素推导为Type[]，混合类型为联合类型数组。

数据会上传到服务器吗？

不会。所有类型推导和代码生成在浏览器本地完成。

interface和type有什么区别？

interface可声明合并，type支持联合/交叉类型。本工具默认生成interface。

JSON转TypeScript接口能做什么？

JSON转TypeScript接口工具是一款免费在线开发者工具，自动将JSON数据转换为TypeScript interface或type定义。支持嵌套对象、数组、基本类型推导，自动生成多个接口定义。可设置export、readonly等选项。适用于API响应类型定义、数据模型生成、TypeScript项目初始化等场景。所有处理在浏览器本地完成。

核心功能

  • 自动类型推导：string/number/boolean/null自动识别
  • 嵌套对象：自动拆分为独立接口定义
  • 数组类型：推导元素类型，支持联合类型数组
  • 可配置：export/readonly/根接口名可调
  • 完整代码：生成可直接使用的TypeScript代码

扩展知识

TypeScript接口（interface）定义对象的形状，提供编译时类型检查。从JSON推导类型时，null值生成联合类型（如string|null），数组根据元素推导（如number[]），嵌套对象自动拆分为独立接口。type和interface的区别：interface可声明合并，type支持联合/交叉类型。实际项目中建议结合jsdoc添加属性说明，使用工具如json-schema-to-typescript处理更复杂的场景。
