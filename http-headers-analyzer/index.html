HTTP请求头分析器

HTTP请求头和响应头是Web安全的重要组成部分。本工具帮助你分析HTTP头字段，检测安全配置缺失、CORS策略问题和缓存配置。所有分析在浏览器本地完成，数据不会上传。

重要安全头

  • Content-Security-Policy (CSP)：防止XSS攻击
  • Strict-Transport-Security (HSTS)：强制HTTPS
  • X-Content-Type-Options：防止MIME嗅探
  • X-Frame-Options：防止点击劫持
  • Referrer-Policy：控制引用信息泄露
  • Permissions-Policy：控制浏览器功能权限
什么是HTTP请求头？
HTTP请求头是HTTP请求和响应中的元数据字段，用于传递关于请求或响应的附加信息，如内容类型、缓存策略、安全策略等。
哪些HTTP头与安全相关？
重要的安全头包括：Content-Security-Policy（CSP）、Strict-Transport-Security（HSTS）、X-Content-Type-Options、X-Frame-Options、X-XSS-Protection、Referrer-Policy等。
如何检测CORS配置？
检查响应头中的Access-Control-Allow-Origin、Access-Control-Allow-Methods、Access-Control-Allow-Headers等字段。本工具会自动分析这些字段的安全性。
数据会上传到服务器吗？
不会。所有分析都在浏览器本地完成。
