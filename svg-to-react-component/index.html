SVG转React组件能做什么？

将SVG代码自动转换为React函数组件，处理所有属性名转换（class→className等），生成Props接口，支持TypeScript类型注解，让SVG图标在React项目中优雅使用。

核心功能

• 属性自动转换：class→className，kebab-case→camelCase

• TypeScript支持：生成带SVGProps类型注解的组件

• Props生成：width/height/fill/stroke等可配置属性

• 拖拽上传：支持拖拽SVG文件自动读取

• 一键下载：直接下载.tsx/.jsx文件

使用教程

步骤1：粘贴SVG代码到输入区域，或拖拽SVG文件。点击「加载示例」可快速体验。

步骤2：设置组件名称（如LogoIcon）、语言模式（TSX/JSX）、默认尺寸。

步骤3：点击「转换为React组件」，复制代码到项目中使用。组件支持Props动态控制。

扩展知识

SVG属性映射：React的JSX不直接支持HTML属性名，需要转换。常见映射：class→className，for→htmlFor，tabindex→tabIndex，stroke-width→strokeWidth，fill-rule→fillRule，clip-path→clipPath，xlink:href→xlinkHref。本工具自动处理所有这些转换。

SVGR工具链：本工具的转换逻辑参考了SVGR（最流行的SVG转React工具），但完全在浏览器本地运行，无需安装任何依赖。对于大型项目，建议使用SVGR的Webpack/Vite插件实现构建时自动转换。

为什么需要将SVG转为React组件？

将SVG转为React组件可以直接在JSX中使用，支持Props动态控制颜色/尺寸，享受TypeScript类型检查，避免内联SVG的代码冗长问题，且组件可复用、可测试。

SVG属性如何转换为JSX属性？

工具自动将SVG的HTML属性转为JSX兼容格式：class→className，stroke-width→strokeWidth，fill-rule→fillRule，xlink:href→xlinkHref等。所有kebab-case属性自动转为camelCase。

支持TypeScript吗？

支持。选择TypeScript模式后，组件会生成带SVGProps类型注解的Props接口，包含width、height、className等常用属性，并继承SVG元素的原始属性类型。

如何自定义SVG的尺寸和颜色？

生成的组件通过Props接收width、height、fill、stroke等属性，使用时传入即可。如<Icon width={24} height={24} fill="red" />。默认值可在组件中设置。

支持拖拽上传SVG文件吗？

支持。将SVG文件拖拽到输入区域即可自动读取内容。也可以直接粘贴SVG代码或点击「加载示例」体验。

转换后的组件可以直接在项目中使用吗？

可以。生成的组件代码是标准的React函数组件，复制到项目中即可使用。建议根据项目规范调整组件名称和默认Props值。
