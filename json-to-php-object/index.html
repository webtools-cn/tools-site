JSON转PHP类工具能做什么？

JSON转PHP类工具是一款免费在线开发者工具，能够将JSON数据一键转换为PHP类定义代码。自动识别JSON字段类型并映射为PHP类型，支持嵌套对象生成独立类、PHP 8命名参数构造函数、类型属性声明、getter/setter方法生成等功能。适用于Laravel/Symfony开发者快速从API响应生成模型类，大幅减少手动编写PHP类的工作量。所有转换在浏览器本地完成，数据绝不上传服务器。

核心功能

  • 智能类型映射：自动将string→string、number→int/float、boolean→bool、array→array、null→可空类型
  • 嵌套类：JSON中的嵌套对象自动生成为独立的PHP类定义
  • 构造函数：支持PHP 8命名参数构造函数或PHP 7传统构造函数
  • Getter/Setter：可选生成getter和setter方法，方便业务逻辑扩展
  • 多版本支持：支持PHP 8.x、7.4+、7.x三种版本语法

使用教程

1. 在JSON输入框中粘贴您的JSON数据，或点击"加载示例"快速体验。

2. 设置类名和转换选项：PHP版本、构造函数、getter/setter等。

3. 点击"转换"按钮，或使用快捷键Ctrl+Enter执行转换。

4. 在下方结果区域查看生成的PHP类代码。

5. 点击"复制代码"将结果复制到剪贴板，或下载为.php文件。

应用场景

场景1：Laravel模型

Laravel开发者对接API时，将JSON响应粘贴到工具中，一键生成Eloquent模型类或DTO类。

场景2：Symfony实体

Symfony开发者从JSON Schema快速生成实体类，配合ORM注解加速开发。

场景3：API客户端

PHP后端开发者调用第三方API时，从响应JSON生成类型安全的响应类。

场景4：配置映射

将JSON配置文件映射为PHP类，实现类型安全的配置访问。

扩展知识

PHP 7.4引入了类型属性（typed properties），允许在类属性声明时指定类型。PHP 8.0进一步增强了类型系统，支持联合类型（union types）如int|string，以及命名参数（named arguments）让构造函数调用更清晰。PHP 8.1引入了枚举（enum），PHP 8.2支持只读类（readonly class）。在Laravel中，Eloquent模型通常配合$casts属性实现JSON字段类型转换。DTO（Data Transfer Object）模式在PHP中越来越流行，用于在层间传递类型安全的数据。

❓ 常见问题

支持嵌套JSON吗？

支持。嵌套对象会自动生成为独立的PHP类定义。

如何处理null值？

开启"可空类型"后，null字段生成为可空类型（如?string），关闭则生成混合类型。

支持哪些PHP版本？

支持PHP 8.x（命名参数+类型属性）、PHP 7.4+（类型属性）、PHP 7.x（传统写法）。

数据会上传到服务器吗？

不会。所有转换在浏览器本地完成，数据绝不离开您的设备。

可以生成getter/setter吗？

可以。勾选"Getter/Setter"选项即可生成对应的getter和setter方法。

生成的代码可以直接使用吗？

可以。生成的代码符合PSR规范，可直接复制到.php文件中使用。
