常见问题


什么是Diff补丁文件？
Diff补丁文件（.patch或.diff）是记录两个文件之间差异的标准格式文件。它使用Unified Diff格式，包含添加行（+）、删除行（-）和上下文行的信息。通过patch命令可以将补丁应用到原始文件，实现自动化的代码修改。


Unified Diff格式是什么？
Unified Diff是最常用的差异格式，由@@行号@@标记每个差异块的位置，+表示新增行，-表示删除行，空格开头表示上下文行。Git diff默认输出就是Unified Diff格式。本工具生成的补丁文件完全兼容git apply和patch命令。


如何应用生成的补丁文件？
下载.patch文件后，在终端运行：patch -p1 < file.patch（标准patch命令）或 git apply file.patch（Git仓库中）。-p1参数表示去掉补丁路径中的第一层目录。建议先在测试环境验证补丁效果再应用到生产环境。


支持哪些对比模式？
本工具支持逐行对比模式，将文本按行分割后逐行比较，适合代码和配置文件。上下文行数可配置（0-10行），默认3行与git diff标准一致。


数据安全性如何保证？
所有文本对比和补丁生成都在你的浏览器本地完成，不会将任何文本内容上传到服务器。工具使用纯JavaScript实现diff算法，支持离线使用，完全保护你的代码和数据安全。

扩展知识

    
Unified Diff格式由GNU diff工具首创，后来被Git等版本控制系统广泛采用。它的核心思想是用最少的上下文信息精确描述文件的变更，使得补丁文件体积小、可读性强。标准格式包括文件头（---和+++）、差异块标记（@@行号@@）和变更内容（+/-/空格开头）。Git的diff输出完全兼容Unified Diff标准，因此本工具生成的补丁可以直接用git apply应用。
