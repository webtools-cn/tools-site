扩展知识

字符编码简史：在计算机发展初期，美国制定了ASCII（American Standard Code for Information Interchange）标准，使用7位二进制数表示128个字符。随着计算机全球化，各语言需要各自的编码方案，于是出现了GB2312/GBK（中文）、Shift-JIS（日文）、EUC-KR（韩文）等。但这些方案互不兼容，导致「乱码」问题频发。Unicode应运而生，它为每个字符分配了唯一的码点，解决了编码混乱问题。

UTF-8为何成为主流：UTF-8是Unicode的一种实现方式（编码格式），由Ken Thompson和Rob Pike在Plan 9操作系统中设计。它的设计目标是兼容ASCII（英文字母和数字只占用1字节），同时支持全球所有字符。对于纯英文文本，UTF-8与ASCII占用空间相同；对于中文，UTF-8每个字符占用3字节。这种可变长度设计使得UTF-8在兼容性和空间效率之间取得了完美平衡，因此成为Web（HTML5默认编码）、Linux和JSON的事实标准。

JavaScript与编码：JavaScript内部使用UTF-16编码字符串，这意味着一个字符可能占用1个或2个代码单元（Code Unit）。对于BMP平面（U+0000到U+FFFF）内的字符，占用1个代码单元；对于辅助平面字符（如Emoji和生僻汉字），需要2个代码单元（代理对Surrogate Pair）。了解这一点对于正确处理字符串长度和索引非常重要。ES6引入了codePointAt和String.fromCodePoint方法，可以正确处理所有Unicode字符。

常见问题 FAQ

什么是Unicode码点？

Unicode码点是每个字符在Unicode标准中的唯一编号，通常表示为U+XXXX（十六进制）。例如字母A的码点是U+0041，中文字「中」的码点是U+4E2D。Unicode标准目前定义了超过140,000个字符，覆盖全球几乎所有书写系统。

UTF-8和UTF-16有什么区别？

UTF-8使用1-4个可变长度字节表示字符，兼容ASCII（英文字母和数字只占1字节），是Web和Linux系统的首选编码。UTF-16使用2或4字节（固定或变长），是Java、JavaScript内部和Windows系统的默认编码。UTF-8更节省空间，UTF-16处理某些语言更快。

HTML实体编码有什么用？

HTML实体编码用于在HTML文档中表示特殊字符，防止它们被浏览器误解析。例如<表示左尖括号，&表示&符号。某些字符如版权符号©可以直接用实体&copy;表示，提高代码可读性。在XSS防护中，对不可信输入进行HTML实体编码是标准做法。

JavaScript转义序列怎么写？

JavaScript支持多种转义方式：\\xXX表示Latin-1编码字符（如\\x41表示A），\\uXXXX表示Unicode BMP字符（如\\u4E2D表示中），\\u{X}表示任意Unicode字符（如\\u{1F600}表示😀）。ES6起支持大括号形式的Unicode转义，可以表示BMP之外的字符。

URL编码和百分号编码是什么关系？

URL编码就是百分号编码（Percent-Encoding），是同一种编码方式。它将非ASCII字符和特殊字符转换为%XX格式（XX为十六进制）。例如空格编码为%20或+，中文字「中」编码为%E4%B8%AD。RFC 3986规定了哪些字符需要编码，RFC 3987扩展了对国际字符的支持。
