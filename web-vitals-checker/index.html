⚠️ 检测提示

LCP多少算好？

LCP≤2.5秒为好（绿色），2.5-4秒需要改进（橙色），超过4秒为差（红色）。LCP衡量页面主要内容加载速度。

如何降低CLS？

为图片和视频设置明确的宽高、避免在视口上方动态插入内容、使用CSS contain属性、预分配广告位空间、避免使用非稳定字体。

INP和FID有什么区别？

FID只衡量首次交互延迟，INP衡量所有交互的响应延迟。INP于2024年3月替代FID成为Core Web Vitals指标，更全面反映交互体验。

检测结果会上传吗？

不会。所有检测在浏览器本地完成，只检测当前页面的性能指标，不会向任何服务器发送数据。

TTFB是什么？

TTFB（Time to First Byte）是浏览器从发起请求到收到服务器第一个字节的等待时间，反映服务器响应速度。TTFB≤800ms为好。

Web Vitals检测工具能做什么？

Web Vitals检测工具是一款纯前端在线工具，用于检测网页的Core Web Vitals核心性能指标。Core Web Vitals是Google定义的一组衡量用户体验质量的关键指标，直接影响搜索排名。工具检测LCP（最大内容绘制）、INP（交互到下次绘制）、CLS（累积布局偏移）三大核心指标，以及FID（首次输入延迟）和TTFB（首字节时间）两个补充指标，并根据结果提供针对性的优化建议。

核心功能

  • LCP检测：最大内容绘制时间，衡量页面主要内容加载速度，≤2.5s为好
  • INP检测：交互到下次绘制延迟，衡量页面交互响应性，≤200ms为好
  • CLS检测：累积布局偏移，衡量视觉稳定性，≤0.1为好
  • FID检测：首次输入延迟，衡量首次交互响应性，≤100ms为好
  • TTFB检测：首字节时间，衡量服务器响应速度，≤800ms为好
  • 优化建议：根据各指标结果提供针对性优化方案

扩展知识

Core Web Vitals由Google于2020年推出，2021年正式成为搜索排名因素。三大核心指标覆盖加载（LCP）、交互（INP）和视觉稳定性（CLS）三个维度。2024年3月，INP替代FID成为核心指标，因为INP衡量所有交互的响应性，而FID只衡量首次交互。优化LCP的关键是减少关键资源加载时间：优化图片（WebP/AVIF）、使用CDN、预加载关键资源、减少服务器响应时间。优化INP需要减少主线程阻塞：代码拆分、减少JavaScript执行时间、使用Web Worker。优化CLS需要确保布局稳定：设置图片/视频尺寸、避免动态插入内容、使用font-display: swap。
