Base64 编码知识

什么是 Base64？

Base64 是一种将二进制数据编码为 ASCII 字符串的编码方案。它将每 3 个字节的数据编码为 4 个 ASCII 字符，常用于在文本环境中传输二进制数据（如邮件附件、图片内嵌、API 传参等）。

标准 Base64 vs URL 安全 Base64

标准 Base64 使用 + 和 /，这两个字符在 URL 中有特殊含义。URL 安全模式将 + 替换为 -，/ 替换为 _，并去掉 = 填充。

常用场景

• JWT Token：JSON Web Token 的 Header 和 Payload 使用 Base64URL 编码

• Data URI：图片内嵌为 data:image/png;base64,...

• 邮件附件：MIME 协议使用 Base64 编码二进制附件

• API 传参：在 HTTP 参数中传递二进制数据

应用场景

Base64编码在Web开发和数据传输中应用广泛，以下是几个典型的使用场景：

Data URI内嵌图片：在网页开发中，将小图标或头像直接嵌入HTML的src属性中，可以减少HTTP请求数量、提升首屏加载速度。格式为data:image/png;base64,iVBORw0KGgo...。本工具可以帮助你快速将图片的Base64编码生成对应的Data URI。

JWT Token解析与调试：JSON Web Token由Header、Payload和Signature三部分组成，其中Header和Payload使用Base64URL编码。开发者在调试认证接口时，可以使用本工具解码JWT的Payload部分，查看其中包含的用户信息、过期时间等声明（Claims）。

邮件附件编码：SMTP/MIME协议要求邮件中的二进制附件（如PDF、图片、压缩包）必须通过Base64编码后才能传输。邮件客户端在发送前会自动完成编码，接收后再解码还原为原始文件。了解这一机制有助于排查邮件乱码或附件损坏问题。

扩展知识

Base64编码原理：Base64是一种将二进制数据映射为ASCII字符的编码方案。它将每3个字节（24位）的数据拆分为4组6位，每组6位对应一个0-63的索引值，再通过固定的64字符查找表（A-Z、a-z、0-9、+、/）转换为可打印字符。如果原始数据不是3的倍数，会在末尾添加1或2个=作为填充（Padding）。

Base64 vs 加密：很多人误以为Base64是加密算法，这是一个常见的误解。Base64是编码，不是加密——任何人都可以轻易解码，不提供任何机密性保护。如果你需要保护敏感数据，应使用AES、RSA等真正的加密算法，并配合HTTPS传输。

相关编码方案：除了标准Base64，还有Base32（使用32个字符，适合不区分大小写的场景）、Base58（去除易混淆字符，广泛用于比特币地址）、Base85（Ascii85，编码效率更高，用于PDF和Git二进制补丁）等变体。不同场景会选择不同的编码方案以平衡可读性、传输效率和兼容性。

Base64 是加密吗？

不是。Base64 是编码方案，不是加密方案。任何人都能解码，不提供任何安全性。如需安全传输，请配合 TLS/SSL 或真正的加密算法使用。

Base64 编码后数据会变大吗？

会。Base64 编码后数据大约增加 33%（3字节变4字符）。例如 300 字节的数据编码后约为 400 字符。

支持中文吗？

支持。工具使用 TextEncoder API 将文本转为 UTF-8 字节再编码，完整支持中文、Emoji 等多字节字符。

Base64编码的用途是什么？

根据 RFC 4648：Base64编码用于在需要文本传输的场合传递二进制数据，如电子邮件附件 (MIME)、在URL/HTML中嵌入图片数据 (Data URIs)、JSON中传输二进制字段等。Base64编码比原始二进制大约增加33%的体积。
