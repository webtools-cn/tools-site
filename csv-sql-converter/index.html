CSV转SQL工具能做什么？

CSV转SQL工具是一款纯前端在线工具，用于将CSV数据转换为SQL语句。支持生成CREATE TABLE建表语句（自动推断列类型）和INSERT插入语句，兼容MySQL、PostgreSQL、SQLite、SQL Server和标准SQL五种方言。支持多种CSV分隔符、单行/批量INSERT模式，以及拖拽文件导入。所有转换在浏览器本地完成，数据绝不上传服务器。

核心功能

  • 建表语句生成：自动分析CSV数据推断列类型（INT/BIGINT/FLOAT/DATE/VARCHAR等），生成CREATE TABLE语句
  • INSERT语句生成：支持单行INSERT和批量INSERT两种模式，正确转义特殊字符
  • 多数据库方言：兼容MySQL、PostgreSQL、SQLite、SQL Server和标准SQL语法差异
  • 多分隔符支持：支持逗号、制表符、分号、管道符四种分隔符
  • 文件拖拽：支持拖拽CSV文件到输入框，自动读取内容

使用教程

步骤1：设置表名、SQL方言、分隔符和INSERT模式。

步骤2：在文本框粘贴CSV数据（第一行为表头），或拖拽CSV文件。

步骤3：点击"转换SQL"或使用Ctrl+Enter，查看生成的SQL语句。

步骤4：点击"复制SQL"复制到剪贴板，或"下载SQL"保存为文件。

扩展知识

CSV（Comma-Separated Values）是最通用的数据交换格式，RFC 4180定义了其标准规范。SQL INSERT语句有两种形式：INSERT INTO table VALUES (...)省略列名（依赖列顺序）和INSERT INTO table (col1, col2) VALUES (...)指定列名（更安全）。批量INSERT将多行合并为一条语句，如INSERT INTO table VALUES (...), (...), (...)，减少网络往返和日志写入，性能比单行INSERT高数倍。不同数据库对批量INSERT的语法略有差异。

常见问题 (FAQ)

支持哪些SQL方言？

支持MySQL、PostgreSQL、SQLite、SQL Server和标准SQL五种方言，自动适配不同数据库的语法差异。

能自动推断列类型吗？

可以。工具自动分析CSV数据，推断每列的数据类型（INT、BIGINT、FLOAT、DATE、VARCHAR等），生成合适的CREATE TABLE语句。

数据会上传服务器吗？

不会。所有CSV解析和SQL生成均在浏览器本地完成，数据绝不上传服务器。

支持哪些CSV分隔符？

支持逗号、制表符、分号和管道符四种分隔符，也可自定义分隔符。

能处理包含引号和换行的CSV吗？

可以。支持RFC 4180标准的CSV解析，正确处理双引号包裹的字段、字段内引号转义和字段内换行。

能生成批量INSERT吗？

可以。支持单行INSERT和多行批量INSERT两种模式，批量模式更高效。
