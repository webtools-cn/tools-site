常见问题

Q1: 什么是JSON Patch？

JSON Patch是RFC 6902定义的标准格式，用于描述对JSON文档的修改操作。它通过一组原子操作（add、remove、replace、move、copy、test）来精确描述文档的变更，常用于REST API的部分更新（PATCH请求），避免传输完整文档。

Q2: JSON Pointer路径怎么写？

JSON Patch使用RFC 6901 JSON Pointer定位元素。路径以/开头：根对象用""表示，/name指向name属性，/skills/0指向数组第一个元素，/a/b指向嵌套属性。特殊字符：~用~0转义，/用~1转义。

Q3: test操作有什么用？

test操作用于验证指定路径的值是否等于给定值。如果值不匹配，整个补丁操作将失败回滚。这在条件更新场景中非常有用——确保在修改前值符合预期，避免并发修改导致数据不一致。
