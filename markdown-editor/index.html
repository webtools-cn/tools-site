使用教程

本 Markdown 编辑器采用双栏布局设计，左侧为编辑区，右侧为实时预览区，支持常用语法快捷插入和多种导出方式。以下是详细的使用指南：

第一步：输入 Markdown 内容。在左侧「Markdown 编辑」区域的文本框中直接输入或粘贴 Markdown 文本。编辑器支持标准 Markdown 语法，包括六级标题、粗体、斜体、删除线、行内代码、代码块、引用、有序/无序列表、任务列表、表格、链接、图片和分割线。如果你需要快速插入某种格式，可以使用工具栏上的快捷按钮。

第二步：使用工具栏快捷插入。编辑器顶部的工具栏提供了常用语法的快捷按钮。选中一段文字后点击「粗体」按钮，会自动在文字两侧添加 **；点击「链接」或「图片」按钮，会自动插入对应的 Markdown 语法模板。工具栏还提供了标题（H1-H3）、列表、引用、代码块、表格和分割线的快捷插入，大幅提升写作效率。

第三步：查看实时预览。右侧「实时预览」区域会根据左侧输入实时渲染 HTML 结果。预览区应用了适合阅读的排版样式，包括代码高亮背景、引用块左边框、表格斑马线等。如果你希望切换预览区的背景主题（深色/浅色），可以点击预览区底部的「切换预览背景」按钮。

第四步：导出与复制。写作完成后，点击「📋 复制HTML」可将当前渲染的 HTML 代码复制到剪贴板，方便粘贴到邮件、CMS 或其他平台。点击「💾 导出HTML文件」则会生成一个包含完整 <html> 结构的独立文件并下载到本地，可直接在浏览器中打开查看。

应用场景

技术文档与博客写作：Markdown 是 GitHub、GitLab、Notion、Jekyll 等众多平台的首选文档格式。使用本工具，你可以在一个无干扰的环境中专注写作，实时查看渲染效果，确保表格、代码块和列表等复杂元素格式正确，减少发布后的排版错误。

README 与项目文档编写：开源项目的 README 文件通常使用 Markdown 编写。本编辑器支持的工具栏快捷插入功能，可以帮助开发者快速构建包含标题层级、代码示例、功能列表和贡献指南的标准 README 结构。导出 HTML 功能也便于将文档转换为适合展示的最终格式。

笔记记录与内容草稿：对于需要跨平台记录笔记的用户，Markdown 是最佳的纯文本格式选择。本工具无需注册、无需安装，随时随地可用。你可以用它记录课堂笔记、会议记录、旅行日记或任何结构化内容，然后通过复制 HTML 或导出文件的方式分享或归档。

扩展知识

Markdown 与 CommonMark 标准：Markdown 最初由 John Gruber 于 2004 年发明，但原始规范留下了许多模糊之处。CommonMark 项目于 2014 年发布了标准化的 Markdown 规范，明确了各种边界情况的处理方式。本编辑器遵循 CommonMark 的基本语法规则，同时支持部分 GitHub Flavored Markdown（GFM）扩展，如表格和任务列表。

Markdown 与富文本编辑器的对比：富文本编辑器（如 Word、在线 WYSIWYG 编辑器）使用专有格式存储样式信息，跨平台兼容性差，且容易产生冗余标签。Markdown 使用纯文本和简单标记符号，具有以下优势：① 文件极小，便于版本控制（Git）；② 格式与内容分离，易于自动化处理；③ 几乎可在任何平台打开和编辑；④ 可轻松转换为 HTML、PDF、Word 等多种格式。

自定义渲染与扩展：本编辑器使用纯 JavaScript 实现 Markdown 解析，不依赖任何外部库。这意味着你可以离线使用，也可以在本地保存页面后继续使用。如果你对渲染样式有特殊需求，可以导出 HTML 后自行修改 CSS 样式表。对于需要更强大功能（如数学公式、流程图、目录生成）的场景，建议结合 Pandoc、Markdown-it 等工具链进行扩展。

这个Markdown编辑器支持哪些语法？

支持标准Markdown语法，包括标题、粗体、斜体、删除线、代码块、行内代码、引用、有序/无序列表、表格、链接、图片、分割线等。

数据会上传服务器吗？

不会。所有处理都在浏览器本地完成，Markdown解析是纯前端JavaScript实现的，数据绝不上传服务器。

可以导出为HTML文件吗？

可以。点击「导出HTML文件」按钮即可将当前Markdown内容渲染为完整HTML页面并下载到本地。
