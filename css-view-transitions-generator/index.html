常见问题

什么是View Transitions API？▼
View Transitions API是浏览器原生API，用于创建页面间的平滑过渡动画。无需JavaScript动画库，仅用CSS即可实现SPA和MPA的页面切换效果。
支持哪些浏览器？▼
Chrome 111+、Edge 111+原生支持。Safari和Firefox正在开发中。不支持时会优雅降级为直接切换，不影响功能。
SPA和MPA过渡有什么区别？▼
SPA过渡使用document.startViewTransition()在同页面内切换DOM，MPA过渡使用meta标签实现跨页面过渡。两者CSS配置相同，JavaScript调用方式不同。
view-transition-name有什么用？▼
view-transition-name为元素指定过渡名称，使相同名称的元素在不同页面间产生形态变换动画。例如列表页的图片和详情页的图片使用相同名称，切换时图片会平滑过渡。

扩展知识

View Transitions API是Chrome 111引入的原生浏览器API，它通过截图旧状态和新状态，在两者之间创建动画过渡。核心机制：浏览器截取旧状态快照→执行DOM更新→截取新状态快照→使用CSS动画在两组快照间过渡。关键伪元素：::view-transition（根容器）、::view-transition-group（过渡组）、::view-transition-old（旧快照）、::view-transition-new（新快照）。相比JavaScript动画库，View Transitions API性能更好、代码更少、兼容性降级更优雅。
