常见问题

什么是OKLCH颜色空间？

OKLCH是CSS Color 4引入的新色彩空间，基于OKLAB模型。L表示感知亮度(0-1)，C表示色度(饱和度)，H表示色相角度(0-360)。相比HSL，OKLCH的亮度变化更符合人眼感知，色度均匀分布。

OKLCH和HSL有什么区别？

OKLCH的L通道是感知均匀的，HSL的L不是；OKLCH的C(色度)在不同色相下视觉饱和度一致；OKLCH支持广色域(P3/Rec2020)，HSL仅限sRGB；OKLCH渐变更平滑自然。

哪些浏览器支持OKLCH？

Chrome 111+、Firefox 113+、Safari 15.4+、Edge 111+均已支持。2024年起主流浏览器均已支持CSS Color 4规范。

OKLCH如何表示广色域颜色？

OKLCH的色度(C)值超过约0.15时，颜色可能超出sRGB色域进入P3或Rec2020广色域。在支持广色域的显示器上颜色更鲜艳，不支持时浏览器自动映射到最接近的sRGB颜色。

如何在CSS中使用OKLCH？

使用oklch()函数：background: oklch(0.7 0.15 180)。加透明度：oklch(0.7 0.15 180 / 0.5)。推荐相对颜色语法：oklch(from var(--base) l c calc(h + 60))。

扩展知识

OKLCH基于Björn Ottosson在2020年提出的OKLAB色彩空间。OKLAB通过改进CIELAB的色相偏移问题，实现了更好的感知均匀性。OKLCH是OKLAB的极坐标形式，L=亮度(0-1)，C=色度(饱和度，0-0.4+)，H=色相(0-360°)。CSS Color 4规范在2022年被W3C正式推荐，所有主流浏览器从2023年起陆续支持oklch()函数。
