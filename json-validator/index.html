使用教程

本JSON校验工具操作简单且功能完善，以下是详细的使用指南：

粘贴JSON：在输入框中粘贴需要校验的JSON文本。工具支持任意合法的JSON格式，包括嵌套对象、数组、字符串、数字、布尔值和null。输入时下方会实时显示字符数和行数统计。

点击校验：点击「校验」按钮，工具会解析JSON并给出详细结果。如果JSON格式正确，会显示绿色的「✅ 有效JSON」状态，并在结果区域展示结构化的语法高亮视图。如果格式有误，会显示红色的错误信息，包括错误类型、所在行号和列号，帮助你快速定位问题。

格式化与压缩：「格式化」按钮会将JSON按层级缩进，方便阅读和分析嵌套结构。「压缩」按钮会移除所有空白字符，将JSON压缩为一行，适合用于网络传输和存储优化。格式化后的结果同样支持语法高亮显示。

加载示例：点击「加载示例」可快速填充一段标准JSON，方便新用户了解工具使用方法。示例包含了对象、数组、字符串、数字、布尔值等多种数据类型。

应用场景

JSON校验工具在Web开发和数据交换中应用广泛，以下是几个典型的使用场景：

API接口调试：开发RESTful API时，前后端通过JSON传输数据。当接口返回异常时，开发者需要将返回内容粘贴到校验工具中，快速确认是JSON格式问题还是业务逻辑问题。特别是遇到500错误时，服务端可能返回了非标准JSON（如HTML错误页面），此时校验工具可以立即识别。

配置文件检查：现代应用大量使用JSON作为配置文件格式（如package.json、tsconfig.json、manifest.json等）。在修改这些文件后，使用校验工具可以确保语法正确，避免因一个多余的逗号导致整个应用无法启动。

数据交换验证：在微服务架构中，服务之间通过JSON消息队列或HTTP请求交换数据。接收方在解析前先用校验工具验证JSON格式，可以提前发现数据质量问题，避免解析异常导致的服务故障。

扩展知识

JSON规范演进：JSON（JavaScript Object Notation）由Douglas Crockford于2001年提出，2006年被RFC 4627标准化，2017年更新为RFC 8259。它源于JavaScript的对象字面量语法，但已经成为独立于语言的数据格式，被几乎所有编程语言原生支持。

常见JSON错误：缺少逗号（如 {"a":1 "b":2}）是最常见的错误；单引号代替双引号（如 {'a':1}）在JavaScript对象中合法但在JSON中不合法；尾部逗号（如 [1,2,]）在ES2019后的JavaScript中合法但JSON不允许；注释（如 // comment）在JSON中完全不被支持，这也是JSON与JSONC/JSON5的主要区别。

JSON替代方案：对于需要注释的场景，可以使用JSON5或JSONC（JSON with Comments）；对于需要更严格类型系统的场景，可以使用Protobuf或Avro；对于配置文件，YAML因支持注释和更简洁的语法而广受欢迎，但YAML的缩进敏感性也导致它更容易出错。

常见问题 FAQ

JSON校验工具能检测哪些错误？

可以检测语法错误（如缺少逗号、括号不匹配）、非法字符、JSON格式违规（如单引号代替双引号、尾部逗号）、数据类型错误等，并精确定位到具体行号和列号。

JSON和JavaScript对象有什么区别？

JSON是数据交换格式，JavaScript对象是编程语言中的数据结构。关键区别：JSON键必须加双引号；JSON不支持函数、undefined、注释；JSON字符串必须用双引号。

为什么JSON不能有尾随逗号？

JSON规范（RFC 8259）不允许对象或数组最后一个元素后有逗号。这是为了简化解析器实现和提高跨语言兼容性。JavaScript对象允许尾随逗号，但JSON不行。

校验结果中的语法高亮是什么颜色？

键名显示为蓝色（key），字符串显示为绿色（"value"），数字显示为黄色（123），布尔值显示为粉色（true），null显示为灰色（null）。

工具支持超大JSON文件吗？

工具在浏览器端运行，建议处理不超过10MB的JSON文件。对于超大文件，建议使用命令行工具如 jq 或专门的JSON处理软件。

数据会上传到服务器吗？

不会。所有处理都在浏览器本地完成，数据绝不上传服务器，保障你的数据隐私安全。

为什么格式化后的JSON有缩进？

格式化功能使用2个空格作为缩进单位，这是业界广泛认可的规范（如Prettier默认配置）。缩进层级与JSON的嵌套深度一致，便于阅读和调试。

可以校验JSON Schema吗？

当前版本仅校验JSON语法正确性，暂不支持JSON Schema验证。如需Schema验证，可使用JSON Schema Validator等专门工具。

JSON验证检查哪些规则？

JSON验证依据 RFC 8259 标准检查：字符串必须使用双引号、不能有尾随逗号、键名必须为字符串、数值不能有前导零、Unicode字符必须正确转义等。
