常见问题

什么是API状态监控？

API状态监控是定期向API端点发送请求，检测其是否可用、响应时间多少、返回什么状态码的过程。通过持续监控可以及时发现API故障、性能下降等问题，确保服务稳定运行。

这个工具如何检测API状态？

本工具使用浏览器的fetch API向目标端点发送HEAD或GET请求，测量响应时间和状态码。所有请求从你的浏览器发出，不经过任何中间服务器。注意：某些API可能因CORS策略限制无法从浏览器直接检测。

CORS限制会影响检测结果吗？

会。如果目标API没有设置允许跨域访问的CORS头，浏览器会阻止请求。对于这种情况，工具会显示CORS错误。建议监控自己控制的API（已配置CORS），或使用公共API（通常已开放CORS）。

支持哪些监控功能？

支持：1) 添加多个API端点；2) 手动或定时检测（30秒/1分钟/5分钟）；3) 记录响应时间和状态码历史；4) 可用性百分比统计；5) 响应时间图表；6) 状态变化通知。

数据会上传服务器吗？

不会。所有监控数据存储在浏览器localStorage中，不会上传到任何服务器。API检测请求直接从你的浏览器发送到目标API，不经过中间服务器。
