📖 使用教程

本Markdown转HTML转换器操作简单且功能完善，以下是详细的使用指南：

实时转换：在左侧输入框中输入或粘贴Markdown文本，右侧预览区域会实时显示转换后的HTML渲染效果。无需任何额外操作，输入即转换，让您即刻看到排版结果。工具支持标准的Markdown语法，包括各级标题、粗体、斜体、删除线、代码块、引用、列表、表格等，以及GitHub Flavored Markdown的扩展特性如任务列表。

导出功能：完成编辑后，点击「下载HTML」按钮，工具会生成一个包含完整HTML结构的文件（包括DOCTYPE声明、head和body标签），并附带美观的基础CSS样式，直接在浏览器中打开即可看到格式良好的内容。如果只需要HTML片段（不含完整HTML文档结构），可以点击「仅导出Body」按钮获取纯HTML内容。

复制功能：点击「复制HTML」按钮，即可将生成的完整HTML代码复制到剪贴板，方便您粘贴到博客系统、CMS后台、邮件编辑器或其他需要HTML代码的地方。所有操作均在浏览器本地完成，您的数据不会上传到任何服务器。

快捷键：支持常用的编辑快捷键，包括Ctrl+A全选、Ctrl+C复制、Ctrl+V粘贴等，提升您的编辑效率。点击「加载示例」按钮可以快速查看一个包含多种Markdown语法的示例文档。

🔍 扩展知识

Markdown与HTML的关系：Markdown是由John Gruber于2004年创建的一种轻量级标记语言，设计目标是让人们用易读易写的纯文本格式编写文档，然后转换为结构化的HTML。Markdown本身不是HTML，而是一种简写语法，最终被解析器转换为HTML。这意味着所有合法的Markdown最终都会被转换为等效的HTML标签，例如## 标题会被转换为<h2>标题</h2>。

GitHub Flavored Markdown (GFM)：GFM是GitHub在标准Markdown基础上扩展的一套语法规范，增加了表格、任务列表、删除线、自动链接、代码块语法高亮等实用功能。本工具完整支持GFM语法，包括使用三个反引号包裹代码块并指定语言（如python）、使用方括号创建任务列表（- [ ]和- [x]）等。

相关工具推荐：如果您需要将HTML转换回Markdown，可以使用本站的「HTML转Markdown」工具。此外，「Markdown编辑器」提供了更丰富的Markdown编辑和预览体验，「Markdown表格生成器」可以帮助您快速创建结构化的Markdown表格。这些工具配合使用，可以覆盖Markdown处理的完整工作流。

❓ 常见问题

Markdown转HTML后样式会保留吗？

转换后的HTML包含标准的语义化标签（如h1、p、table等），但本身不包含CSS样式。您可以自行添加CSS样式表来美化显示效果，或者使用本工具的导出功能生成带基础样式的完整HTML文件。

支持哪些Markdown语法？

支持标准Markdown语法及GitHub Flavored Markdown扩展，包括：标题、粗体、斜体、删除线、代码行内、代码块、引用块、无序/有序列表、任务列表、链接、图片、表格、水平线等。

转换后的数据会上传服务器吗？

不会。所有转换操作均在浏览器本地完成，您的Markdown内容不会上传到任何服务器，确保数据安全和隐私。

导出的HTML文件可以直接在浏览器中打开吗？

可以。导出功能会生成一个包含完整HTML结构（DOCTYPE、head、body）的文件，并附带基础CSS样式，直接用浏览器打开即可看到格式良好的内容。

支持数学公式渲染吗？

当前版本不支持LaTeX数学公式渲染。如果您的文档包含数学公式，建议先使用其他工具（如MathJax或KaTeX）进行渲染，再将结果HTML粘贴使用。

可以处理大量文本吗？

可以。由于所有处理在浏览器本地完成，理论上没有文本大小限制（受浏览器内存限制）。对于超大文档（超过10MB），建议分批处理以提升响应速度。

导出的HTML代码包含哪些内容？

导出功能提供两种模式：「下载HTML」会生成包含完整HTML文档结构的文件（含DOCTYPE、html、head、body标签及基础CSS样式）；「仅导出Body」则只输出body标签内的HTML内容，方便嵌入到其他网页模板中。

如何批量处理多个Markdown文件？

当前版本为单文件处理模式。如需批量转换，建议将多个Markdown文件的内容分别粘贴到工具中，逐份导出HTML。对于大量文件的批量处理需求，建议使用命令行工具如Pandoc。
