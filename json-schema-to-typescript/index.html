JSON Schema转TypeScript能做什么？

JSON Schema转TypeScript是一款免费在线工具，能够将JSON Schema定义自动转换为TypeScript接口（interface）和类型别名（type）。无论你是API开发者、前端工程师还是全栈开发者，只需粘贴JSON Schema，即可获得符合TypeScript规范的类型定义代码。工具采用纯前端技术，所有转换在浏览器本地完成，数据绝不上传服务器。

核心功能

  • 自动类型推断：根据JSON Schema的type、properties、items等字段，自动生成对应的TypeScript接口和类型别名
  • 嵌套对象支持：递归处理嵌套的对象和数组结构，生成完整的类型层级
  • 枚举与联合类型：自动识别enum、oneOf、anyOf、allOf等约束，生成对应的TypeScript联合类型和枚举
  • $ref引用解析：自动解析$ref引用，将引用的定义提取为独立类型，保持类型引用的完整性
  • 灵活配置：支持export修饰符、JSDoc注释、readonly属性、可选属性标记等生成选项
  • 多格式导出：支持一键复制到剪贴板或下载为.d.ts类型定义文件
  • 历史记录：自动保存最近5次转换记录，方便回顾和对比

使用教程

1. 在输入框中粘贴或输入JSON Schema定义，也可以拖拽JSON文件导入。

2. 点击"常用预设"按钮可快速加载典型JSON Schema示例。

3. 根据需要调整生成选项：是否添加export、是否生成JSDoc注释、根接口名称等。

4. 点击"转换"按钮，工具会实时生成TypeScript类型定义代码。

5. 点击"复制代码"将结果复制到剪贴板，或点击"下载.d.ts"保存为文件。

扩展知识

JSON Schema是一种用于描述和验证JSON文档结构的声明式语言，当前最新版本为Draft 2020-12。它定义了type、properties、items、required、enum、$ref等关键字来约束JSON数据的形状。TypeScript的类型系统与JSON Schema有天然的对应关系：object对应interface，array对应Array<T>，enum对应联合类型，$ref对应类型引用。常见的转换挑战包括：oneOf/anyOf的联合类型映射、additionalProperties的索引签名处理、patternProperties的动态键名、以及递归$ref的循环引用检测。业界主流的转换库如json-schema-to-typescript采用预编译方式处理复杂场景，而本工具采用纯运行时解析，无需安装任何依赖即可使用。

❓ 常见问题

这个工具免费吗？

完全免费，无需注册，无需下载。所有功能在浏览器本地运行，数据不会上传到任何服务器。

数据会经过服务器吗？

不会。本工具是纯前端应用，所有转换操作都在您的浏览器中本地完成，数据绝不离开您的设备。

支持哪些JSON Schema版本？

支持Draft 04、Draft 06、Draft 07、Draft 2019-09和Draft 2020-12等主流版本的核心关键字，包括type、properties、items、required、enum、$ref、oneOf、anyOf、allOf等。

如何处理循环引用？

工具会检测$ref的循环引用，遇到循环时使用类型别名（type）而非接口（interface）来定义，确保生成的TypeScript代码不会产生无限递归。

转换结果可以直接用于生产项目吗？

可以。生成的TypeScript代码符合标准语法，可以直接复制到项目的.d.ts或.ts文件中使用。建议先检查生成结果，确保类型定义符合项目需求。

支持拖拽文件导入吗？

支持。可以将.json或.schema文件直接拖拽到输入区域，工具会自动读取文件内容并填入输入框。
