常见问题

Q1: JSON Merge Patch和JSON Patch有什么区别？

JSON Merge Patch（RFC 7396）是声明式的——直接描述目标文档的最终状态，null表示删除字段。JSON Patch（RFC 6902）是操作式的——描述一系列操作步骤（add/remove/replace等）。Merge Patch更简洁直观，但不支持数组元素级别的精确操作。

Q2: 如何删除字段？

在Merge Patch文档中将目标字段设为null即可删除。例如 {"city": null} 会删除原始文档中的city字段。注意：如果原始文档中该字段不存在，null不会产生任何效果。

Q3: 数组怎么处理？

Merge Patch对数组的处理是整体替换，不支持部分修改。如果patch中包含数组，它会完全替换原始文档中的对应数组。如需精确修改数组元素，建议使用JSON Patch（RFC 6902）。

常见问题 (FAQ)

JSON Merge Patch和JSON Patch有什么区别？

Merge Patch是声明式（描述最终状态），JSON Patch是操作式（描述操作步骤）。Merge Patch更简洁但不支持数组元素级操作。

如何删除字段？

在patch中将字段设为null即可删除，如 {"city": null}。

数组怎么处理？

数组整体替换，不支持部分修改。需要精确修改数组元素请用JSON Patch。
