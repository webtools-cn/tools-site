URL 编码知识

什么是 URL 编码？

URL 编码（又称百分号编码，Percent-Encoding）是一种将 URL 中的特殊字符转换为 %XX 格式的机制，其中 XX 是字符 ASCII/UTF-8 字节的十六进制表示。URL 标准定义于 RFC 3986，规定 URL 中只能包含 ASCII 字母、数字及少数保留字符，其余字符必须进行编码。

什么时候需要 URL 编码？

  • URL 中包含中文或非 ASCII 字符：如 https://example.com/搜索/关键词 中的中文需要编码
  • 查询参数值包含特殊字符：参数值中的 & = ? 等字符会破坏 URL 结构
  • 构建 API 请求 URL：确保参数值被正确传输和解析
  • URL 重定向：在 Location 头或 redirect 参数中传递目标 URL
  • 锚点/片段标识符：锚点中包含特殊字符时需要编码
  • mailto / tel 链接：邮件主题、正文中的特殊字符需要编码

encodeURI vs encodeURIComponent 区别

JavaScript 提供两个 URL 编码函数，核心区别在于对保留字符的处理：

• encodeURI：编码整个 URL，保留 URL 结构字符 ; , / ? : @ & = + $ #，适用于编码完整的 URL 地址。

• encodeURIComponent：编码 URL 的某个部分（如参数值），编码所有保留字符，确保参数值中的特殊字符不会破坏 URL 结构。

简单记法：编码整个 URL 用 encodeURI，编码参数值用 encodeURIComponent。

常见使用场景

  • API 参数传递：?q=（搜索关键词编码后作为参数值）
  • URL 重定向：?redirect=（目标 URL 编码后作为参数值，需用 encodeURIComponent）
  • 锚点导航：#（锚点中包含中文时编码）
  • mailto 链接：mailto:?subject=（邮件主题编码）
  • 路径中的中文：/path/（路径段编码）

URL 编码后长度会增加多少？

每个被编码的字符会变为 3 个字符（如空格变为 %20）。中文字符在 UTF-8 下通常占 3 字节，编码后变为 9 个字符（如"中"变为 %E4%B8%AD）。因此含中文或特殊字符较多的 URL 编码后长度会显著增加。

encodeURI 和 encodeURIComponent 应该选哪个？

如果编码整个 URL（含协议、域名、路径），使用 encodeURI，它会保留 :/?#&= 等分隔符。如果编码 URL 中的某个参数值或片段，使用 encodeURIComponent，它会编码所有特殊字符，避免参数值中的特殊字符破坏 URL 结构。

URL 编码支持中文吗？

支持。JavaScript 的 encodeURI 和 encodeURIComponent 会自动将中文字符按 UTF-8 编码后转为百分号编码。例如"你好"会被编码为 %E4%BD%A0%E5%A5%BD。本工具完整支持中文、Emoji 及所有 Unicode 字符。

空格被编码成什么？%20 和 + 有什么区别？

encodeURI 和 encodeURIComponent 将空格编码为 %20。在 application/x-www-form-urlencoded 格式（如表单提交）中，空格会被编码为 +。本工具使用标准的 %20 编码。如需 + 号编码，可自行替换。

批量模式如何使用？

勾选"批量模式"后，工具会对输入框中的每一行分别进行编解码，每行一个 URL。空行会被保留。如果某行解码失败，会显示错误提示而不影响其他行。

URL 编码是加密吗？安全吗？

不是。URL 编码是编码方案，不是加密。任何人都可以解码，不提供任何安全性。它的目的是确保 URL 中的特殊字符能被正确传输和解析，而非保护数据安全。如需安全传输，请使用 HTTPS 或真正的加密算法。
