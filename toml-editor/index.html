什么是TOML

TOML（Tom's Obvious Minimal Language）是一种最小化的配置文件格式，由GitHub联合创始人Tom Preston-Werner于2013年创建。TOML的设计目标是易于阅读和编写，同时能无歧义地映射到哈希表。它比YAML更简单明确，没有缩进陷阱；比JSON更易读，支持注释。TOML已成为许多项目的首选配置格式，包括Rust的Cargo.toml、Python的pyproject.toml、Hugo的config.toml等。

TOML语法要点

  • 键值对：key = "value"，字符串必须加引号
  • 表（Table）：[table-name]，相当于JSON的对象
  • 内联表：name = {first = "Tom", last = "Preston-Werner"}
  • 数组表：[[products]]，相当于JSON的对象数组
  • 数组：colors = ["red", "yellow", "green"]
  • 字符串：基本("...")、多行("""...""")、字面量('...')、多行字面量('''...''')
  • 数字：整数(42)、浮点数(3.14)、正负无穷(inf)、非数(nan)
  • 布尔值：true 或 false
  • 日期时间：1979-05-27、07:32:00、1979-05-27T07:32:00Z
  • 注释：以 # 开头到行尾

使用教程

编辑TOML：在编辑器中输入或粘贴TOML内容。编辑器会实时显示行数。输入时注意TOML的语法规则：字符串值必须用引号包裹，表名用方括号包裹，键值对用等号连接。

验证TOML：点击「✅ 验证」按钮检查TOML语法是否正确。如果存在错误，状态栏会显示红色错误信息和行号。修正错误后再次验证。

TOML转JSON：点击「🔄 TOML→JSON」按钮将TOML内容转换为JSON格式。转换结果会显示在输出区域，方便查看TOML的数据结构或用于程序处理。

JSON转TOML：在编辑器中输入JSON内容，点击「🔄 JSON→TOML」按钮将JSON转换为TOML格式。注意JSON中的嵌套对象会转换为TOML的表。

常见问题

Q: 什么是TOML格式？

A: TOML（Tom's Obvious Minimal Language）是一种最小化的配置文件格式，由Tom Preston-Werner创建。设计目标是易于阅读和编写，同时能无歧义地映射到哈希表。广泛用于Rust项目的Cargo.toml、Python的pyproject.toml等配置文件中。

Q: TOML和YAML、JSON有什么区别？

A: TOML比YAML更简单明确，没有YAML的缩进陷阱和复杂特性；比JSON更易读，支持注释。TOML适合配置文件，JSON适合数据交换，YAML适合需要复杂结构的配置。TOML的类型系统更严格，解析更简单可靠。

Q: 支持哪些TOML特性？

A: 支持TOML v1.0规范的所有特性，包括：键值对、表（table）、内联表、数组表、数组、字符串（基本/多行/字面量）、整数、浮点数、布尔值、日期时间等。
