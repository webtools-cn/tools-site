常见问题

什么是 CHANGELOG？

CHANGELOG 是一个按时间倒序排列的版本变更记录文件。遵循 Keep a Changelog 规范，按 Added（新增）、Changed（变更）、Deprecated（弃用）、Removed（移除）、Fixed（修复）、Security（安全）分类记录每个版本的变更内容。

语义化版本（SemVer）是什么？

语义化版本（SemVer）格式为 MAJOR.MINOR.PATCH。MAJOR 是不兼容的 API 变更，MINOR 是向下兼容的功能新增，PATCH 是向下兼容的问题修复。CHANGELOG 应该与 SemVer 严格对应。

每次发布都要更新 CHANGELOG 吗？

是的。每次发布新版本都应该更新 CHANGELOG，包括版本号、发布日期、变更内容。建议在开发过程中就记录变更，而不是发布前一次性回忆，这样可以避免遗漏细节。
