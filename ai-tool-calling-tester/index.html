扩展知识

Function Calling（函数调用）是LLM与外部工具交互的核心机制。OpenAI于2023年6月推出Function Calling，Anthropic随后推出Tool Use。两者格式略有不同：OpenAI使用function和parameters字段，Anthropic使用input_schema。2024-2026年，MCP（Model Context Protocol）正在统一工具定义标准，使工具可以跨平台复用。

常见问题

什么是Function Calling？

Function Calling是LLM根据用户输入决定调用哪个函数、提取什么参数的能力。LLM不直接执行函数，而是返回函数名和参数，由应用层执行。

OpenAI和Anthropic格式有什么区别？

OpenAI使用function.parameters，Anthropic使用input_schema。参数结构基本相同，但包装方式不同。本工具可一键切换两种格式。

如何定义好的工具Schema？

好的Schema需要：清晰的工具名称和描述、明确的参数类型和描述、合理的required字段、使用enum限制可选值。

模拟调用的结果准确吗？

模拟调用展示的是LLM可能返回的格式，实际参数由LLM根据用户消息提取。模拟结果仅供参考格式验证。

数据会上传服务器吗？

不会。所有Schema生成和验证完全在浏览器本地完成。
