HTTP Mock服务器生成器能做什么？

HTTP Mock服务器生成器是一款纯前端在线工具，用于快速创建Mock API路由配置。通过可视化界面添加路由、设置HTTP方法、路径、状态码、延迟和响应体，工具自动生成json-server配置或MSW（Mock Service Worker）代码。支持RESTful风格路由，可模拟各种HTTP响应场景。所有处理在浏览器本地完成。

核心功能

  • 多HTTP方法：支持GET/POST/PUT/PATCH/DELETE
  • 状态码模拟：200/201/400/401/403/404/500等
  • 延迟模拟：0-5000ms可调，模拟网络延迟
  • 多格式导出：json-server配置、MSW Handler代码
  • 路由管理：添加/删除路由，实时预览

使用教程

步骤1：选择HTTP方法，输入路径和响应数据。

步骤2：设置状态码和延迟时间。

步骤3：点击"添加路由"，可添加多个路由。

步骤4：点击"导出配置"或"导出MSW"，复制或下载结果。

应用场景

前端开发

在后端API未就绪时，用Mock数据驱动前端开发，不阻塞进度。

接口测试

模拟各种异常响应（404/500/超时），测试前端错误处理逻辑。

演示Demo

用Mock数据构建产品演示，无需真实后端服务。

扩展知识

Mock Server是前端开发中常用的工具。json-server是一个轻量级Mock服务器，通过JSON文件定义API路由。MSW（Mock Service Worker）在浏览器Service Worker层拦截请求，无需启动服务器。两者各有优势：json-server适合独立Mock服务，MSW适合集成到前端项目中。本工具生成的配置可配合两者使用。

常见问题 (FAQ)

什么是HTTP Mock服务器？

HTTP Mock服务器是模拟真实API行为的工具，返回预设的响应数据，用于前端开发、接口测试等场景。

支持哪些HTTP方法？

支持GET、POST、PUT、PATCH、DELETE等常见HTTP方法。

能模拟延迟吗？

可以。支持设置响应延迟（0-5000ms），模拟网络慢速和服务器处理时间。

支持自定义状态码吗？

支持。可设置200/201/400/401/403/404/500等任意HTTP状态码。

数据会上传服务器吗？

不会。所有Mock配置和生成在浏览器本地完成，数据绝不上传服务器。

生成的Mock配置怎么用？

可导出为json-server配置或MSW Handler代码，配合相应工具使用。
