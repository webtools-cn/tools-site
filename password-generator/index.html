什么是强密码？

强密码是指具有足够熵值（随机性）的密码，能够有效抵御暴力破解和字典攻击。根据 NIST SP 800-63B 标准，现代密码安全不强制要求特殊字符，而是强调长度优先。研究表明，一个16位以上的随机密码，其安全性远超8位含特殊符号的密码。

我们的密码生成器采用浏览器内置的 Web Crypto API（crypto.getRandomValues），这是密码学级别的安全随机数生成器，而非普通的 Math.random()。所有密码均在你的设备本地生成，绝不会上传到任何服务器，你甚至可以在断网环境下使用本工具。

WiFi密码怎么设最安全？

WiFi密码需要满足两个条件：① 足够安全防止蹭网；② 方便在手机/电视上输入。我们的WiFi密码预设会自动生成仅含大写字母和数字的12位密码，排除了易混淆字符（如0和O），确保你在电视遥控器上也能准确输入。

  • 推荐使用 WPA3 加密协议
  • 密码长度建议 12-16 位
  • 定期更换（建议每3-6个月）
  • 避免使用生日、手机号等个人信息

银行密码为什么不能用特殊符号？

国内多数银行系统（如工行、建行、招行等）的网银/手机银行出于兼容性考虑，通常不支持特殊符号，只允许数字和字母组合。选择我们的银行网银预设，即可一键生成符合银行密码规则的12位安全密码。

不同平台的密码规则差异很大：苹果ID不允许冒号，微软要求至少3种字符组合，微信支持符号但网页版可能有兼容问题。使用场景预设功能，可以确保生成的密码能直接用于目标平台。

开发者：API Key 生成

API Key 需要高熵值且格式规范。我们的 API Key 预设 默认生成32位十六进制字符串（128位熵值），满足绝大多数 API 密钥的安全要求。你也可以切换为64位（256位熵值）用于高安全场景。

密码知识速查

  • 熵值计算：密码安全性的量化指标，计算公式为 长度 × log₂(字符集大小)
  • 8位密码：仅含小写字母时约37bits，可被普通电脑在几分钟内暴力破解
  • 16位密码：全字符集时约104bits，需要超级计算机运行数百年才能破解
  • 可读密码：如 Tiger-Blue-42-Wall，符合 NIST 最新建议，长且好记

常见问题 (FAQ)

这个密码生成器安全吗？会记录我的密码吗？

绝对安全。所有密码均在你的浏览器本地通过 Web Crypto API 生成，代码运行在纯前端（没有任何后端交互）。你可以按 F12 打开开发者工具查看网络请求，确认没有任何数据上传。本页面甚至可以下载到本地断网使用。

多长的密码才算安全？

根据 OWASP 和 NIST 建议：最低12位，推荐16位以上。8位密码（即使含特殊符号）在现代GPU暴力破解面前只需几分钟。我们的密码生成器会实时显示熵值，熵值 > 80 bits 即为极强安全等级。

为什么有些网站不支持特殊符号？

部分老旧系统（如银行网银、企业内部系统）的字符编码和输入处理存在限制。我们的场景预设功能可以针对特定平台生成合规密码。如果你遇到生成密码被拒的情况，请使用"排除特定符号"功能移除该平台不支持的字符。

生成的密码能不能保存？

我们不提供密码保存功能。出于安全考虑，我们不建议你依赖任何在线工具的"保存"功能。推荐使用专业的密码管理器（如 Bitwarden、1Password）或浏览器自带的密码管理器来安全存储密码。

批量生成的密码怎么用？

批量生成功能适合需要一次性创建多个密码的场景，例如：批量注册账号、为团队生成初始密码、创建大量API密钥等。每个密码都可以单独点击复制，也可以一次性导出为文本使用。

为什么密码要用等宽字体显示？

等宽字体（如 JetBrains Mono）中每个字符宽度一致，且对易混淆字符有特殊设计：数字 0 带有斜杠，小写 l 有底座，大写 I 有衬线。这大大降低了手动输入时的错误率，特别是通过手机或电视遥控器输入时。
