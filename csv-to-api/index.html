❓ 支持哪些API功能？

分页（page/pageSize）、字段筛选（fields）、条件过滤（field=value）、排序（sort/order）、单条查询（/id）。

CSV转REST API能做什么？

CSV转REST API是一款面向开发者的在线工具，能够将CSV数据快速转换为模拟REST API响应格式。上传或粘贴CSV后，自动解析表头和数据行，生成符合RESTful规范的JSON API响应，支持分页、字段筛选、条件过滤和排序，是前端开发和API原型设计的理想工具。

核心功能

  • 智能CSV解析：自动检测分隔符（逗号/Tab/分号），支持引号包裹字段和换行符，自动识别首行为表头
  • RESTful API格式：生成标准REST API响应，包含data数组、分页信息（total/page/pageSize/totalPages）
  • 分页/筛选/排序：支持?page=1&pageSize=10、?field=value过滤、?sort=name&order=asc排序
  • 多格式导出：导出JSON数据、OpenAPI 3.0规范文档、fetch/axios请求代码
  • 自定义配置：自定义资源名称、ID字段、分页大小，生成符合项目规范的API

使用教程

1. 拖拽CSV文件到上传区域，或直接在文本框粘贴CSV内容。工具自动检测分隔符并解析。

2. 配置API参数：设置资源名称（如users、products）、选择ID字段、调整每页条数。

3. 查看API端点预览，了解可用的查询参数和URL格式。响应预览区显示JSON格式的API返回数据。

4. 导出所需格式：下载JSON数据用于前端Mock，下载OpenAPI文档用于API设计，下载请求代码直接集成到项目。

扩展知识

REST（Representational State Transfer）是一种API设计风格，核心原则包括资源导向、统一接口、无状态通信。标准REST API使用HTTP方法（GET/POST/PUT/DELETE）操作资源，响应通常为JSON格式。分页是大型数据集API的关键特性，常见方案有偏移分页（page+pageSize）和游标分页（cursor）。OpenAPI（原名Swagger）是REST API的描述规范，可用于自动生成文档、客户端代码和测试用例。本工具生成的OpenAPI 3.0文档可直接导入Swagger UI或Postman使用。
