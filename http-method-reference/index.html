❓ 常见问题

GET和POST有什么区别？

GET获取资源，参数在URL中，可缓存；POST创建资源，参数在请求体中，更安全无长度限制。

PUT和PATCH有什么区别？

PUT是全量替换需发送完整资源；PATCH是部分更新只发修改字段。PUT幂等，PATCH不一定幂等。

什么是幂等性？

幂等性指同一请求执行一次和多次效果相同。GET、PUT、DELETE是幂等的，POST不一定是。

什么时候用DELETE？

DELETE用于删除资源。REST API中DELETE /users/123删除ID为123的用户，是幂等的。

OPTIONS方法有什么用？

OPTIONS获取资源支持的方法，浏览器跨域请求时自动发送OPTIONS预检请求。

HEAD和GET有什么区别？

HEAD与GET相同但只返回响应头不返回响应体，用于检查资源是否存在而不下载内容。

扩展知识

HTTP方法（也称HTTP动词）定义了对资源的操作语义。RFC 9110（HTTP Semantics）定义了核心方法：GET、POST、PUT、DELETE、HEAD、OPTIONS、TRACE、CONNECT。RFC 5789定义了PATCH方法。幂等性（Idempotent）意味着同一请求执行多次效果相同；安全性（Safe）意味着请求不会修改服务器状态。REST架构风格要求正确使用HTTP方法语义，如GET不应对资源产生副作用，PUT应是幂等的全量替换。
