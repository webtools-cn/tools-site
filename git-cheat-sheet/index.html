Git简介

Git是目前最流行的分布式版本控制系统，被全球数百万开发者用于管理代码版本。无论是个人项目还是大型团队协作，Git都能高效地跟踪文件变化、管理分支和合并代码。

核心概念

仓库（Repository）：Git用来存储项目所有文件和历史记录的地方。暂存区（Staging Area）：提交前的临时存储区域。分支（Branch）：独立于主线开发的代码版本。远程（Remote）：托管在网络上的仓库副本，如GitHub、GitLab。

使用场景

版本控制、多人协作开发、代码审查、持续集成/持续部署（CI/CD）、开源项目贡献等。本速查表涵盖了日常开发中最常用的Git命令，支持实时搜索过滤。

Git是什么？

Git是一个分布式版本控制系统，用于跟踪文件变化，协调多人协作开发。由Linus Torvalds于2005年创建，已成为软件开发的标准工具。

git clone和git pull有什么区别？

git clone用于首次从远程仓库复制整个项目到本地，包括所有提交历史。git pull用于将远程仓库的最新更改拉取到已有的本地仓库中，相当于git fetch + git merge。

git merge和git rebase有什么区别？

git merge创建一个新的合并提交来整合两个分支，保留完整的分支历史。git rebase将当前分支的提交移动到目标分支顶部，创建更线性的提交历史。

如何撤销git add？

使用git reset HEAD <文件> 或 git restore --staged <文件> 可以撤销暂存区的文件，使其回到未暂存状态。
