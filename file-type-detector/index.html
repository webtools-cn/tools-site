📖 使用教程

在线文件类型识别器是一款完全免费的网页工具，通过读取文件的Magic Bytes（文件头签名）来自动识别文件格式类型。以下是详细的使用指南：

上传文件：你可以通过两种方式上传文件进行分析。第一种是直接将文件拖放到页面中央的大方框区域（虚线边框区域），支持同时拖放多个文件进行批量识别。第二种是点击方框区域，在弹出的文件选择对话框中选择一个或多个文件。上传后工具会立即读取文件的前64字节进行分析。

查看识别结果：文件上传后，工具会在下方显示详细的分析结果卡片，包含检测到的文件类型、推荐的文件扩展名、MIME类型、文件大小等信息。对于未知文件类型，工具会显示十六进制文件头，帮助你手动判断文件格式。

十六进制视图解读：十六进制视图分为三列：左侧是十六进制偏移量（从文件开头开始的字节位置），中间是对应的字节值（十六进制格式），右侧是对应的ASCII字符表示（不可打印字符显示为点号）。这个视图可以帮助你了解文件的内部结构。

支持的文件类型：工具支持识别图片（PNG/JPEG/GIF/BMP/WebP/TIFF/ICO）、文档（PDF/DOCX/XLSX/PPTX/ZIP/RAR/7Z）、音频（MP3/WAV/FLAC/OGG/AAC）、视频（MP4/AVI/MKV/MOV）、可执行文件（ELF/EXE/Mach-O）以及Web文件（HTML/CSS/JS/JSON/XML/SVG）等50多种常见文件格式。

🎯 应用场景

文件类型识别器在多种场景下都非常实用，以下是几个典型的应用案例：

文件恢复与取证：当文件扩展名被恶意修改（如将.exe重命名为.jpg）或文件扩展名丢失时，可以通过文件头签名准确判断文件的真实类型，避免安全风险。在数字取证工作中，文件头分析是判断文件类型的重要手段。

批量文件整理：当你从各处收集了大量文件，但文件名没有扩展名或扩展名混乱时，可以快速识别每个文件的真实类型并进行分类整理。这对于处理从旧系统中导出的文件尤其有用。

开发调试：Web开发者在处理文件上传功能时，需要确保后端接收的文件类型与预期一致。通过文件头分析可以验证文件上传组件是否正确识别了文件类型，帮助排查Content-Type相关的问题。

学习与教学：计算机科学学生和爱好者可以通过这个工具直观了解不同文件格式的内部结构，学习Magic Bytes的工作原理。十六进制视图可以帮助理解二进制文件的组织方式。

🔬 扩展知识

什么是Magic Bytes？Magic Bytes（魔数/文件签名）是文件开头几个特定的字节序列，用于标识文件的格式类型。这个名称来源于早期Unix系统的file命令使用"magic number"来识别文件类型。例如，PNG图像文件总是以字节序列89 50 4E 47开头，JPEG图像文件总是以FF D8 FF开头。这些签名是国际标准化的，由各种文件格式规范定义。

文件扩展名 vs 文件头：文件扩展名（如.jpg、.png）只是文件名的一部分，可以轻易被修改，因此不可靠。文件头签名是文件内容的固有属性，无法通过重命名文件来更改，因此是判断文件类型更可靠的方法。现代操作系统和安全软件通常都会同时检查文件扩展名和文件头来确认文件类型。

MIME类型：MIME（Multipurpose Internet Mail Extensions）类型是用于标识互联网上文件格式的标准标识符，由IANA维护。例如image/jpeg、application/pdf等。每个MIME类型对应一组文件扩展名和文件签名。文件类型识别器将检测到的签名映射为对应的MIME类型，方便在Web应用中使用。

❓ 常见问题

文件类型识别器的识别原理是什么？

文件类型识别器通过读取文件的前几个字节（称为Magic Bytes或文件签名）来判断文件类型。每种文件格式都有特定的字节签名，例如PNG文件以89 50 4E 47开头，JPEG文件以FF D8 FF开头。工具将这些签名与已知的数据库进行匹配，从而识别文件格式。

可以识别加密的文件或损坏的文件吗？

可以识别加密或损坏文件的原始格式（如果文件头未损坏），但无法读取其内容。加密文件的文件头通常是完整的，因此仍能识别类型。如果文件头损坏，则无法准确识别。

支持哪些文件类型？

支持50多种常见文件类型，包括图片(PNG/JPEG/GIF/BMP/WebP/TIFF/ICO)、文档(PDF/DOCX/XLSX/ZIP/RAR/7Z)、音频(MP3/WAV/FLAC/OGG/AAC)、视频(MP4/AVI/MKV/MOV)、可执行文件(ELF/EXE/Mach-O)以及Web文件(HTML/CSS/JS/JSON/XML)等。

文件会上传到服务器吗？

不会。本工具完全在浏览器本地运行，使用JavaScript的FileReader API读取文件，数据不会离开你的设备。你可以放心使用，即使是包含敏感信息的文件。

识别结果不准确怎么办？

如果识别结果不准确，可能是文件头被修改或文件格式较新。请查看十六进制视图中的文件头信息，并与参考表进行对比。如果确认是新型文件格式，欢迎通过邮件反馈，我们将持续更新识别库。

最大支持多大的文件？

工具仅读取文件的前64字节进行识别，因此理论上支持任意大小的文件。但过大的文件（如几个GB的视频）在浏览器中处理可能需要较长时间，建议对超大文件耐心等待。

可以同时识别多个文件吗？

支持。你可以一次性拖放或选择多个文件，工具会逐个分析并显示结果。每个文件的分析是独立的，结果会依次展示在页面上。

为什么有些文件显示为ZIP格式但实际不是ZIP？

Microsoft Office文档（DOCX、XLSX、PPTX）和现代电子书（EPUB）实际上都是ZIP压缩包，文件内容以XML格式存储在ZIP包内。因此它们与ZIP文件有相同的Magic Bytes。工具会优先识别为Office文档，如果无法确定则显示为ZIP。
