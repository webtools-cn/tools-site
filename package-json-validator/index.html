❓ package.json验证器检查哪些内容？

检查必填字段(name/version)、推荐字段(description/keywords/license/repo)、版本号格式(semver)、依赖版本冲突、scripts语法、engines字段等。

❓ 如何检测依赖冲突？

工具会检查dependencies和devDependencies中是否存在同名但版本不同的包，以及peerDependencies是否在dependencies中声明。

❓ 这个工具会上传我的数据吗？

不会。所有验证均在浏览器本地完成，不发送任何数据到服务器。

❓ 支持哪些package.json版本？

支持npm v7+的package.json格式，包括workspaces、overrides、peerDependenciesMeta等新字段。

❓ 可以修复发现的问题吗？

工具提供修复建议，部分问题可以一键修复（如添加缺失字段、排序字段），修复后可下载更新后的package.json。

❓ 支持从文件导入package.json吗？

支持。可以直接拖拽package.json文件到输入区域，或点击加载示例快速体验。

package.json验证器能做什么？

package.json验证器是一款免费的在线工具，用于检查Node.js项目的package.json文件是否符合规范。支持必填字段检查、版本号格式验证、依赖冲突检测、脚本语法检查等功能，帮助开发者确保package.json配置正确无误。

核心功能

  • 字段完整性检查：验证name、version等必填字段，检查description、keywords、license等推荐字段
  • 版本号格式验证：检查所有依赖版本号是否符合semver规范，检测无效版本范围
  • 依赖冲突检测：检查dependencies和devDependencies中的重复声明和版本冲突
  • 脚本语法检查：验证scripts字段中的命令格式
  • 自动修复：一键修复缺失字段、排序字段顺序等常见问题
  • 文件导入：支持拖拽package.json文件直接验证

使用教程

1. 在输入框中粘贴package.json内容，或拖拽文件导入

2. 点击"验证"按钮，查看检查结果

3. 错误(红色)必须修复，警告(黄色)建议修复，通过(绿色)表示正常

4. 点击"自动修复"一键修复常见问题

5. 使用"下载修复版"保存修正后的package.json

6. 支持快捷键Ctrl+Enter快速验证

应用场景

场景1：NPM包发布前检查

发布NPM包前需要确保package.json完整规范，使用本工具快速检查必填字段和版本号格式。

场景2：项目初始化验证

新项目创建后验证package.json配置是否正确，避免后续安装和构建问题。

场景3：依赖升级审查

升级依赖后检查版本号格式和依赖冲突，确保升级不会引入兼容性问题。

场景4：代码审查

审查PR中的package.json变更时，使用本工具快速验证修改的正确性。

扩展知识

package.json是Node.js项目的核心配置文件，npm v7+支持workspaces、overrides等新字段。必填字段只有name和version，但推荐添加description、keywords、license、repository等字段以提升包的可发现性。版本号遵循semver规范（主版本.次版本.补丁版本），依赖版本范围使用^、~等前缀。peerDependencies用于声明宿主环境需要提供的依赖，npm v7+会自动安装peerDeps。
