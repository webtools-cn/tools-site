上传图片后，您可以通过灰度强度滑块实时调节黑白程度。滑到 0% 时保持原图色彩，滑到 100% 时为完全黑白效果。同时提供四种灰度算法供选择：

  • 加权平均（推荐）：基于人眼视觉感知权重，转换效果最自然

  • 亮度去饱和：基于HSL色彩空间的去饱和操作
  • 去色最大值：取RGB中最大值，对比度较高

扩展知识

什么是灰度图像？

灰度图像是指每个像素仅用单一亮度值表示的图像，不含色彩信息。从 0（纯黑）到 255（纯白）的亮度范围可以表现丰富的明暗层次。灰度图在计算机视觉、医学影像、印刷出版等领域有广泛应用。

灰度算法的区别

不同灰度算法基于不同的数学模型：Gray = 0.299×R + 0.587×G + 0.114×B 是 ITU-R BT.709 标准推荐的加权公式，因为人眼对绿色最敏感、对蓝色最不敏感。简单平均 (R+G+B)/3 计算最快但视觉效果不如加权平均。去饱和算法基于 HSL 空间转换，能保留原始亮度关系。

图片处理完全在本地

本工具使用浏览器原生的 HTML5 Canvas API 直接操作图片像素数据（ImageData），通过 getImageData 读取像素、修改灰度值、再用 putImageData 写回，整个过程不经过任何网络传输。即使是 4K 高清图片也能在毫秒级完成转换。

常见问题 FAQ

Q: 灰度转换会改变图片分辨率吗？

A: 不会。转换后的图片保持原始分辨率不变，仅将颜色信息去除，像素数量、宽高比例均与原图一致。

Q: 支持哪些图片格式？

A: 支持 JPG、PNG、WebP、GIF、BMP 等主流浏览器支持的图片格式。输出格式为 PNG，以保持最高质量。

Q: 图片会上传到服务器吗？

A: 不会。所有处理都在浏览器本地完成，使用 HTML5 Canvas API 进行像素级操作，图片数据不会离开您的设备。

Q: 什么是加权灰度算法？

A: 加权灰度算法根据人眼对不同颜色的敏感度，为红、绿、蓝三通道赋予不同权重（0.299、0.587、0.114），使转换后的黑白效果更符合人眼视觉感受。

Q: 转换后的图片可以商用吗？

A: 可以。本工具不会为图片添加任何水印或限制，转换后的图片完全归您所有，可自由用于个人或商业用途。

Q: GIF动图也能转换吗？

A: 可以上传GIF，但工具会提取其中一帧进行转换，输出为静态PNG图片。如需批量处理GIF所有帧，建议使用专业的GIF编辑工具。
