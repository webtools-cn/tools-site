📖 使用教程

本CSS过渡动画生成器是一款面向前端开发者的可视化工具，帮助您快速生成符合需求的CSS transition代码。以下是详细的使用指南：

第一步：选择过渡属性

从下拉菜单中选择您想要添加过渡效果的CSS属性。常见的属性包括opacity（不透明度）、transform（变换）、background-color（背景色）等。每个属性支持的起始值和目标值格式不同，例如opacity使用0到1之间的数值，transform使用rotate()、scale()等函数。

第二步：设置起始值与目标值

在"起始值"和"目标值"输入框中填写对应的CSS值。例如设置opacity从1变为0.5，或设置transform从scale(1)变为scale(1.2)。工具会自动将这些值应用到预览元素上。

第三步：调整时间与缓动

使用滑块调整过渡持续时间（0.1秒到5秒）和延迟时间。在缓动函数下拉菜单中选择适合的效果，或使用预设的cubic-bezier曲线。自定义cubic-bezier模式允许您精确控制动画的加速和减速过程。

第四步：预览与复制

点击"更新预览"按钮，然后将鼠标悬停在预览区域查看过渡效果。满意后点击"复制代码"即可将生成的CSS代码复制到剪贴板，直接粘贴到您的项目中使用。

🎯 应用场景

CSS过渡动画在现代Web开发中应用广泛，以下是几个典型的使用场景：

按钮交互反馈：当用户将鼠标悬停在按钮上时，通过transition实现背景色渐变、轻微放大或阴影增强的效果，能够显著提升用户的操作反馈感和界面品质感。这种微妙的动画比突然的颜色切换更具专业感。

卡片展开效果：在内容展示页面中，鼠标悬停在信息卡片上时触发宽度和高度的平滑过渡，配合阴影变化，可以创造出层次分明的视觉体验。这种效果常用于产品展示、博客文章列表等场景。

图片滤镜过渡：使用filter属性（如brightness、contrast、blur）的transition效果，可以实现图片从灰度到彩色的平滑过渡，或从清晰到模糊的聚焦效果，非常适合摄影作品集和画廊展示。

导航菜单动画：在响应式导航栏中，下拉菜单的显示和隐藏通过opacity和transform的组合transition实现，比简单的display切换更加流畅自然，提升整体用户体验。

🔬 扩展知识

CSS transition的工作原理：CSS transition通过监听元素属性值的变化来触发过渡效果。当指定的属性值从起始值变为目标值时，浏览器会自动在两者之间插入中间帧，按照设定的持续时间和缓动函数进行插值计算。这个过程由浏览器的合成器线程处理，通常非常高效。

GPU加速与性能：并非所有的CSS属性过渡都能获得GPU加速。只有那些不影响页面布局的属性（如transform和opacity）才能被合成器线程直接处理。对width、height、margin等会触发重排的属性使用transition会导致浏览器重新计算布局，性能开销较大。因此，在追求流畅动画时，应优先使用transform和opacity。

cubic-bezier曲线详解：cubic-bezier函数定义了一条三次贝塞尔曲线，通过两个控制点来精确控制动画的速度变化。P0(0,0)和P3(1,1)是固定的起点和终点，P1(x1,y1)和P2(x2,y2)是两个可调控制点。y值大于x值时动画加速，y值小于x值时动画减速，y值可以超出0-1范围以创造回弹效果。

复合过渡：transition属性支持同时设置多个属性的过渡效果，使用逗号分隔。例如transition: width 0.3s ease, opacity 0.5s linear, transform 0.2s ease-out。每个属性可以拥有独立的持续时间和缓动函数，实现更丰富的动画组合。

❓ 常见问题

CSS transition和animation有什么区别？

CSS transition用于在元素状态变化时（如:hover、:focus、类名切换）产生平滑的过渡效果，需要触发条件。CSS animation使用@keyframes定义关键帧动画，可以自动播放，不需要状态变化触发。transition适合简单的状态过渡，animation适合复杂的连续动画。

transition可以同时设置多个属性吗？

可以。使用逗号分隔多个属性的过渡设置，例如：transition: width 0.3s ease, opacity 0.5s linear。也可以简写为transition: all 0.3s ease，但这样性能较差，建议只指定需要过渡的属性。

哪些CSS属性支持transition？

大多数具有中间值的CSS属性都支持transition，包括opacity、transform、width、height、background-color、color、border-color、border-radius、margin、padding、box-shadow、filter等。不支持transition的属性包括display、position、float、overflow等离散值属性。

cubic-bezier是什么意思？

cubic-bezier是一种定义贝塞尔曲线的函数，用于自定义缓动效果。它接收四个参数(x1, y1, x2, y2)，分别控制曲线的两个控制点坐标。通过调整这四个值，可以创造出线性的、加速的、减速的、弹性的等各种过渡效果。

transition的性能优化建议？

优化transition性能的核心建议是只对那些不会触发重排的属性使用transition。最佳属性是transform和opacity，因为它们可以由GPU加速。避免对width、height、margin、padding等会触发重排的属性使用transition。另外，使用will-change属性提示浏览器提前优化。

transition在哪些场景下会失效？

transition在以下场景可能失效：1) 属性值没有变化；2) 使用了display:none或display:block切换；3) 元素在DOM创建后立即设置样式；4) 在CSS中未指定transition属性。

什么是will-change属性？

will-change是CSS的一个提示属性，用于告诉浏览器某个元素即将发生变化，让浏览器提前做好优化准备。例如will-change: transform可以让浏览器将该元素提升到独立的合成层，从而提升transform和opacity动画的性能。但不应滥用，因为过多的合成层会占用更多内存。

transition支持哪些时间函数？

CSS transition内置了五种时间函数：linear（匀速）、ease（默认，慢-快-慢）、ease-in（慢速开始）、ease-out（慢速结束）、ease-in-out（慢速开始和结束）。此外，可以使用cubic-bezier()自定义曲线，以及steps()实现逐帧动画效果。
